# SCR-011 — Оценка инструктора

| Поле | Значение |
| :-- | :-- |
| **ID** | SCR-011 |
| **Название** | Оценка инструктора |
| **Тип** | Bottom sheet |
| **Приоритет** | Must |
| **Use Case / FR** | UC-006; FR-012 |
| **Источники** | [Q 5.1, 5.2](../1-elicitation/customer-questions.md#5-оценки-инструкторов); [BR-006](../2-requirements/business-requirements.md) |
| **Связанные экраны** | SCR-009 (деталь записи), push-напоминание |

---

## Назначение

Дать клиенту быстрый способ оценить инструктора **после посещённой** тренировки: только звёзды 1–5, без текстового отзыва. Экран не навязывает срок — оценить можно в любой момент после посещения. Повторная оценка одной и той же тренировки запрещена.

---

## Точки входа

| Источник | Условие |
| :-- | :-- |
| **SCR-009** (деталь записи) | Статус брони «Посещена», оценка ещё не выставлена; CTA «Оценить инструктора» |
| **Push-напоминание** | Напоминание об оценке после тренировки (deep link на SCR-011 с `bookingId`) |
| **SCR-008** (мои записи) | Опционально: тап по карточке посещённой записи без оценки → SCR-009 → SCR-011 |

---

## Точки выхода

| Действие | Переход |
| :-- | :-- |
| Отправка оценки | Закрытие sheet, экран благодарности (toast / inline), возврат на SCR-009 |
| Закрытие без оценки (свайп / крестик) | SCR-009 или предыдущий экран без сохранения |
| Ошибка API | Sheet остаётся открытым, сообщение об ошибке, повторная попытка |

---

## Структура экрана

Bottom sheet поверх SCR-009 (или поверх приложения при переходе из push).

```
┌─────────────────────────────────┐
│  ───  (drag handle)             │
│                                 │
│  [Фото]  Имя инструктора        │
│                                 │
│  Как прошла тренировка?         │
│  ☆ ☆ ☆ ☆ ☆   (интерактивные)    │
│                                 │
│  [ Отправить ]  (disabled →     │
│                  enabled)       │
└─────────────────────────────────┘
```

После успешной отправки — краткое сообщение «Спасибо за оценку!» (внутри sheet или toast при закрытии).

## Элементы UI

| Элемент | Описание |
| :-- | :-- |
| Drag handle | Стандартный индикатор bottom sheet |
| Аватар инструктора | Круглое фото; placeholder с инициалами при отсутствии фото |
| Имя инструктора | Из данных слота/брони |
| Заголовок / подсказка | Нейтральная формулировка, напр. «Оцените тренировку с [Имя]» |
| Блок звёзд | 5 крупных тапабельных звёзд; выбранные — заливка, невыбранные — контур |
| Кнопка «Отправить» | Primary; неактивна, пока не выбрана хотя бы 1 звезда |
| Кнопка закрытия | Крестик или свайп вниз — без подтверждения |
| Состояние «уже оценено» | На SCR-009: вместо CTA — «Вы оценили: ★★★★☆» (read-only), sheet не открывается |

**Не включать:** поле текстового отзыва, переключатель анонимности, выбор инструктора (инструктор один — из брони).

## Состояния

| Состояние | Поведение |
| :-- | :-- |
| **Initial** | 0 звёзд, кнопка «Отправить» disabled |
| **Star selected** | 1–5 звёзд подсвечены, кнопка enabled; повторный тап меняет оценку до отправки |
| **Submitting** | Loader на кнопке, звёзды и закрытие заблокированы |
| **Success** | «Спасибо за оценку!», авто-закрытие через 1–2 с или по тапу |
| **Error** | Inline-ошибка под кнопкой, данные оценки сохранены локально |
| **Already rated** | Sheet не показывается; на SCR-009 отображается прошлая оценка |

## Сценарии и переходы

### Основной (из SCR-009)

1. Клиент открывает посещённую запись (SCR-009).
2. Видит CTA «Оценить инструктора».
3. Тап → открывается SCR-011.
4. Выбирает 1–5 звёзд → «Отправить».
5. API сохраняет оценку → благодарность → закрытие → SCR-009 с read-only оценкой.

### Из push

1. Клиент получает push «Оцените тренировку с [Имя]».
2. Тап → deep link открывает SCR-011 с контекстом `bookingId`.
3. Далее — как в основном сценарии.

### Отказ от оценки

1. Клиент закрывает sheet свайпом или крестиком.
2. Возврат на SCR-009; CTA «Оценить» остаётся доступным без ограничения по времени.

### Повторная попытка

1. Клиент с уже выставленной оценкой открывает SCR-009.
2. CTA заменён на отображение звёзд; sheet не предлагается.

## Данные с API

### Вход (при открытии)

| Поле | Источник | Назначение |
| :-- | :-- | :-- |
| `bookingId` | deep link / SCR-009 | Идентификатор брони |
| `instructor.id` | бронь / слот | Отправка оценки |
| `instructor.name` | бронь / слот | Заголовок |
| `instructor.photoUrl` | бронь / слот | Аватар |
| `ratingSubmitted` | бронь | Скрыть CTA, если `true` |
| `existingRating` | бронь | Read-only звёзды на SCR-009 |

### Исход (отправка)

```
POST /bookings/{bookingId}/rating
Body: { "stars": 1..5 }
Response: 201 — успех; 409 — уже оценено; 403 — бронь не «посещена»
```

---

## Правила и ограничения

- Оценка **только после** статуса «Посещена» (Q 5.1).
- **Без ограничения по сроку** — CTA и push доступны в любой момент после посещения (Q 5.1).
- Формат: **только звёзды 1–5**, текстовый отзыв не предусмотрен (Q 5.2).
- **Одна оценка на одну бронь** — повторная отправка блокируется UI и API.
- Оценки **не анонимны** для бэкенда (имя клиента уже в профиле); в UI имя клиента не показывать.
- Публичный рейтинг инструктора на карточках слотов — отдельная логика SCR-004; здесь только ввод оценки.
- Язык интерфейса: **только русский** (Q 9.3).

## Заметки для дизайнера

- Звёзды — главный интерактивный элемент: достаточный размер зоны тапа (≥ 44 pt), чёткая разница между пустой и заполненной звездой.
- Не использовать модальное окно на весь экран — bottom sheet сохраняет контекст записи под ним.
- Сообщение благодарности — короткое, без upsell и без запроса дополнительных действий.
- При переходе из push sheet может открываться поверх любого экрана; предусмотреть загрузку данных брони (skeleton на аватаре и имени).
- Если бронь ещё не в статусе «Посещена» (устаревший push), показать нейтральное сообщение «Оценить можно после тренировки» и кнопку «К записи».
- Согласовать визуал звёзд с отображением рейтинга инструктора на SCR-004 (единая иконография, разный размер).
