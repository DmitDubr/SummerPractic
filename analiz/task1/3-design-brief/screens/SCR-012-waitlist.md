# SCR-012 — Лист ожидания

| Поле | Значение |
| :-- | :-- |
| **ID** | SCR-012 |
| **Название** | Лист ожидания |
| **Тип** | Экран / секция + push-уведомление |
| **Приоритет** | Must |
| **Use Case / FR** | UC-002 (альт.); FR-006, FR-007 |
| **Источники** | [Q 1.4, 6.1, 10.1](../1-elicitation/customer-questions.md); [BR-008, BR-010](../2-requirements/business-requirements.md) |
| **Связанные экраны** | SCR-004 (деталь слота), SCR-005 (оформление записи), SCR-006 (успех), SCR-008 (мои записи), SCR-009 (деталь записи) |

---

## Назначение

Когда группа заполнена (**8/8** на новичковых или **16/16** на остальных), дать клиенту встать в **лист ожидания** вместо отказа. При освобождении места (отмена другого клиента) — **push** с призывом завершить запись. Клиент видит статус очереди в «Мои записи» и может **покинуть лист ожидания** в любой момент.

---

## Точки входа

| Источник | Условие |
| :-- | :-- |
| **SCR-004** (деталь слота) | `freeSpots === 0`; primary CTA «Встать в очередь» вместо «Записаться» |
| **Push «Место освободилось»** | Клиент первый в очереди, место стало доступно; deep link на SCR-005 или SCR-012 |
| **SCR-008 / SCR-009** | Просмотр существующей записи со статусом «В очереди» |

---

## Точки выхода

| Действие | Переход |
| :-- | :-- |
| Подтверждение вступления в очередь | Экран/секция подтверждения → SCR-008 или остаётся на SCR-004 с обновлённым CTA |
| Push → завершить запись | SCR-005 (оформление) → SCR-006 при успехе |
| Покинуть очередь | Подтверждение (лёгкий dialog) → удаление из waitlist → SCR-008 / SCR-004 |
| Истёк срок / место занято другим | Уведомление, статус «Место занято»; предложение вернуться к расписанию (SCR-001) |

---

## Структура экрана

SCR-012 может быть **отдельным экраном** после действия на SCR-004 или **секцией** на SCR-004 / SCR-009. Рекомендуемая композиция:

### A. Вступление в очередь (после SCR-004)

```
┌─────────────────────────────────┐
│  ←  Лист ожидания               │
│                                 │
│  [Карточка слота: дата, время,   │
│   формат, инструктор, 8/8]      │
│                                 │
│  Группа заполнена. Мы сообщим,  │
│  когда освободится место.       │
│                                 │
│  [ Встать в очередь ]           │
│  [ Назад к расписанию ]         │
└─────────────────────────────────┘
```

### B. Подтверждение вступления

```
┌─────────────────────────────────┐
│         ✓                       │
│  Вы в очереди                   │
│  Тренировка: Пт, 18:00          │
│  Болдеринг · Анна               │
│                                 │
│  При освобождении места придёт  │
│  уведомление. У вас будет время │
│  подтвердить запись.            │
│                                 │
│  [ Мои записи ]                 │
└─────────────────────────────────┘
```

### C. В «Мои записи» (SCR-008) и детали (SCR-009)

Карточка со статусом **«В очереди»**, те же поля слота, действие **«Покинуть очередь»**.

## Элементы UI

| Элемент | Описание |
| :-- | :-- |
| Карточка слота | Дата, время, зона/формат, инструктор, счётчик **8/8** или **16/16** |
| Информационный текст | Объяснение механики: уведомление при освобождении места |
| CTA «Встать в очередь» | Primary на SCR-004 / SCR-012 |
| CTA «Покинуть очередь» | Destructive / secondary на SCR-009 |
| Бейдж статуса | «В очереди» — отличимый цвет (напр. янтарный), не путать с «Записан» |
| Push-уведомление | Заголовок: «Место освободилось»; тело: дата, время, формат; кнопка «Записаться» |
| Empty / conflict | Если место уже занято к моменту открытия push — понятное сообщение без мёртвого CTA |

## Состояния

| Состояние | Где | Поведение |
| :-- | :-- | :-- |
| **Слот полон** | SCR-004 | CTA «Встать в очередь»; счётчик 0 свободных |
| **Уже в очереди** | SCR-004 | CTA «Вы в очереди» / переход в SCR-009; повторное вступление недоступно |
| **В очереди (активно)** | SCR-008, SCR-009 | Бейдж «В очереди», кнопка «Покинуть очередь» |
| **Место доступно (push)** | Push + SCR-005 | Окно для завершения записи; таймер удержания места — уточнить с бэкендом |
| **Запись завершена** | SCR-006 | Статус меняется на «Записан»; waitlist закрыт |
| **Покинул очередь** | SCR-008 | Запись исчезает или архивируется |
| **Место занято** | После push | Сообщение «К сожалению, место уже занято»; CTA к расписанию |

## Сценарии и переходы

### Вступление в очередь

1. Клиент открывает SCR-004; группа **8/8** или **16/16**.
2. Тап «Встать в очередь» → SCR-012 (или inline-подтверждение).
3. `POST /waitlist` → экран «Вы в очереди».
4. Запись появляется в SCR-008 со статусом «В очереди».

### Освобождение места (happy path)

1. Другой клиент отменяет бронь заблаговременно (Q 3.4 — место сразу доступно).
2. Первый в очереди получает push **«Место освободилось»** (Q 6.1).
3. Тап на push → SCR-005 с предзаполненным слотом.
4. Клиент завершает оформление (снаряжение, при первой записи — SCR-013) → SCR-006.
5. Статус в SCR-008: «Записан».

### Покинуть очередь

1. SCR-009 → «Покинуть очередь».
2. Лёгкое подтверждение: «Вы потеряете место в очереди».
3. `DELETE /waitlist/{id}` → карточка удалена из SCR-008.

### Конфликт при открытии push

1. Клиент открыл push с задержкой; место уже забронировано.
2. Показать экран/баннер с объяснением; предложить SCR-001 или снова встать в очередь, если слот снова полон.

### Ограничение «1 запись в день» (Q 1.3)

- При попытке встать в очередь на слот в день, когда уже есть активная бронь или другая очередь, показать блокирующее сообщение до вызова API.

## Данные с API

### Вход

| Поле | Назначение |
| :-- | :-- |
| `slotId` | Идентификатор слота |
| `slot.date`, `slot.time` | Отображение в карточке |
| `slot.format`, `slot.instructor` | Контекст тренировки |
| `slot.capacity`, `slot.booked` | 8/8 или 16/16 |
| `waitlistEntry.id` | Покинуть очередь |
| `waitlistEntry.status` | `waiting` \| `spot_available` \| `expired` \| `converted` |
| `waitlistEntry.position` | Опционально: «Вы №2 в очереди» — если бэкенд отдаёт |

### Операции

```
POST   /slots/{slotId}/waitlist          — вступить в очередь
DELETE /waitlist/{waitlistEntryId}       — покинуть очередь
GET    /bookings?status=waitlist         — список для SCR-008
POST   /bookings                         — завершение записи после push (из SCR-005)
```

### Push payload (пример)

```json
{
  "type": "waitlist_spot_available",
  "slotId": "...",
  "waitlistEntryId": "...",
  "deepLink": "vertical://booking/complete?slotId=..."
}
```

## Правила и ограничения

- Лист ожидания доступен **только при полной группе** 8/8 (новичковый формат) или 16/16 (Q 1.4, BR-008).
- **В MVP обязателен** вместе с записью и отменой (Q 10.1).
- Push **«освободилось место»** — обязательный тип уведомления в MVP (Q 6.1); только push, без SMS (Q 6.2).
- Место после отмены клиентом становится доступным **сразу**, в т. ч. для очереди (Q 3.4).
- Один клиент — **одна запись в день**; проверка при вступлении в очередь (Q 1.3).
- Клиент записывает **только себя** (Q 1.2).
- Не показывать лист ожидания, если слот отменён скалодромом или недоступен из-за проката (Q 2.4).
- Статус «В очереди» — отдельный от «Записан»; в реестре экранов согласован с SCR-008, SCR-009.

## Заметки для дизайнера

- На SCR-004 визуально отличать «мест нет» от «можно в очередь»: не серый dead-end, а понятный следующий шаг.
- Бейдж «В очереди» не должен выглядеть как успешная запись — иначе клиент не придёт после push.
- Push: короткий заголовок + конкретика (дата/время); action button «Записаться» в стиле платформы (iOS/Android).
- Экран подтверждения вступления снижает тревожность — явно сказать, что **ничего оплачивать не нужно** до фактической записи (оплата на месте, Q 7.1).
- «Покинуть очередь» — не прятать в overflow; достаточно заметная secondary-кнопка на SCR-009.
- Если бэкенд отдаёт позицию в очереди — показать дискретно («Вы первые в очереди»), без обещания точного времени.
- Deep link из push должен вести сразу к **завершению записи** (SCR-005), а не только к детали слота — иначе место могут занять.
