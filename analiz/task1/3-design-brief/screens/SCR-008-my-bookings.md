# SCR-008 — Мои записи

| Поле | Значение |
| :-- | :-- |
| **ID** | SCR-008 |
| **Название** | Мои записи |
| **Тип** | Экран (вкладка нижней навигации) |
| **Приоритет** | Must |
| **Use Case / FR** | UC-003; FR-007; Q 9.2 |
| **Связанные экраны** | SCR-009 (деталь), SCR-001 (CTA из empty state) |

---

## Назначение

Дать клиенту единый список своих броней с актуальными статусами. Экран — основная точка контроля расписания: что предстоит, что уже прошло, и что изменилось (отмена, посещение, очередь). Поддерживает офлайн-просмотр последнего закэшированного списка (Q 9.2).

---

## Точки входа

| Источник | Действие |
| :-- | :-- |
| Нижняя навигация | Вкладка **«Мои записи»** (вторая вкладка рядом с «Расписание») |
| SCR-006 | Ссылка «Перейти к моим записям» после успешной записи |
| SCR-010 | Возврат после отмены брони (toast + обновлённый список) |
| Push (напоминание, отмена скалодромом) | Deep link ведёт на SCR-009; пользователь может вернуться назад на SCR-008 |

---

## Точки выхода

| Действие | Куда |
| :-- | :-- |
| Тап по карточке брони | SCR-009 (деталь записи) |
| Empty state → «Посмотреть расписание» | SCR-001 |
| Pull-to-refresh (онлайн) | Обновление списка на месте |

---

## Структура экрана

```
┌─────────────────────────────────────┐
│  Мои записи                    [↻]  │  ← заголовок; pull-to-refresh
├─────────────────────────────────────┤
│  [ Предстоящие | Прошедшие ]        │  ← сегментный переключатель
├─────────────────────────────────────┤
│  ⚠ Офлайн. Показаны сохранённые     │  ← баннер (только без сети)
│    данные                           │
├─────────────────────────────────────┤
│  ┌─────────────────────────────┐    │
│  │ Ср, 9 июл · 18:30           │    │
│  │ Трассы с верёвкой           │    │
│  │ Инструктор: Алексей К.      │    │
│  │ [Записан]                   │    │  ← карточка брони
│  └─────────────────────────────┘    │
│  ┌─────────────────────────────┐    │
│  │ ...                         │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
│  Расписание  |  Мои записи  |  ...  │  ← нижняя навигация
└─────────────────────────────────────┘
```

**Сегменты:**

- **Предстоящие** — брони, у которых время начала слота ≥ текущего момента (статусы: активна, лист ожидания; отменённые скалодромом с будущей датой слота тоже здесь).
- **Прошедшие** — брони, у которых время начала слота < текущего момента (статусы: посещена, отменена клиентом, отменена скалодромом).

Сортировка внутри сегмента: по дате/времени слота (предстоящие — от ближайших к дальним; прошедшие — от недавних к старым).

## Элементы UI

| Элемент | Описание |
| :-- | :-- |
| Заголовок | «Мои записи», фиксированный |
| Сегментный контрол | Две вкладки: **Предстоящие** / **Прошедшие** |
| Офлайн-баннер | Жёлтый/предупреждающий блок под сегментами; текст: «Нет сети. Показаны сохранённые данные» |
| Карточка брони | Дата и время слота, зона/формат, имя инструктора, бейдж статуса |
| Бейдж статуса | См. таблицу статусов ниже |
| Empty state | Иллюстрация + заголовок + подзаголовок + CTA |
| Индикатор загрузки | Скелетоны карточек при первой загрузке (онлайн) |

**Бейджи статусов на карточке:**

| Статус (API) | Текст бейджа | Цвет (рекомендация) |
| :-- | :-- | :-- |
| `active` | Записан | Зелёный / primary |
| `waitlist` | В очереди | Синий / info |
| `cancelled_by_client` | Отменена вами | Нейтральный / серый |
| `cancelled_by_gym` | Отменена скалодромом | Оранжевый / warning |
| `attended` | Посещена | Нейтральный / серый |

На карточке **не** показывать причину отмены скалодромом — только бейдж; детали на SCR-009.

## Состояния

| Состояние | Поведение |
| :-- | :-- |
| **Загрузка (онлайн)** | Скелетоны 3–4 карточек |
| **Данные (онлайн)** | Актуальный список с API; баннер офлайна скрыт |
| **Офлайн с кэшем** | Баннер виден; список из локального кэша; pull-to-refresh недоступен или показывает «Нет сети» |
| **Офлайн без кэша** | Баннер + empty state: «Нет сохранённых записей. Подключитесь к интернету» |
| **Empty — Предстоящие** | «У вас нет предстоящих записей» + CTA «Посмотреть расписание» → SCR-001 |
| **Empty — Прошедшие** | «Здесь появятся завершённые тренировки» (без CTA или с тем же CTA на усмотрение дизайна) |
| **Ошибка сети (первая загрузка)** | Сообщение об ошибке + кнопка «Повторить» |
| **Ошибка + есть кэш** | Показать кэш + баннер «Не удалось обновить. Показаны сохранённые данные» |

## Сценарии и переходы

1. **Открытие вкладки (онлайн):** запрос `GET /bookings` → отображение списка → сохранение в кэш.
2. **Переключение сегмента:** фильтрация локально, без повторного запроса.
3. **Тап по карточке:** переход на SCR-009 с `booking_id`.
4. **Pull-to-refresh:** повторный запрос; при успехе — обновление списка и кэша.
5. **Потеря сети при открытом экране:** показать офлайн-баннер; данные не пропадают.
6. **Возврат после отмены (SCR-010):** toast «Запись отменена»; карточка переходит в «Прошедшие» / обновляет бейдж на «Отменена вами».
7. **Push об отмене скалодромом:** пользователь может открыть SCR-009 по deep link; при возврате на SCR-008 карточка уже с бейджем «Отменена скалодромом».

## Данные с API

**Запрос:** `GET /bookings` (или `GET /bookings?segment=upcoming|past` — на усмотрение бэкенда; UI может фильтровать на клиенте).

**Поля для карточки (минимум):**

| Поле | Использование на экране |
| :-- | :-- |
| `id` | Навигация на SCR-009 |
| `status` | Бейдж |
| `slot.starts_at` | Дата и время |
| `slot.format` / `slot.zone` | Зона/формат тренировки |
| `slot.instructor.name` | Имя инструктора |
| `waitlist_position` | Опционально на карточке: «Место в очереди: N» (основной блок — на SCR-009) |

**Кэш (офлайн):** последний успешный ответ `GET /bookings` + timestamp синхронизации (для подписи «Обновлено …» — опционально, не обязательно в MVP).

## Правила и ограничения

- Офлайн доступен **только для просмотра**; отмена, выход из очереди и оценка — только онлайн (действия на SCR-009/010).
- Все тексты — **только на русском** (Q 9.3).
- Статусы отображаются согласно реестру экранов и FR-007.
- Бронь со статусом «лист ожидания» показывается в **Предстоящие**.
- Не показывать цену на карточке списка (цена — на детальном экране, Q 7.1).
- Не более 1 активной записи в день (Q 1.3) — на уровне бэкенда; UI просто отображает факт.

## Заметки для дизайнера

- Карточка должна быть целиком кликабельной (tap target ≥ 44 pt).
- Бейдж статуса — в правом верхнем углу карточки или под заголовком даты; контраст достаточный для чтения.
- Офлайн-баннер не перекрывает контент и не исчезает автоматически — только при восстановлении сети.
- Empty state для «Предстоящие» — дружелюбный, с явным CTA на расписание; визуально согласовать с empty state SCR-001.
- Скелетоны повторяют структуру карточки (дата, формат, инструктор, бейдж).
- При проектировании учесть, что в «Прошедших» могут быть и отменённые, и посещённые — визуально различать бейджами, не смешивать в один «серый» стиль.
- Нижняя навигация: вкладка «Мои записи» в активном состоянии при нахождении на SCR-008 и SCR-009 (деталь — дочерний экран, не отдельная вкладка).
