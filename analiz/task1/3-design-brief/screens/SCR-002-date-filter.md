# SCR-002 — Фильтр периода дат

| Поле | Значение |
| :-- | :-- |
| ID | SCR-002 |
| Тип | Bottom sheet (модальный оверлей) |
| Приоритет | Must |
| Связь | UC-001; FR-002; US-002; Q 1.5; R-027 |

## Назначение

Позволяет клиенту расширить или сузить период просмотра расписания **за пределами дефолтных 7 дней** (R-027, FR-002). Открывается поверх SCR-001; после применения обновляет список слотов и отображает выбранный диапазон чипом на главном экране.

## Точки входа

- SCR-001: тап по чипу **«Период»** / **«7 дней»** / **«Ближайшие 7 дней»**.
- SCR-001 empty state: CTA «Изменить период» (если предусмотрен в макете empty).

## Точки выхода

| Действие | Результат |
| :-- | :-- |
| **Применить** (валидный диапазон) | Закрытие sheet → SCR-001 с новым периодом, перезагрузка слотов, обновление чипа периода |
| **Сбросить** | Возврат к дефолту 7 дней → закрытие sheet → SCR-001 |
| Свайп вниз / тап по затемнённой области | Закрытие **без применения** (откат к последнему применённому периоду) |
| Системная кнопка «Назад» (Android) | То же, что свайп вниз |

## Структура экрана

```
┌─────────────────────────────────────┐
│  ░░░░░░░ SCR-001 (затемнён) ░░░░░░░ │
├─────────────────────────────────────┤
│  ───  (drag handle)                 │
│  Период                             │
│                                     │
│  ┌─────────────────────────────┐    │
│  │   Календарь / date range      │    │  ← выбор «с» и «по»
│  │   picker                      │    │
│  └─────────────────────────────┘    │
│                                     │
│  С  12 июля 2026                    │  ← текстовое отображение выбора
│  По 25 июля 2026                    │
│                                     │
│  [ Сбросить ]    [ Применить ]      │  ← футер sheet, sticky
└─────────────────────────────────────┘
```

Высота sheet: ~70–85 % экрана на телефоне; на маленьких экранах допустим полноэкранный modal с кнопкой «Закрыть».

## Элементы UI

### Drag handle

- Горизонтальная полоска по центру верхней кромки sheet — сигнал свайпа для закрытия.

### Заголовок

- **«Период»** — по центру или слева, 18–20 sp, semibold.

### Date range picker

- Компонент выбора **диапазона дат** (дата начала + дата окончания).
- Рекомендуемый паттерн: календарная сетка с выделением диапазона (Material Date Range Picker / iOS inline calendar).
- Первая дата диапазона — **не раньше сегодня** (прошлые дни disabled).
- Вторая дата — **не раньше** первой.
- Подсветка «сегодня» отдельным стилем.

### Текстовое резюме выбора

- Две строки под календарём:
  - **«С»** — «12 июля 2026»
  - **«По»** — «25 июля 2026»
- Обновляются синхронно с календарём.
- Если выбрана только начальная дата — «По» в состоянии placeholder: «Выберите дату окончания»; кнопка «Применить» disabled.

### Кнопки действий (футер)

| Кнопка | Стиль | Поведение |
| :-- | :-- | :-- |
| **Сбросить** | Secondary / text | Сброс к дефолту 7 дней, немедленное применение и закрытие |
| **Применить** | Primary | Применить выбранный диапазон и закрыть |

Кнопки в одной строке, равная ширина или «Применить» accent справа.

### Подсказка (опционально)

- Мелкий текст под заголовком: «По умолчанию показаны ближайшие 7 дней».

## Состояния (loading / empty / error / offline)

### Loading

- При открытии sheet **не требует** загрузки — календарь локальный.
- При «Применить»: на SCR-001 показывается loading/skeleton; на sheet кратковременно disabled кнопка «Применить» + спиннер (опционально) до закрытия.

### Empty

- Не применимо к самому sheet. Пустой результат после применения обрабатывается на SCR-001 (empty state FR-004).

### Error

- Валидация на клиенте:
  - Не выбрана конечная дата → «Применить» disabled.
  - Диапазон превышает лимит (см. правила) → inline-сообщение под календарём: «Максимальный период — N дней».
- Ошибка API после применения — на SCR-001 (error state), sheet уже закрыт.

### Offline

- Sheet открывается; при «Применить» без сети — toast на SCR-001: «Нет подключения. Проверьте интернет»; sheet можно оставить открытым или закрыть без смены периода.

## Сценарии и переходы

| # | Сценарий | Поведение |
| :-- | :-- | :-- |
| 1 | Открытие при дефолте 7 дней | Календарь с предвыбранным диапазоном: сегодня → сегодня+6 |
| 2 | Открытие при кастомном периоде | Календарь показывает последний применённый диапазон |
| 3 | Выбор нового диапазона | Подсветка в календаре + обновление строк «С» / «По» |
| 4 | Применить | `dateFrom`, `dateTo` → запрос слотов → SCR-001; чип: «12–25 июл» |
| 5 | Сбросить | Период = 7 дней; чип SCR-001: «7 дней» (без акцента) |
| 6 | Закрыть без применения | Период на SCR-001 не меняется |
| 7 | Диапазон без слотов | SCR-001 показывает empty «Пока нет доступных тренировок» |

## Данные с API

Sheet **не вызывает** отдельный endpoint. Результат выбора передаётся в запрос SCR-001:

```
GET /api/v1/slots?dateFrom=YYYY-MM-DD&dateTo=YYYY-MM-DD
```

Параметры сохраняются в состоянии экрана SCR-001 (или глобальном store фильтров) до сброса или смены пользователем.

### Ограничения диапазона (рекомендация для контракта)

| Параметр | Значение |
| :-- | :-- |
| Минимум | 1 день (from = to допустимо) |
| Максимум | 30 календарных дней (уточнить с бэкендом; в UI — disabled даты за пределами) |
| Начало | ≥ сегодня |

Если бэкенд вернёт ошибку 400 на слишком широкий диапазон — показать сообщение на SCR-001.

## Правила и ограничения

- Дефолт при первом входе и после «Сбросить»: **7 дней** от сегодня включительно (R-027, Q 1.5).
- Формат чипа на SCR-001 при кастомном периоде: краткий, напр. **«12–18 июл»**; если разные месяцы — **«28 июл – 3 авг»**.
- При дефолтном периоде чип **не** должен выглядеть как «активный фильтр» (нейтральный стиль).
- Только русская локаль календаря (названия месяцев, понедельник — первый день недели по российской привычке).
- Не смешивать с фильтрами слотов (SCR-003): период и слотовые фильтры **комбинируются** (AND).
- Accessibility: озвучивание выбранного диапазона для screen reader.

## Заметки для дизайнера

- Bottom sheet — стандартный паттерн для фильтров; не перегружать второстепенными опциями (пресеты «На неделю» / «На месяц» — nice-to-have, не в MVP).
- Календарь должен быть **крупным** — пользователи в зале, возможно при слабом освещении.
- Чётко различать **Сбросить** (вернуть 7 дней) и закрытие жестом (отмена текущего редактирования).
- После применения длинного периода список на SCR-001 может быть длинным — предусмотреть быстрый скролл к «Сегодня» (опционально FAB «К сегодня» — Could, не MVP).
- Прорисовать состояния: дефолтный предвыбор, неполный выбор (disabled Apply), ошибка max range.
- Согласовать с SCR-001 отображение чипа периода в активном и неактивном состоянии.
