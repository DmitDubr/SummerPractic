# SCR-013 — Контактные данные (профиль при записи)

| Поле | Значение |
| :-- | :-- |
| **ID** | SCR-013 |
| **Название** | Контактные данные |
| **Тип** | Секция на SCR-005 / bottom sheet (без отдельной вкладки) |
| **Приоритет** | Must |
| **Use Case / FR** | UC-002; FR-005, FR-016 |
| **Источники** | [Q 1.1, 7.2](../1-elicitation/customer-questions.md); [Q 10.1](../1-elicitation/customer-questions.md#10-приоритизация-mvp) |
| **Связанные экраны** | SCR-005 (оформление записи), SCR-006 (успех), SCR-008 (мои записи — косвенно) |

---

## Назначение

Собрать **имя и телефон** клиента при **первой записи** — этого достаточно для идентификации без полноценной регистрации (Q 1.1). Для возвращающихся клиентов поля предзаполняются. Бейдж **«Постоянный клиент»** — только визуальная метка лояльности, без скидок и приоритетов (Q 7.2). Отдельный таб «Профиль» в MVP **не нужен**.

---

## Точки входа

| Источник | Условие |
| :-- | :-- |
| **SCR-005** (оформление записи) | Первая запись: `profile.isComplete === false` — секция контактов **встроена** в форму, блокирует отправку до заполнения |
| **SCR-005** | Повторная запись: поля предзаполнены, редактирование по ссылке «Изменить контакты» |
| **Ссылка в форме** | «Контактные данные» → bottom sheet с теми же полями (для правки перед отправкой) |

---

## Точки выхода

| Действие | Переход |
| :-- | :-- |
| Сохранение в составе SCR-005 | Продолжение оформления записи на том же экране |
| Сохранение из sheet | Закрытие sheet, обновление сводки на SCR-005 |
| Успешная запись | SCR-006; профиль сохранён на бэкенде |
| Отмена / назад | SCR-004 без сохранения черновика профиля (кроме уже сохранённого на сервере) |

## Структура экрана

### Вариант 1: Inline-секция на SCR-005 (первая запись)

```
┌─────────────────────────────────┐
│  Оформление записи              │
│  ... (слот, снаряжение) ...     │
│                                 │
│  ── Контактные данные ──        │
│  [🏷 Постоянный клиент]         │  ← только если API flag
│                                 │
│  Имя *                          │
│  [________________]             │
│                                 │
│  Телефон *                      │
│  [+7 (___) ___-__-__]           │
│                                 │
│  [ Записаться ]                 │
└─────────────────────────────────┘
```

### Вариант 2: Bottom sheet (редактирование)

```
┌─────────────────────────────────┐
│  ───                            │
│  Контактные данные              │
│  [🏷 Постоянный клиент]         │
│  Имя *    [________]            │
│  Телефон *[+7 (___)...]         │
│  [ Сохранить ]                  │
└─────────────────────────────────┘
```

На SCR-005 при заполненном профиле — компактная сводка: «Иван · +7 900 ***-45-67» + ссылка «Изменить».

## Элементы UI

| Элемент | Описание |
| :-- | :-- |
| Заголовок секции | «Контактные данные» или «Как к вам обращаться» |
| Поле «Имя» | Текстовый input, обязательный; placeholder «Ваше имя» |
| Поле «Телефон» | Маска **+7 (XXX) XXX-XX-XX**; цифровая клавиатура; обязательный |
| Бейдж «Постоянный клиент» | Неброский chip/badge рядом с заголовком; **только отображение**, без tooltip со скидками |
| Сводка (повторный визит) | Имя + маскированный телефон; линк «Изменить» |
| Валидация inline | Под полем: «Укажите имя», «Введите корректный номер» |
| Кнопка основного действия | На SCR-005 — общая «Записаться»; в sheet — «Сохранить» |

**Не включать:** email, пароль, аватар, отдельный экран настроек, переключатели уведомлений.

## Состояния

| Состояние | Поведение |
| :-- | :-- |
| **Первая запись, пустой профиль** | Оба поля пустые, бейдж скрыт (если не regular), «Записаться» disabled до валидных данных |
| **Первая запись, regular client** | Бейдж виден сразу после загрузки профиля с флагом `isRegularClient` |
| **Повторная запись** | Сводка предзаполнена; поля в основном потоке свёрнуты |
| **Редактирование (sheet)** | Поля с текущими значениями; «Сохранить» активна при изменениях и валидности |
| **Валидация ошибки** | Красная обводка поля + текст ошибки; фокус на первое невалидное |
| **Сохранение** | Loader на кнопке SCR-005 или sheet |
| **Офлайн** | Блокировка новой записи, если профиль не сохранён локально и нет сети — сообщение на SCR-005 |

## Сценарии и переходы

### Первая запись

1. Клиент выбирает слот → SCR-005.
2. Секция SCR-013 развёрнута, поля пустые.
3. Вводит имя и телефон → валидация в реальном времени.
4. Выбирает снаряжение → «Записаться».
5. `PATCH /profile` + `POST /bookings` → SCR-006.

### Повторная запись

1. SCR-005 загружает профиль → сводка «Мария · +7 916 ***-12-34».
2. Клиент может сразу записаться или тапнуть «Изменить».
3. Sheet SCR-013 → правка → «Сохранить» → возврат на SCR-005.

### Постоянный клиент

1. API возвращает `isRegularClient: true`.
2. На секции отображается бейдж «Постоянный клиент».
3. Никаких изменений в цене, приоритете записи или UI кнопок (Q 7.2).

### Waitlist → завершение записи

1. Push по SCR-012 → SCR-005.
2. Если профиль уже есть — только снаряжение + подтверждение.
3. Если первый визит — снова полная секция SCR-013.

## Данные с API

### Чтение профиля

```
GET /profile
```

| Поле | Тип | UI |
| :-- | :-- | :-- |
| `name` | string | Поле «Имя» / сводка |
| `phone` | string (E.164 или нормализованный) | Маска +7 |
| `isComplete` | boolean | Показать полную секцию vs сводку |
| `isRegularClient` | boolean | Бейдж «Постоянный клиент» |

### Сохранение

```
PATCH /profile
Body: { "name": "...", "phone": "+79001234567" }
```

Профиль может сохраняться **до** или **вместе с** бронированием; UI должен гарантировать наличие валидных контактов до `POST /bookings`.

### Локальное хранение

- После первого успешного сохранения — кэш для предзаполнения и офлайн-просмотра в «Мои записи» (имя в деталях брони — по данным бэкенда).

## Правила и ограничения

- Для записи достаточно **имени и телефона**, отдельная регистрация/логин не требуется (Q 1.1).
- Оба поля **обязательны** при первой записи.
- Телефон: формат РФ, маска ввода, валидация длины (10 цифр после +7).
- Бейдж постоянного клиента — **только визуальный**, без скидок, приоритета в очереди и отдельного раздела лояльности (Q 7.2).
- **Отдельная вкладка «Профиль» не создаётся** — inline на SCR-005 или sheet по ссылке.
- Один человек записывает только себя — имя в профиле = участник тренировки (Q 1.2).
- Язык: только русский (Q 9.3).
- Цену на SCR-005 показываем, оплата на месте (Q 7.1) — в SCR-013 не дублировать.

## Заметки для дизайнера

- Секция должна ощущаться частью формы записи, а не «регистрацией в приложении» — минимум полей, дружелюбный тон.
- Маска телефона: автоподстановка `+7`, курсор не прыгает при вводе; поддержка вставки из буфера с нормализацией.
- Бейдж «Постоянный клиент» — позитивный, но скромный (не как промо-баннер); без иконки «скидка».
- На повторных визитах не заставлять видеть большую форму — **сводка + «Изменить»** экономит вертикальное пространство SCR-005.
- Sheet для правки контактов — тот же визуальный язык, что SCR-002/SCR-003 (единый паттерн bottom sheet в приложении).
- Не запрашивать разрешения (контакты, SMS) на этом экране — только ручной ввод.
- Доступность: label связан с input, ошибки озвучиваются screen reader, контраст бейджа достаточный.
- Если имя уже известно бэкенду, но телефон пустой (краевой случай миграции) — показать только недостающее поле.
