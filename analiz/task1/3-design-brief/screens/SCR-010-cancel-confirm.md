# SCR-010 — Подтверждение отмены

| Поле | Значение |
| :-- | :-- |
| **ID** | SCR-010 |
| **Название** | Подтверждение отмены |
| **Тип** | Dialog / bottom sheet (модальный) |
| **Приоритет** | Must |
| **Use Case / FR** | UC-004; FR-008; Q 3.1, 3.2, 3.4 |
| **Связанные экраны** | SCR-009 (вызов), SCR-008 (возврат после успеха) |

---

## Назначение

Запросить явное подтверждение отмены активной брони. При поздней отмене (менее чем за 1 час до начала) показать предупреждение без блокировки действия в MVP. После успешной отмены — уведомить пользователя и вернуть к списку записей.

---

## Точки входа

| Источник | Действие |
| :-- | :-- |
| SCR-009 | Тап «Отменить запись» (только статус `active`, онлайн) |

---

## Точки выхода

| Действие | Куда |
| :-- | :-- |
| «Оставить запись» / закрытие / тап вне sheet | Закрытие модала, остаёмся на SCR-009 |
| «Отменить запись» (подтверждение) | API → toast → SCR-008 (или SCR-009 с обновлённым статусом + назад на SCR-008) |
| Ошибка API | Модал закрывается или остаётся открытым с ошибкой + «Повторить» |

**Рекомендуемый поток после успеха:** закрыть модал → toast «Запись отменена» → навигация на SCR-008 с обновлённым списком (сегмент «Прошедшие» или обновление «Предстоящие»).

---

## Структура экрана

### Вариант A — Bottom sheet (рекомендуется)

```
┌─────────────────────────────────────┐
│         (SCR-009 затемнён)          │
├─────────────────────────────────────┤
│  ───                                │
│  Отменить запись?                   │
│                                     │
│  Ср, 9 июл · 18:30                  │
│  Трассы с верёвкой                  │
│                                     │
│  ┌─ Предупреждение (если < 1 ч) ─┐  │
│  │ ⚠ До начала меньше часа.      │  │
│  │   Место может не успеть       │  │
│  │   достаться другим.           │  │
│  └───────────────────────────────┘  │
│                                     │
│  [ Оставить запись ]    primary     │
│  [ Отменить запись ]    destructive │
└─────────────────────────────────────┘
```

### Вариант B — Center dialog

Та же структура контента, компактнее по вертикали; две кнопки в столбик или в ряд (на маленьких экранах — столбик).

## Элементы UI

| Элемент | Описание |
| :-- | :-- |
| Заголовок | «Отменить запись?» |
| Краткое напоминание | Дата, время, формат тренировки (из контекста SCR-009) |
| Блок предупреждения | **Только если** до `slot.starts_at` < 60 минут; жёлтый фон, иконка ⚠ |
| Текст предупреждения | «До начала тренировки осталось меньше часа. Место может не успеть освободиться для других участников.» |
| Кнопка «Оставить запись» | Primary / filled — безопасное действие по умолчанию |
| Кнопка «Отменить запись» | Destructive (красный текст или outline) — подтверждение отмены |
| Индикатор загрузки | На кнопке «Отменить запись» во время запроса; обе кнопки disabled |
| Toast (после успеха) | «Запись отменена» — вне модала, на SCR-008 |

**Блок предупреждения не показывается**, если до начала ≥ 1 час (заблаговременная отмена, Q 3.2).

---

## Состояния

| Состояние | Поведение |
| :-- | :-- |
| **Ранняя отмена (≥ 1 ч)** | Заголовок + напоминание + две кнопки; без жёлтого блока |
| **Поздняя отмена (< 1 ч)** | Жёлтый блок предупреждения; отмена **разрешена** (MVP, Q 3.1) |
| **Отправка запроса** | Loading на confirm-кнопке |
| **Успех** | Модал закрыт → toast → SCR-008 |
| **Ошибка сети** | «Нет подключения. Проверьте интернет» в модале |
| **Ошибка сервера (4xx/5xx)** | «Не удалось отменить запись» + «Повторить» / «Закрыть» |
| **Бронь уже отменена** | «Запись уже отменена» → закрыть → обновить SCR-009/008 |

---

## Сценарии и переходы

1. **Заблаговременная отмена (UC-004, ≥ 1 ч):**
   - SCR-009 → «Отменить запись» → SCR-010 (без предупреждения)
   - «Отменить запись» → `POST /bookings/{id}/cancel` → место освобождается (Q 3.4)
   - Toast → SCR-008; бронь со статусом `cancelled_by_client`

2. **Поздняя отмена (< 1 ч, MVP):**
   - SCR-010 с жёлтым предупреждением
   - Клиент всё равно может подтвердить отмену (штрафов нет, Q 3.1)
   - Место освобождается на бэкенде; поздняя отмена учитывается для аналитики (Q 3.3)

3. **Отказ от отмены:**
   - «Оставить запись» или свайп вниз / тап на overlay → модал закрыт, SCR-009 без изменений

4. **Двойной тап:**
   - Debounce на confirm; повторный запрос не отправлять

## Данные с API

**Входные данные (из SCR-009, без отдельного запроса):**

| Поле | Использование |
| :-- | :-- |
| `booking.id` | Параметр отмены |
| `slot.starts_at` | Расчёт: `starts_at - now < 1 hour` → показать предупреждение |
| `slot.format` | Текст напоминания |

**Запрос при подтверждении:**

```
POST /bookings/{id}/cancel
```

**Ответ (успех):**

| Поле | Использование |
| :-- | :-- |
| `status` | `cancelled_by_client` |
| `cancelled_at` | Опционально для детали |

Клиент после успеха инвалидирует кэш списка (SCR-008) и детали (SCR-009).

---

## Правила и ограничения

- Модал доступен **только онлайн**; на SCR-009 в офлайне кнопка «Отменить запись» disabled.
- В MVP **нет штрафов** и **нет блокировки** поздней отмены — только предупреждение (Q 3.1).
- Порог «заранее» = **1 час и более** до начала (Q 3.2).
- При заблаговременной отмене место сразу доступно другим и для листа ожидания (Q 3.4, FR-008).
- Повторное открытие SCR-010 для уже отменённой брони невозможно (кнопка на SCR-009 скрыта).
- Тексты только на русском.

**Логика порога (для разработки и QA):**

```
is_late_cancel = (slot.starts_at - now) < 1 hour AND (slot.starts_at - now) > 0
```

Если тренировка уже началась — кнопка отмены на SCR-009 не показывается (вне скоупа SCR-010).

## Заметки для дизайнера

- **Безопасное действие по умолчанию** — «Оставить запись» (primary, верхняя или левая кнопка). Destructive — вторичная по визуальному весу, но доступная.
- Жёлтый блок предупреждения — не error (не красный); иконка warning, читаемый контраст.
- Не пугать формулировками про штрафы — в MVP их нет; текст нейтрально-информативный.
- Bottom sheet предпочтительнее center dialog: на мобильном удобнее дотянуться до кнопок; sheet можно закрыть свайпом (= «Оставить запись»).
- Toast после успеха — краткий, 2–3 сек, не блокирует взаимодействие со списком.
- При loading не закрывать модал до ответа API.
- Зарезервировать место под блок предупреждения в макете, чтобы layout не прыгал при появлении жёлтого блока (или анимировать появление плавно).
- Для согласованности: тот же стиль модалов, что SCR-007 (ошибка записи) и confirm «Покинуть очередь» на SCR-009.
