# Вопросы и ответы заказчика — скалодром «Вертикаль»

> Этап выявления требований. Источник: [brief-climbing.md](../0-customer-brief/brief-climbing.md).
> Дата фиксации ответов: 03.07.2026.

---

## 1. Запись и расписание

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 1.1 | Кто может записаться на тренировку? Нужна ли регистрация/авторизация, или достаточно телефона и имени? | Достаточно **имени и телефона** при первой записи |
| 1.2 | Может ли один человек записать нескольких участников (ребёнок, друзья)? | **Нет** — один человек записывает только себя |
| 1.3 | Сколько активных записей может иметь один клиент одновременно? Можно ли записаться на две тренировки в один день? | Не более **1 записи в день** на одного человека |
| 1.4 | Если группа заполнена (8/8 или 16/16) — показываем только «мест нет» или нужен лист ожидания при отмене? | **Лист ожидания** |
| 1.5 | **Подтверждение (R-027):** по умолчанию показываем слоты на 7 дней, более длинный период — через фильтр дат? | **Да**, подтверждено брифом |
| 1.6 | Нужны ли клиенту фильтры: формат (болдеринг / трассы), инструктор, время суток, уровень? | **Инструктор**, **время суток**, **уровень** (формат болдеринг/трассы — не в первой версии) |

---

## 2. Форматы тренировок и безопасность

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 2.1 | Как система определяет «новичковую» тренировку — отдельный тип слота в расписании или правило «если формат X — лимит 8»? | **Правило по формату** (лимит 8 человек привязан к формату, не к отдельному типу слота) |
| 2.2 | Нужна ли проверка уровня клиента? Может ли новичок записаться на «трассы посложнее»? | **Нет** — запись на усмотрение клиента |
| 2.3 | Что именно выбирает клиент при прокате — скальники, страховочная система, оба? Влияет ли выбор на цену? | Клиент **сам выбирает**, что нужно (скальники / страховка / оба). На цену влияет **незначительно**; отдельно — случай **потери прокатного оборудования** |
| 2.4 | Ограничен ли прокатный фонд снаряжения? Что показывать, если прокат на слот закончился? | Слот **недоступен** для записи |

---

## 3. Отмена клиентом

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 3.1 | Правило «за 10 минут до начала» — это жёсткий запрет отмены, штраф, предупреждение или только пометка для администратора? | В MVP — **предупреждение**; **штрафы** — вторая итерация |
| 3.2 | Что считается «заранее» — 24 часа, 3 часа, любое время до начала слота? | **За 1 час и более** до начала |
| 3.3 | Нужен ли учёт частых поздних отмен и неявок? Есть ли последствия для клиента? | Учитываются только **поздние отмены** (неявки отдельно не трекаем) |
| 3.4 | После отмены клиентом место сразу становится доступным для других в приложении? | Место **сразу** доступно другим (и для листа ожидания) |

---

## 4. Отмена скалодромом и профилактика

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 4.1 | **Подтверждение (R-008):** при отмене скалодромом бронь получает статус «Отменена скалодромом» с причиной, клиенту приходит push, повторная запись на тот же слот запрещена? | Push-уведомление с возможностью **перезаписаться на другой день/время**. На **тот же отменённый слот** — запрещено. Статус «Отменена скалодромом» + причина — по брифу |
| 4.2 | Если позже подключим онлайн-оплату — что делать с деньгами при отмене скалодромом: возврат, перенос, абонемент? | Если клиент **согласен на перенос** — возврат не нужен. Если **отказывается переносить** — **возврат** |

## 5. Оценки инструкторов

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 5.1 | Когда клиент может поставить оценку — только после посещённой тренировки? В течение какого срока? | **Только после посещённой** тренировки; срок — **без ограничений** |
| 5.2 | Какой формат оценки: звёзды 1–5, текстовый отзыв, анонимно или с именем? | Только **звёзды** (без текстового отзыва) |
| 5.3 | Видят ли другие клиенты рейтинги инструкторов при выборе слота, или оценки только для владельца? | **Клиенты видят** оценки при выборе слота |

## 6. Уведомления

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 6.1 | Какие уведомления обязательны в MVP: напоминание за день / за 2 часа, подтверждение записи, отмена, освободилось место? | **Все перечисленные:** напоминание за день, за 2 часа, подтверждение записи, отмена (клиентом / скалодромом), освободилось место (лист ожидания) |
| 6.2 | Достаточно ли push в приложении, или нужны SMS / email / Telegram? | Только **push** в приложении |

## 7. Оплата и лояльность

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 7.1 | В MVP без онлайн-оплаты клиент записывается бесплатно и платит на месте? Нужно ли показывать цену в приложении? | **Платить на месте**; **цену в приложении показываем** (онлайн-оплата не в первой версии) |
| 7.2 | Что именно нужно для «постоянных клиентов» в первой версии — метка, скидка, приоритет записи? Или откладываем? | **Отмечаем** в системе — без скидок и приоритетов на старте |

## 10. Приоритизация MVP

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 10.1 | При ограниченном бюджете что точно входит в первую версию: запись + просмотр своих записей + отмена? | **Всё перечисленное:** запись, мои записи, отмена, лист ожидания |
| 10.2 | Оценки инструкторов и уведомления — must-have или можно во вторую итерацию? | **Must-have** (согласно брифу и приоритетам заказчика) |
