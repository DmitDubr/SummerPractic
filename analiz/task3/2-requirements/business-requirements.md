# Бизнес-требования — картинг-центр «Апекс»

> Источник: [domain-description.md](../1-elicitation/domain-description.md).
> Приоритет: **Must** — MVP v1; **Should** — v2 (backlog в домене); **Could** — вне текущего скоупа.

| ID | Формулировка | Приоритет | Источник |
| :- | :-- | :--: | :-- |
| BR-001 | Центр должен предоставить клиентам **самообслуживание записи** на заезды через мобильное приложение, чтобы снизить нагрузку на ручную обработку заявок (Telegram, доска маркером). | Must | [domain-description.md §1, §7](../1-elicitation/domain-description.md#7-болевые-точки) |
| BR-002 | Система записи должна **исключить двойное бронирование** одного карта, чтобы клиенты не приезжали на уже занятый заезд. | Must | [domain-description.md §3, §7](../1-elicitation/domain-description.md#3-бизнес-правила) |
| BR-003 | Клиент должен **самостоятельно видеть доступность** заездов («есть места / мест нет») и записываться без участия владельца в каждой операции. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#вместимость-и-расписание) |
| BR-004 | Операционное управление расписанием, маршалами, отменами по погоде и учёт поздних отмен остаётся в **существующей инфраструктуре**; новое решение дополняет её клиентским каналом. | Must | [domain-description.md §1, §6](../1-elicitation/domain-description.md#6-границы-системы) |
| BR-005 | При отмене заезда **центром** (в т.ч. погода) клиент должен **получать информацию** о статусе брони, причине отмены и возможности перезаписаться на другой заезд. | Must | [domain-description.md §3, §5.6](../1-elicitation/domain-description.md#отмена-центром-и-погода-r-008) |
| BR-006 | Центр должен получать **обратную связь по качеству работы маршалов** от клиентов после заездов для оценки нагрузки и удовлетворённости. | Should | [domain-description.md §3, §5.5](../1-elicitation/domain-description.md#оценки-маршалов-v2) |
| BR-007 | Система должна способствовать **снижению неявок** на записанные заезды за счёт напоминаний клиентам. | Should | [domain-description.md §3, §5.4](../1-elicitation/domain-description.md#54-напоминания-v2) |
| BR-008 | Заезды должны соблюдать **лимиты вместимости**: до **14** картов в центре; лимит **8** на новичковый заезд определяется **типом слота**. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#вместимость-и-расписание) |
| BR-009 | Клиент должен иметь возможность указать **свою или прокатную** экипировку (шлем, подшлемник) при записи, чтобы центр планировал выдачу инвентаря. | Must | [domain-description.md §2, §5.2](../1-elicitation/domain-description.md#бронирование-и-экипировка) |
| BR-010 | При **ранней** отмене клиентом (≥ 1 ч до начала) освобождённый карт должен **сразу** становиться доступным для других. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#отмена-клиентом) |
| BR-011 | **Постоянные клиенты** должны быть **отмечены** в системе для удержания аудитории сезона. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#постоянные-клиенты) |
