## Обзор

Bottom sheet запрашивает явное подтверждение отмены активной брони на заезд. При **поздней отмене** (менее чем за **1 час** до начала) показывается жёлтое предупреждение о возможном простое картов **без блокировки** действия в MVP. После успешной отмены — toast «Запись отменена» и возврат к списку записей с обновлённым статусом `CANCELLED_BY_CLIENT`. Штрафов в MVP нет.

### User Story

> Как клиент, я хочу подтвердить отмену записи на заезд с понятным предупреждением о последствиях, чтобы осознанно отменить бронь и освободить карты.

**Не в MVP:** лист ожидания при `NO_SPOTS` (FR-012); фильтр по конфигурации/уровню; Android; штрафы и блокировка поздней отмены; порог 3 часа (для «Апекс» — **1 час**, LOGIC-004).

## Применяемые логики

| Логика | Элемент / триггер | Описание |
| :-- | :-- | :-- |
| [LOGIC-004_Отмена-правило-1-часа](../../5-mobile-app-spec/09_Логики/LOGIC-004_Отмена-правило-1-часа.md) | Блок предупреждения; видимость кнопки на SCR-009 | Ранняя отмена ≥ 1 ч — без warning; поздняя < 1 ч — warning, отмена разрешена; штрафов нет |

**Формула для разработки и QA:**

```
minutesUntilStart = (slot.startsAt - now) в минутах
canCancel = status == ACTIVE AND minutesUntilStart > 0
isEarlyCancel = canCancel AND minutesUntilStart >= 60
isLateCancel = canCancel AND minutesUntilStart < 60
showWarning = isLateCancel
```

Если заезд уже начался — кнопка «Отменить» на SCR-009 не показывается (вне скоупа SCR-010).

## Используемые запросы

### cancelBooking

**Метод:** POST  
**Путь:** `/bookings/{bookingId}/cancel`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `cancelBooking`

**Обработка ответа (`CancelBookingResponse`):**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 200 + `status: CANCELLED_BY_CLIENT` | Закрыть sheet → toast «Запись отменена» → SCR-008; при ранней отмене карты освобождаются на бэкенде (FR-015) |
| 200 + `isLateCancellation: true` | То же; поздняя отмена учитывается для аналитики (FR-016) |
| 401 | «Сессия истекла» → поток повторной идентификации |
| 409 `ALREADY_CANCELLED` | «Запись уже отменена» → закрыть sheet → обновить SCR-009/008 |
| 409 / 403 прочие | «Не удалось отменить запись» + «Повторить» / «Закрыть» |
| 5xx / timeout | «Не удалось отменить запись» + «Повторить»; sheet остаётся открытым |
| Offline | Запрос не отправляется; на SCR-009 кнопка «Отменить» disabled |

**Поля ответа:**

| Поле | Использование |
| :-- | :-- |
| `id` | Подтверждение отменённой брони |
| `status` | Всегда `CANCELLED_BY_CLIENT` |
| `cancelledAt` | Опционально для детали SCR-009 |
| `isLateCancellation` | Согласование с клиентским расчётом LOGIC-004 |

**Сценарии и переходы:**

1. **Ранняя отмена (UC-004, ≥ 1 ч):** SCR-009 → «Отменить запись» → SCR-010 (без предупреждения) → `cancelBooking` → карты **сразу** освобождаются (FR-015) → toast → SCR-008.
2. **Поздняя отмена (< 1 ч, MVP):** SCR-010 с жёлтым предупреждением → клиент подтверждает (штрафов нет, FR-016) → `cancelBooking` → SCR-008.
3. **Отказ от отмены:** «Оставить запись» или свайп / overlay → sheet закрыт, SCR-009 без изменений.
4. **Двойной тап:** debounce на confirm; повторный запрос не отправлять.

После успеха клиент инвалидирует кэш списка (SCR-008) и детали (SCR-009).

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Заголовок | «Отменить запись?» | — | Всегда виден |
| Краткое напоминание | Дата, время, конфигурация трассы, маршал | `slot.startsAt`, `slot.trackConfiguration.name`, `slot.marshal.fullName` | Формат «Сб, 12 июл · 18:00» |
| Блок предупреждения | Жёлтый фон, иконка ⚠ | LOGIC-004 `showWarning` | **Только** при `isLateCancel`; не error, без упоминания штрафов |
| Текст предупреждения | «До начала заезда осталось меньше часа. Карты могут не успеть освободиться для других участников.» | LOGIC-004 | Нейтрально-информативный; отмена **не блокируется** |
| Кнопка «Оставить запись» | Primary / filled — безопасное действие по умолчанию | — | Закрывает sheet без API |
| Кнопка «Отменить запись» | Destructive (красный текст или outline) | — | Вызывает `cancelBooking`; loader во время запроса |
| Индикатор загрузки | На кнопке «Отменить запись» | Состояние Submitting | Обе кнопки disabled; sheet **не закрывается** до ответа |
| Toast (после успеха) | «Запись отменена» | — | 2–3 сек, на SCR-008, не блокирует список |

**Критерии дизайна:**

- **Безопасное действие по умолчанию** — «Оставить запись» (primary, верхняя или левая кнопка).
- Жёлтый блок — не error; читаемый контраст; зарезервировать место под блок, чтобы layout не прыгал (или плавная анимация).
- **Не пугать** формулировками про штрафы — в MVP их нет (FR-016).
- Согласовать стиль sheet с SCR-007 (ошибка записи).
- Модал доступен **только online**; в офлайне кнопка «Отменить» на SCR-009 disabled.

**Терминология:** **маршал**, **заезд**, **конфигурация трассы**; прокат (шлем, подшлемник) **не влияет на цену** (FR-013).

## Состояния экрана

| Состояние | Условие | Отображение |
| :-- | :-- | :-- |
| Content (ранняя) | `isEarlyCancel = true` | Заголовок + напоминание + две кнопки; **без** жёлтого блока |
| Content (поздняя) | `isLateCancel = true` | Жёлтый блок предупреждения; отмена **разрешена** (MVP, FR-016) |
| Submitting | `cancelBooking` в процессе | Loader на «Отменить запись»; обе кнопки disabled |
| Success | 200 OK | Sheet закрыт → toast → SCR-008; статус `CANCELLED_BY_CLIENT` |
| Error (сеть) | Offline / timeout | «Нет подключения. Проверьте интернет» в sheet |
| Error (сервер) | 4xx / 5xx | «Не удалось отменить запись» + «Повторить» / «Закрыть» |
| Already cancelled | 409 `ALREADY_CANCELLED` | «Запись уже отменена» → закрыть → обновить SCR-009/008 |

> Сквозной паттерн Submitting — [LOGIC-008](../../5-mobile-app-spec/09_Логики/LOGIC-008_Паттерн-состояний-экрана.md).

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** до начала заезда ≥ 1 ч и статус `ACTIVE`, **Когда** открыт SCR-010, **Тогда** жёлтый блок предупреждения **не** показывается. |
| AC-002 | **Дано** до начала заезда < 1 ч и > 0, **Когда** открыт SCR-010, **Тогда** показывается warning-блок о простое картов и кнопка «Отменить запись» **активна** (FR-016). |
| AC-003 | **Дано** пользователь на SCR-010 с ранней отменой, **Когда** нажимает «Отменить запись» и `cancelBooking` вернул 200 с `status=CANCELLED_BY_CLIENT`, **Тогда** sheet закрывается, показывается toast «Запись отменена», выполняется переход на SCR-008. |
| AC-004 | **Дано** пользователь на SCR-010, **Когда** нажимает «Оставить запись» или закрывает sheet свайпом, **Тогда** sheet закрывается без API-запроса и SCR-009 остаётся без изменений. |
| AC-005 | **Дано** `cancelBooking` в процессе, **Когда** ожидается ответ, **Тогда** loader на confirm-кнопке, обе кнопки disabled, sheet не закрывается. |
| AC-006 | **Дано** `cancelBooking` вернул 409 `ALREADY_CANCELLED`, **Когда** отображается ошибка, **Тогда** sheet закрывается и список/деталь записи обновляются. |
| AC-007 | **Дано** нет сети, **Когда** пользователь на SCR-009, **Тогда** кнопка «Отменить запись» disabled и SCR-010 недоступен. |
| AC-008 | **Дано** заезд уже начался (`slot.startsAt <= now`), **Когда** открыт SCR-009, **Тогда** кнопка «Отменить запись» **скрыта** (LOGIC-004). |
| AC-009 | **Дано** поздняя отмена подтверждена, **Когда** `cancelBooking=200`, **Тогда** в ответе `isLateCancellation=true` и штраф клиенту **не** применяется. |
