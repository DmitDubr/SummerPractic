## Обзор

Bottom sheet фильтрации расписания на SCR-001 по двум критериям MVP v1: **время суток** и **маршал** (FR-003). Фильтры по **конфигурации трассы** и **уровню намеренно исключены** из первой версии. При «Применить» sheet закрывается и передаёт параметры в SCR-001 для `listSlots`. Период (SCR-002) **не сбрасывается** при смене фильтров.

### User Story

> Как клиент, я хочу отфильтровать заезды по времени суток и маршалу, чтобы быстрее найти подходящий заезд.

**Не в MVP:** фильтр по конфигурации трассы и уровню; рейтинг маршала на чипах (FR-028 — v2); Android.

## Используемые запросы

### listMarshals

**Метод:** GET  
**Путь:** `/marshals`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `listMarshals`

**Обработка ответа:**

| HTTP | UI-реакция |
| :-- | :-- |
| 200 + items | Чипы маршалов (`fullName`, опционально `photoUrl`); рейтинг **не показывается** в MVP v1 |
| 500 / timeout | Inline «Не удалось загрузить список маршалов» + «Повторить» в секции; время суток работает |
| 200 + empty | «Маршалы пока не добавлены»; секция disabled; фильтр по времени доступен |

### listSlots (косвенно через SCR-001)

**Метод:** GET  
**Путь:** `/slots`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `listSlots`

**Query после «Применить»:**

| Параметр | Источник |
| :-- | :-- |
| `marshalIds` | Выбранные маршалы (OR внутри группы); не передаётся при `[]` |
| `timeOfDay` | Сегмент времени суток: `morning` 06:00–11:59, `afternoon` 12:00–16:59, `evening` 17:00–23:59 |
| `dateFrom`, `dateTo` | Сохраняются из SCR-002 без изменений |

## Макет экрана

```
┌─────────────────────────────────────┐
│  ░░░░░░░ SCR-001 (затемнён) ░░░░░░░ │
├─────────────────────────────────────┤
│  ───  (drag handle)                 │
│  Фильтры                            │
│                                     │
│  Время суток                        │
│  [ Утро ] [ День ] [ Вечер ]        │  ← single select, mutually exclusive chips
│                                     │
│  Маршал                             │
│  [ Все ] [Алексей К.] [Дмитрий П.]  │  ← multi-select chips
│  [ Показать всех ▾ ]                │  ← если маршалов >5
│                                     │
│  [ Сбросить ]    [ Применить ]      │  ← sticky footer
└─────────────────────────────────────┘
```

Контент sheet — вертикальный скролл при нехватке места. Футер с кнопками — sticky; padding-bottom, чтобы не перекрывать последнюю секцию.

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Drag handle | Свайп закрытия | Константа UI | Без применения |
| Заголовок «Фильтры» | Название sheet | Константа UI | Semibold, 18–20 sp |
| Секция «Время суток» | Single select (mutually exclusive chips) | Локально | **Утро** (`morning` 06:00–11:59), **День** (`afternoon` 12:00–16:59), **Вечер** (`evening` 17:00–23:59); повторный тап снимает выбор |
| Секция «Маршал» | Multi-select чипы | `listMarshals.items[]` | Один или несколько маршалов; чип «Все маршалы» / снятие всех = без фильтра; при >5 — первые 4 + «Ещё N» |
| Чип маршала — имя | Отображаемое имя | `items[].fullName` | Ellipsis при длинных именах; полное имя в accessibility label |
| Чип маршала — аватар | Опциональное фото | `items[].photoUrl` | Плейсхолдер при отсутствии |
| Рейтинг на чипе | Публичный рейтинг | `items[].avgRating` | **Не в MVP v1** (FR-028); только имя (+ опционально аватар) |
| «Сбросить» | Сброс всех фильтров sheet | Локально | Очистить время суток и маршалов; применить и закрыть; период SCR-002 не сбрасывается |
| «Применить» | Подтверждение | Локально | Активна всегда (пустой выбор = все заезды периода); закрывает sheet → reload SCR-001 |
| Badge на SCR-001 | Число активных категорий | LOGIC-005 | +1 если `timeOfDay` задан; +1 если `marshalIds.length > 0`; max **2**, не количество маршалов |

**Badge пример:** 2 маршала + «Вечер» → badge **«2»**, не «3».

## Состояния экрана

| Состояние | Условие | Отображение |
| :-- | :-- | :-- |
| Loading | `listMarshals` в процессе, нет кэша | Skeleton чипов в секции «Маршал» (3–5 placeholder); «Время суток» — статично |
| Content | Справочник загружен | Все секции интерактивны |
| Empty | Список маршалов пуст | «Маршалы пока не добавлены»; секция disabled |
| Error | Ошибка `listMarshals` | Inline error + retry; время суток работает |
| Offline | Нет сети, нет кэша | Секция маршалов в error; есть кэш — показать с пометкой «Обновлено …» (опционально) |
| Submitting | «Применить» / «Сбросить» | Кнопки disabled до закрытия |

> 0 заездов после применения — Empty state на SCR-001, не в sheet.

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** открыт SCR-003, **Когда** sheet открывается, **Тогда** вызывается `listMarshals` и отображаются маршалы без рейтинга. |
| AC-002 | **Дано** выбраны 2 маршала и «Вечер», **Когда** «Применить», **Тогда** SCR-001 перезагружает `listSlots` с `marshalIds` и `timeOfDay=evening`, badge = 2. |
| AC-003 | **Дано** выбран только «Утро», **Когда** «Применить», **Тогда** query содержит `timeOfDay=morning`, badge = 1. |
| AC-004 | **Дано** активны фильтры, **Когда** «Сбросить», **Тогда** query без `marshalIds`/`timeOfDay`, badge = 0. |
| AC-005 | **Дано** `listSlots` вернул пусто при фильтрах, **Когда** SCR-001, **Тогда** «Ничего не найдено» + «Сбросить фильтры». |
| AC-006 | **Дано** изменён период на SCR-002, **Когда** применены фильтры SCR-003, **Тогда** `dateFrom`/`dateTo` и фильтры переданы в один `listSlots`. |
| AC-007 | **Дано** ошибка загрузки маршалов, **Когда** retry в sheet, **Тогда** повторный `listMarshals` без закрытия sheet. |
| AC-008 | **Дано** пользователь закрыл sheet без «Применить», **Когда** возврат на SCR-001, **Тогда** список и badge не изменились. |
