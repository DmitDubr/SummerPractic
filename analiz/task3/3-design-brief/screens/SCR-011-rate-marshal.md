## Обзор

iOS bottom sheet для оценки **маршала** после **посещённого** заезда. Экран поддерживает только выбор звёзд 1–5 и отправку `createOrUpdateMarshalRating`; **текстовый отзыв, теги и иные поля не предусмотрены** (Q 5.2).

Оценка доступна только при `Booking.status = ATTENDED` и только в пределах **7 дней** после начала заезда (`slot.startsAt`). Модель оценки — **upsert по паре клиент–маршал**: у клиента не более одной оценки на маршала; повторная отправка обновляет существующую (FR-027, Q 5.4). Агрегированный рейтинг отображается другим клиентам на SCR-001 / SCR-004 (FR-028) — на SCR-011 только ввод.

Функция относится к **v2 (Should)** — в MVP v1 экран и CTA на SCR-009 не реализуются (Q 10.2).

### User Story

> Как клиент, я хочу быстро поставить оценку маршалу после заезда, чтобы повлиять на его публичный рейтинг без длинного отзыва.

**Не в MVP v1:** текстовые отзывы; отдельный полноэкранный экран оценки; оценка до посещения; оценка после 7 дней; отображение публичного рейтинга маршала в расписании (FR-028).

## Входные данные

| Название | Тип | Источник | Описание |
| :-- | :-- | :-- | :-- |
| `bookingId` | UUID | Навигация (из SCR-009 / push) | Идентификатор брони для проверки контекста и статуса ATTENDED |
| `marshalId` | UUID | `getBooking.slot.marshal.id` | Идентификатор маршала для upsert оценки |
| `marshalName` | string | `getBooking.slot.marshal.fullName` | Имя маршала в заголовке sheet |
| `marshalPhotoUrl` | string \| null | `getBooking.slot.marshal.photoUrl` | Аватар маршала |
| `existingStars` | integer \| null | `getBooking.marshalRating.stars` | Предзаполнение в режиме редактирования; `null` — новая оценка |
| `heatStartAt` | datetime | `getBooking.slot.startsAt` | Начало заезда; расчёт окна 7 дней: `now - heatStartAt ≤ 7d` |
| `canRate` | boolean | Клиент (из SCR-009) | `status = ATTENDED` && в пределах 7 дней; при `false` sheet не открывается |

## Используемые запросы

### createOrUpdateMarshalRating

**Метод:** POST  
**Путь:** `/ratings`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `createOrUpdateMarshalRating`

**Тело запроса (`CreateRatingRequest`):**

| Поле | Обязательность | Описание |
| :-- | :-- | :-- |
| `marshalId` | Да | UUID маршала |
| `stars` | Да | Целое 1–5 |
| `bookingId` | Да при вызове из SCR-011 | UUID брони для проверки `ATTENDED` при первой оценке |

**Пример:**

```json
{
  "bookingId": "4d5e6f7a-8b9c-0d1e-2f3a-4b5c6d7e8f9a",
  "marshalId": "7c9e6679-7425-40de-944b-e07fc1f90ae7",
  "stars": 4
}
```

**Ответ (`CreateRatingResponse`):** `id`, `marshalId`, `stars`, `createdAt`; опционально `bookingId`, `updatedAt`.

**Обработка ответа:**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 201 + data | Новая оценка создана → краткое «Спасибо за оценку!» → закрытие sheet → toast на SCR-009 |
| 200 + data | Оценка обновлена (upsert) → то же; на SCR-009 read-only звёзды + скрытие CTA «Оценить» |
| 400 `VALIDATION_ERROR` | Inline-ошибка под CTA, кнопка снова активна, выбранные звёзды сохранены |
| 403 `BOOKING_NOT_ATTENDED` | Sheet закрывается; на SCR-009: «Оценить можно после заезда» |
| 403 `RATING_PERIOD_EXPIRED` | Sheet закрывается; на SCR-009: «Срок оценки истёк»; CTA скрыт |
| 401 | Переход к повторной идентификации (сессия истекла) |
| 5xx / timeout | Inline-ошибка + «Повторить»; звёзды сохранены локально |

## Макет экрана

Bottom sheet поверх SCR-009 (или поверх приложения при переходе из push). Не использовать модальное окно на весь экран — контекст записи остаётся видимым под sheet.

```
┌─────────────────────────────────┐
│  ───  (drag handle)             │
│                                 │
│  [Фото]  Алексей К.             │
│                                 │
│  Оцените заезд с маршалом       │
│  ☆ ☆ ☆ ☆ ☆   (интерактивные)    │
│                                 │
│  [ Отправить ]  (disabled →     │
│                  enabled)       │
└─────────────────────────────────┘
```

При **изменении** существующей оценки — звёзды предзаполнены `existingStars`; кнопка «Сохранить» вместо «Отправить».

После успешной отправки — краткое сообщение «Спасибо за оценку!» (внутри sheet 1–2 с или toast при закрытии).

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Drag handle | Индикатор iOS bottom sheet | UI | Закрытие свайпом вниз без подтверждения |
| Кнопка закрытия | Крестик в углу sheet (опционально) | UI | Закрытие без сохранения |
| Аватар маршала | Круглое фото / placeholder с инициалами | `marshalPhotoUrl` | Placeholder при `null`; skeleton при загрузке из push |
| Имя маршала | Заголовок карточки | `marshalName` | Обязательное; термин **маршал**, не «инструктор» |
| Подсказка | Нейтральная формулировка | Константа UI | «Оцените заезд с маршалом» или «Как прошёл заезд?» |
| Блок звёзд | 5 tappable иконок | Локальный state | Только целые 1–5; заливка выбранных, контур невыбранных |
| CTA «Отправить» / «Сохранить» | Primary action | Локальный state | Disabled при `stars = null`; «Сохранить» при `existingStars != null`; loader в Submitting |
| Inline сообщение | Ошибка / успех | API response | Успех: «Спасибо за оценку!»; ошибка: текст + «Повторить» |

**Не включать:** поле текстового отзыва; переключатель анонимности; выбор маршала (один маршал из брони); отображение публичного агрегированного рейтинга (это SCR-001 / SCR-004).

**Терминология:** **маршал**, **заезд**, **конфигурация трассы**; не «класс», «шеф», «инструктор», «тренировка». Прокат (шлем, подшлемник) **не влияет на цену** (FR-013).

## Состояния экрана

| Состояние | Условие | Отображение |
| :-- | :-- | :-- |
| Blocked | `canRate = false` или `status ≠ ATTENDED` | Sheet **не открывается**; сообщение на SCR-009 |
| Loading | Push: ожидание `getBooking` coordinator'ом | Skeleton на аватаре и имени (до present sheet) |
| Content (новая оценка) | `existingStars = null` | 0 звёзд, CTA «Отправить» disabled |
| Content (изменение) | `existingStars != null` | Звёзды предзаполнены, CTA «Сохранить» enabled при изменении |
| Star selected | Выбрано 1–5 звёзд | Подсветка звёзд, CTA enabled |
| Submitting | POST `/ratings` в процессе | Loader на CTA; звёзды и закрытие заблокированы |
| Success | 201 / 200 | «Спасибо за оценку!» → авто-закрытие 1–2 с или по тапу |
| Error | 401 / 5xx / timeout | Inline-ошибка + «Повторить»; звёзды сохранены |
| Expired | `now - heatStartAt > 7d` | Sheet не открывается; на SCR-009 CTA скрыт, read-only оценка если была |
| NotAttended | Статус не `ATTENDED` (устаревший push) | Sheet: «Оценить можно после заезда» + CTA «К записи» → SCR-009 |

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** бронь со статусом `ATTENDED` и с момента `slot.startsAt` прошло не более 7 дней, **Когда** клиент открывает SCR-011, **Тогда** sheet доступен и позволяет выбрать 1–5 звёзд. |
| AC-002 | **Дано** клиент не выбрал ни одной звезды (`existingStars = null`), **Когда** открыт SCR-011, **Тогда** кнопка «Отправить» неактивна. |
| AC-003 | **Дано** клиент отправляет оценку с валидными `bookingId`, `marshalId`, `stars`, **Когда** API возвращает 201, **Тогда** sheet закрывается, показывается «Спасибо за оценку!», на SCR-009 отображается сохранённая оценка read-only. |
| AC-004 | **Дано** у клиента уже есть оценка этого маршала (`existingStars != null`), **Когда** в окне 7 дней выполняется повторная отправка, **Тогда** вызывается `createOrUpdateMarshalRating` и при 200 обновляется существующая оценка (upsert), CTA на SCR-009 — «Изменить оценку». |
| AC-005 | **Дано** прошло более 7 дней после `slot.startsAt`, **Когда** клиент пытается открыть SCR-011, **Тогда** sheet не открывается, CTA на SCR-009 скрыт; при наличии прошлой оценки — read-only звёзды. |
| AC-006 | **Дано** статус брони не `ATTENDED`, **Когда** клиент открывает экран по CTA или deep link, **Тогда** отправка блокируется и показывается «Оценить можно после заезда». |
| AC-007 | **Дано** пользовательский интерфейс SCR-011, **Когда** пользователь взаимодействует с формой, **Тогда** доступны только звёзды 1–5; поле текстового отзыва, теги и иные поля **отсутствуют**. |
| AC-008 | **Дано** экран в состоянии Content, **Когда** пользователь тапает по звёздам, **Тогда** каждая звезда имеет tappable-зону не менее 44 pt и визуально различаются выбранное / невыбранное состояние. |
| AC-009 | **Дано** MVP v1, **Когда** приложение работает без v2-фичи, **Тогда** SCR-011 и CTA «Оценить маршала» на SCR-009 не отображаются. |
| AC-010 | **Дано** успешное сохранение оценки, **Когда** другой клиент открывает SCR-001 / SCR-004 (v2), **Тогда** агрегированный рейтинг маршала обновлён на бэкенде (отображение — не на SCR-011). |
