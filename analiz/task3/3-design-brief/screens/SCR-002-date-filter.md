## Обзор

Bottom sheet для выбора периода показа расписания **за пределами дефолтных 7 дней** (R-027, FR-002). Открывается поверх SCR-001; сам sheet не выполняет API-запросов — сохраняет выбранный диапазон и возвращает `dateFrom`/`dateTo` в SCR-001, где выполняется `listSlots`. Фильтры заездов (SCR-003) при смене периода **сохраняются** (AND).

### User Story

> Как клиент, я хочу выбрать произвольный диапазон дат, чтобы посмотреть заезды не только на ближайшие 7 дней.

**Не в MVP:** пресеты периодов («На неделю» / «На месяц»); автозагрузка без нажатия «Применить»; Android.

## Входные данные

| Название | Тип | Источник | Описание |
| :-- | :-- | :-- | :-- |
| `dateFrom` | `string(date)?` | SCR-001 | Уже применённая дата начала |
| `dateTo` | `string(date)?` | SCR-001 | Уже применённая дата окончания |
| `today` | `string(date)` | Системное время устройства | Нижняя граница выбора; прошлые дни disabled |
| `maxRangeDays` | `number` | Конфиг клиента | Максимальная длина диапазона (рекомендация: 30 дней) |

## Используемые запросы

### listSlots (косвенный вызов из SCR-001 после применения периода)

**Метод:** GET  
**Путь:** `/slots`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `listSlots`

**Query-параметры, которые передаёт SCR-002 через SCR-001:**

| Параметр | Значение |
| :-- | :-- |
| `dateFrom` | Выбранная дата начала (ISO 8601 date) |
| `dateTo` | Выбранная дата окончания (ISO 8601 date) |
| `marshalIds`, `timeOfDay` | Сохраняются из SCR-003 без изменений |

**Обработка ответа (на стороне SCR-001):**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 200 + data | SCR-001 показывает список заездов в выбранном диапазоне; чип периода: «12–25 июл» |
| 200 + empty | SCR-001 показывает Empty («Пока нет доступных заездов» / «Ничего не найдено») |
| 400 | SCR-001 показывает Error по параметрам периода |
| 500 / timeout | SCR-001 показывает Error + «Повторить» |
| Нет сети | Toast на SCR-001: «Нет подключения. Проверьте интернет» |

## Макет экрана

```
┌─────────────────────────────────────┐
│  ░░░░░░░ SCR-001 (затемнён) ░░░░░░░ │
├─────────────────────────────────────┤
│  ───  (drag handle)                 │
│  Период                             │
│  По умолчанию — ближайшие 7 дней    │
│                                     │
│  ┌─────────────────────────────┐    │
│  │   Календарь / date range      │    │  ← iOS inline calendar
│  │   picker                      │    │
│  └─────────────────────────────┘    │
│                                     │
│  С  12 июля 2026                    │  ← текстовое отображение выбора
```

Высота sheet: ~70–85 % экрана; на маленьких экранах допустим полноэкранный modal с кнопкой «Закрыть». Safe area учитывается для sticky footer.

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Drag handle | Полоса для свайпа вниз | Константа UI | Закрывает sheet **без** применения |
| Заголовок «Период» | Название шторки | Константа UI | 18–20 sp, semibold; фиксирован в верхней части |
| Подсказка | Инфо о дефолте 7 дней | Константа UI | «По умолчанию показаны ближайшие 7 дней»; неинтерактивна |
| Date range picker | Выбор диапазона «с» и «по» | Локальное состояние | iOS inline calendar; `dateFrom >= today`, `dateTo >= dateFrom`; понедельник — первый день недели; русская локаль |
| Строка «С» | Дата начала | Локальное состояние | «12 июля 2026»; синхронно с календарём |
| Строка «По» | Дата окончания | Локальное состояние | «25 июля 2026» или placeholder «Выберите дату окончания» |
| Кнопка «Сбросить» | Сброс на дефолтный период | Локальное состояние | Secondary / text; возврат `dateFrom/dateTo = null`, немедленное применение и закрытие |
| Кнопка «Применить» | Подтверждение выбора | Локальное состояние | Primary; disabled, если не выбрана конечная дата или диапазон невалиден |

**Формат чипа на SCR-001 после применения:** «12–18 июл»; разные месяцы — «28 июл – 3 авг». Дефолтный период — нейтральный стиль чипа «7 дней».

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** открыт SCR-002 из SCR-001 с дефолтным периодом, **Когда** sheet отображается, **Тогда** предустановлен диапазон «сегодня → сегодня + 6 дней». |
| AC-002 | **Дано** открыт SCR-002 при ранее применённом кастомном периоде, **Когда** sheet отображается, **Тогда** календарь показывает последний применённый диапазон. |
| AC-003 | **Дано** пользователь выбрал только дату начала, **Когда** дата окончания не выбрана, **Тогда** кнопка «Применить» неактивна. |
| AC-004 | **Дано** пользователь выбрал валидный диапазон, **Когда** нажал «Применить», **Тогда** sheet закрывается и в SCR-001 передаются `dateFrom`/`dateTo`, выполняется `listSlots`. |
| AC-005 | **Дано** пользователь нажал «Сбросить», **Когда** действие выполнено, **Тогда** в SCR-001 применяется дефолтный режим 7 дней без `dateFrom`/`dateTo`, чип: «7 дней». |
| AC-006 | **Дано** пользователь закрыл sheet свайпом вниз, **Когда** возврат в SCR-001 завершён, **Тогда** период остаётся последним применённым (без изменений). |
| AC-007 | **Дано** после применения диапазона `listSlots` вернул пусто, **Когда** открыт SCR-001, **Тогда** отображается корректный Empty state. |
| AC-008 | **Дано** активны фильтры SCR-003, **Когда** пользователь меняет период на SCR-002, **Тогда** `marshalIds` и `timeOfDay` сохраняются в запросе `listSlots`. |
