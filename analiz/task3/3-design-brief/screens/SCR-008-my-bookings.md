## Обзор

Экран показывает список броней текущего клиента на заезды картинг-центра «Апекс» и разделяет его на сегменты «Предстоящие» и «Прошедшие». Данные загружаются через `listBookings` под `ClientSession`; при отсутствии сети используется локальный кэш последнего успешного ответа (NFR-009). Несколько записей в один день отображаются отдельными карточками (FR-011).

### User Story

> Как клиент, я хочу видеть свои предстоящие и прошедшие записи на заезды, чтобы контролировать посещения и быстро открывать детали нужной брони.

**Не в MVP:** лист ожидания при `NO_SPOTS` (FR-012) — только возврат к расписанию; фильтр по конфигурации/уровню; Android; рейтинги маршалов в UI (FR-028 — v2); редактирование аллергий.

## Инициализация

### Запросы при открытии

| № | operationId | Критичный | Условие |
| :-: | :-- | :--: | :-- |
| 1 | `listBookings` | Да | Первое открытие экрана или `refresh=true` |

> Сегментация «Предстоящие» / «Прошедшие» выполняется на клиенте по `slot.startsAt` и `status`. Повторный API-запрос при переключении сегмента **не** выполняется.

**Правила сегментации:**

| Сегмент | Условие включения |
| :-- | :-- |
| **Предстоящие** | `slot.startsAt >= now` (в т.ч. `ACTIVE`, отменённые центром с будущей датой слота) |
| **Прошедшие** | `slot.startsAt < now` (статусы: `ATTENDED`, `CANCELLED_BY_CLIENT`, `CANCELLED_BY_CENTER`) |

Сортировка: предстоящие — от ближайших к дальним; прошедшие — от недавних к старым.

## Используемые запросы

### listBookings

**Метод:** GET  
**Путь:** `/bookings`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `listBookings`

**Обработка ответа:**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 200 + `items.length > 0` | Content; сохранение в офлайн-кэш (NFR-009) |
| 200 + `items.length = 0` | Empty (текст зависит от активного сегмента) |
| 401 | Завершение сессии, переход в поток первичной идентификации |
| 5xx / timeout | Error + «Повторить» |
| offline + есть кэш | Offline state + баннер «Показаны сохранённые данные» |
| offline + нет кэша | Empty: «Нет сохранённых записей. Подключитесь к интернету» |

**Поля `BookingSummary` для карточки:**

| Поле | Использование |
| :-- | :-- |
| `id` | Навигация на SCR-009 |
| `status` | Бейдж статуса |
| `participantCount` | «N участника» / «N участников» (если > 1; иначе скрыть) |
| `totalPrice` | Сумма к оплате на месте |
| `slot.startsAt` | Дата и время |
| `slot.trackConfiguration.name` | Конфигурация трассы (кратко) |
| `slot.marshal.fullName` | Имя маршала |

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Сегмент «Предстоящие/Прошедшие» | Локальное переключение двух списков | UI state + `listBookings.items` | Не вызывает повторный API-запрос |
| Карточка записи | Дата, время, конфигурация, маршал, участники, цена, статус | `slot.*`, `participantCount`, `totalPrice`, `status` | Вся карточка кликабельна (tap target ≥ 44 pt), переход на SCR-009 |
| Бейдж статуса | Текстовое состояние записи | `status` | `ACTIVE`→«Записан»; `CANCELLED_BY_CLIENT`→«Отменена вами»; `CANCELLED_BY_CENTER`→«Отменена центром»; `ATTENDED`→«Посещён» |
| Число участников | Подпись под маршалом | `participantCount` | Показывать только при `participantCount > 1` |
| Сумма | Итог к оплате | `totalPrice` | Не пересчитывать на клиенте; прокат не влияет (LOGIC-003) |
| Офлайн-баннер | Сообщение о кэше | Статус сети + timestamp кэша | Показывается только в offline режиме; не исчезает автоматически |
| Empty state | Сообщение об отсутствии записей | Пустой список в активном сегменте | «Предстоящие»: «У вас нет предстоящих записей» + CTA «Посмотреть расписание»; «Прошедшие»: «Здесь появятся завершённые заезды» |
| Pull-to-refresh | Обновление данных | Жест пользователя (`UIRefreshControl`) | Активен только online |

**На карточке не показывать:** причину отмены центром (`cancellationReason`) — только бейдж; детали на SCR-009 (R-008).

**Терминология:** **маршал**, **заезд**, **конфигурация трассы**; прокат (шлем, подшлемник) **не влияет на цену** (FR-013).

## Состояния экрана

| Состояние | Условие | Отображение |
| :-- | :-- | :-- |
| Loading | Первый запрос в процессе | Skeleton карточек (3–4 шт.) |
| Content | 200 + записи есть в выбранном сегменте | Список карточек |
| Empty | 200 + выбранный сегмент пуст | Текст + CTA (для «Предстоящие») |
| Error | 4xx / 5xx на первой загрузке без кэша | Ошибка + «Повторить» |
| Error + кэш | Ошибка при наличии кэша | Кэшированный список + баннер «Не удалось обновить. Показаны сохранённые данные» |
| Offline | Нет сети, кэш есть | Кэшированный список + офлайн-баннер |
| Refreshing | Pull-to-refresh online | Индикатор обновления поверх списка; при fail — snack, список не очищается |

> Сквозной паттерн — [LOGIC-008](../../5-mobile-app-spec/09_Логики/LOGIC-008_Паттерн-состояний-экрана.md).

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** открыта вкладка «Мои записи» online, **Когда** `listBookings` возвращает список, **Тогда** показывается сегмент «Предстоящие» по умолчанию и карточки ближайших записей. |
| AC-002 | **Дано** в сегменте «Предстоящие» нет записей, **Когда** экран перешёл в Empty, **Тогда** показывается текст «У вас нет предстоящих записей» и CTA «Посмотреть расписание» → SCR-001. |
| AC-003 | **Дано** пользователь переключил сегмент, **Когда** меняется `selectedSegment`, **Тогда** список фильтруется локально без повторного вызова `listBookings`. |
| AC-004 | **Дано** устройство offline и есть кэш списка, **Когда** экран открыт, **Тогда** показываются сохранённые данные и баннер «Показаны сохранённые данные». |
| AC-005 | **Дано** выполнен tap по карточке записи, **Когда** передан `bookingId`, **Тогда** открывается SCR-009 с деталями выбранной брони. |
| AC-006 | **Дано** пользователь вернулся из SCR-010 после успешной отмены, **Когда** экран получает `refresh=true`, **Тогда** карточка отменённой записи показывает статус «Отменена вами» (`CANCELLED_BY_CLIENT`). |
| AC-007 | **Дано** на карточке отображается `totalPrice` и выбран прокат, **Когда** строится UI, **Тогда** сумма берётся из API и не меняется из-за проката (LOGIC-003). |
| AC-008 | **Дано** `participantCount = 2`, **Когда** отображается карточка, **Тогда** видна подпись «2 участника»; при `participantCount = 1` подпись скрыта. |
| AC-009 | **Дано** push об отмене центром (v2) открыл SCR-009, **Когда** пользователь возвращается назад, **Тогда** на SCR-008 карточка показывает бейдж «Отменена центром» без текста причины. |
| AC-010 | **Дано** в один день две активные брони, **Когда** список загружен, **Тогда** обе отображаются отдельными карточками, отсортированными по времени слота (FR-011). |
