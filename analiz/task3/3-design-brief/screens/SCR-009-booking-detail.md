## Обзор

Экран показывает полную информацию о конкретной брони на заезд и доступные действия по статусу. Открывается из списка SCR-008 и как **целевая страница deep link** из push-уведомлений v2 (отмена центром, напоминание, перенос — FR-023, FR-025, FR-029). При `CANCELLED_BY_CENTER` отображается `cancellationReason` (R-008). Поддерживает офлайн-просмотр закэшированной детали (NFR-009).

**Перезапись после отмены центром (FR-024, v2)** — **только из push-уведомления** на SCR-004 (другой слот, не отменённый — FR-018). На SCR-009 **нет** CTA «Записаться на другой заезд» / «Перезаписаться».

### User Story

> Как клиент, я хочу открыть детальную карточку записи на заезд, чтобы понять актуальный статус, причину отмены центром, число участников и выполнить действие (отмена, оценка маршала).

**Не в MVP:** лист ожидания при `NO_SPOTS` (FR-012); фильтр по конфигурации/уровню; Android; рейтинги маршалов в блоке заезда (FR-028 — v2); редактирование аллергий; CTA перезаписи на экране.

## Используемые запросы

### getBooking

**Метод:** GET  
**Путь:** `/bookings/{bookingId}`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `getBooking`

**Обработка ответа:**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 200 + data | Content: заполнение всех блоков, CTA по статусу; сохранение в офлайн-кэш |
| 200 + `status=CANCELLED_BY_CENTER` | Warning-блок с `cancellationReason` (R-008) |
| 404 | Экран «Запись не найдена» + кнопка «К моим записям» → SCR-008 |
| 401 | Завершение сессии и выход в поток повторной идентификации |
| 5xx / timeout | Error + «Повторить» |
| offline + кэш | Offline state с read-only деталями |

**Ключевые поля `Booking`:**

| Поле | Использование |
| :-- | :-- |
| `id`, `slotId` | Навигация, deep link; `slotId` — проверка FR-018 на бэкенде |
| `status` | Бейдж и условные блоки (`BookingStatus`) |
| `participantCount` | Блок «Ваша запись»: «N участник(а/ов)» |
| `totalPrice` | Итог к оплате на месте |
| `briefingRequired` | Инфо об инструктаже |
| `equipment` | Своё / прокат (шлем, подшлемник) |
| `cancellationReason` | Только при `CANCELLED_BY_CENTER` |
| `cancelledAt` | Для отменённых клиентом / центром |
| `canCancel`, `isLateCancellation` | Видимость CTA отмены; контекст для SCR-010 |
| `slot.startsAt`, `slot.endsAt` | Дата и интервал заезда (~15–20 мин) |
| `slot.trackConfiguration` | Конфигурация трассы |
| `slot.marshal` | Маршал |
| `center.address`, `center.meetingPoint` | Адрес и место сбора |
| `marshalRating` | `null` — можно оценить (v2); объект — read-only оценка |

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Кнопка «Назад» | Возврат на SCR-008 | — | Сохраняет позицию списка |
| Бейдж статуса | Крупный индикатор | `status` | `ACTIVE`→«Записан»; `CANCELLED_BY_CLIENT`→«Отменена вами»; `CANCELLED_BY_CENTER`→«Отменена центром»; `ATTENDED`→«Посещён» |
| Дата и время | День недели, дата, интервал | `slot.startsAt`, `slot.endsAt` | Реальный интервал из API |
| Блок «Информация о заезде» | Конфигурация, маршал, адрес, место сбора, цена | `slot.*`, `center.*`, `totalPrice` | Рейтинг маршала **не** в MVP v1 (FR-028) |
| Блок «Ваша запись» | Участники, экипировка, инструктаж | `participantCount`, `equipment`, `briefingRequired` | Участники: «N участник(а/ов)»; экипировка: «Своё» или «Прокат (шлем, подшлемник)» |
| Блок инструктажа | Инфо о необходимости briefing | `briefingRequired` | `true`→«Перед заездом — инструктаж с маршалом»; `false`→«Инструктаж не потребуется» или блок скрыт |
| Warning отмены центром | Причина отмены | `cancellationReason` | Только для `CANCELLED_BY_CENTER`; фон warning, **выше сгиба** при открытии из push |
| CTA «Отменить запись» | Переход в SCR-010 | `canCancel`, LOGIC-004 | Только `ACTIVE`, заезд в будущем, online |
| CTA «Оценить маршала» | Переход в SCR-011 | LOGIC-006 | v2: `ATTENDED`, `marshalRating=null`, ≤ 7 дней |
| Read-only оценка | Звёзды после оценки | `marshalRating` | v2: «Вы оценили маршала: ★★★★☆» |
| Ссылка «Посмотреть расписание» | Вторичная навигация | `status` | Опционально при `CANCELLED_BY_CENTER`; **не** заменяет push-перезапись |
| Офлайн-баннер | Как на SCR-008 | Статус сети | Mutating CTA disabled + подпись «Требуется интернет» |

### Блоки по статусу

| Статус | Дополнительный UI | Основной CTA | Вторичный CTA |
| :-- | :-- | :-- | :-- |
| `ACTIVE` | «Место на заезде забронировано» | **Отменить запись** → SCR-010 | — |
| `CANCELLED_BY_CLIENT` | «Вы отменили эту запись»; `cancelledAt` | — | — |
| `CANCELLED_BY_CENTER` | Warning с `cancellationReason` | — | «Посмотреть расписание» → SCR-001 (опционально) |
| `ATTENDED` | «Заезд завершён» | **Оценить маршала** → SCR-011 (v2) | — |
| `ATTENDED` (оценено, v2) | Read-only звёзды | — | «Изменить оценку» → SCR-011 |

**Терминология:** **маршал**, **заезд**, **конфигурация трассы**; прокат (шлем, подшлемник) **не влияет на цену** (FR-013).

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** вход на экран по `bookingId`, **Когда** `getBooking=200`, **Тогда** отображаются дата/время, конфигурация трассы, маршал, `participantCount`, `totalPrice`, `briefingRequired`, статус и экипировка. |
| AC-002 | **Дано** запись со статусом `CANCELLED_BY_CENTER`, **Когда** открыт SCR-009, **Тогда** показывается warning-блок с текстом `cancellationReason` и **нет** CTA «Записаться на другой заезд». |
| AC-003 | **Дано** push v2 с перезаписью (FR-024), **Когда** клиент тапает действие в уведомлении, **Тогда** открывается SCR-004 с **другим** `slotId`, не отменённым центром (FR-018). |
| AC-004 | **Дано** запись `ACTIVE` и `canCancel=true`, **Когда** пользователь нажимает «Отменить запись», **Тогда** выполняется переход на SCR-010 с параметрами отмены. |
| AC-005 | **Дано** запись `ATTENDED`, `marshalRating=null` и с момента заезда прошло не более 7 дней, **Когда** открыт экран (v2), **Тогда** доступен CTA «Оценить маршала». |
| AC-006 | **Дано** `briefingRequired=true`, **Когда** экран загружен, **Тогда** отображается текст «Перед заездом — инструктаж с маршалом». |
| AC-007 | **Дано** устройство offline и есть кэш детали, **Когда** открыт SCR-009, **Тогда** показываются кэшированные данные, а кнопка «Отменить запись» disabled с пояснением «Требуется интернет». |
| AC-008 | **Дано** push об отмене центром (FR-023), **Когда** deep link `app://bookings/{id}`, **Тогда** открывается SCR-009 со статусом и причиной **без прокрутки** до дополнительных ссылок. |
| AC-009 | **Дано** `participantCount=3`, **Когда** отображается блок «Ваша запись», **Тогда** видно «3 участника» и `totalPrice` соответствует серверному значению (LOGIC-003). |
| AC-010 | **Дано** класс уже начался (`slot.startsAt <= now`), **Когда** открыт SCR-009, **Тогда** кнопка «Отменить запись» **скрыта** (LOGIC-004). |
