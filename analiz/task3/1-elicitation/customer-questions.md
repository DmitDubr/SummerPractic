## 1. Запись и расписание

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 1.1 | Кто может записаться на заезд? Нужна ли регистрация/авторизация, или достаточно **имени и телефона** при первой записи? | Достаточно **имени и телефона** |
| 1.2 | Может ли один человек записать **нескольких участников** (ребёнок, друзья на одном заезде)? | **Да** — можно записать нескольких участников |
| 1.3 | Сколько **активных записей** может иметь один клиент одновременно? Можно ли записаться на **два заезда в один день**? | **Можно несколько заездов в один день** |
| 1.4 | Если все карты на заезд заняты — показываем только «мест нет» или нужен **лист ожидания** при освобождении места? | Только **«мест нет»** (лист ожидания не нужен) |
| 1.5 | **Подтверждение (R-027):** по умолчанию показываем слоты на **7 дней**, более длинный период — через фильтр дат? | **Да**, по R-027 |
| 1.6 | Какие **фильтры** нужны клиенту в первой версии: конфигурация трассы (короткая / длинная), время суток, уровень (новичок / опытный), маршал? | **Время суток**, **маршал** (конфигурация трассы и уровень — не в первой версии) |
| 1.7 | Длительность заезда **всегда 15–20 минут** плюс инструктаж или может отличаться? Нужно ли показывать **время окончания** и длительность брифинга в приложении? | Заезд **всегда 15–20 мин**; инструктаж **не всегда** (если клиент уже проходил — не нужен) |
| 1.8 | Нужно ли в карточке заезда показывать **описание конфигурации трассы** — полностью, кратко или только название? | **Кратко** |

## 2. Заезды, карты и вместимость

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 2.1 | Как система определяет лимит **8 человек на новичковый заезд** — отдельный тип слота, конфигурация трассы (короткая) или правило «если заезд для новичков — лимит 8»? | Определяется **типом слота** |
| 2.2 | Нужна ли **проверка уровня** клиента? Может ли новичок записаться на заезд на **длинной трассе**? | **Нет** — проверка не нужна |
| 2.3 | Что именно выбирает клиент при записи: **своя экипировка** (шлем, подшлемник) или **прокат** (шлем, подшлемник, оба)? Влияет ли выбор на **цену**? | Выбор своё / прокат; **прокат не влияет на цену** |
| 2.4 | Ограничен ли **прокатный фонд** экипировки? Что показывать, если прокат на слот закончился — слот **недоступен** или запись **только со своим**? | Слот **недоступен** для записи |
| 2.5 | Как показывать **карты с ограничением скорости** — отдельная пометка в карточке заезда или клиенту не важно (назначает маршал на месте)? | **Назначает маршал** на месте (в UI не показываем) |
| 2.6 | **14 картов** — общий фонд центра; на каждый слот бэкенд отдаёт **свободные карты** отдельно? Показывать клиенту «осталось N картов» или достаточно «есть места / мест нет»? | Достаточно **«есть места» / «мест нет»** |

## 3. Отмена клиентом

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 3.1 | Правило «**за 10 минут до старта**» — жёсткий **запрет** отмены, **штраф**, **предупреждение** или только пометка для администратора? | **Предупреждение** клиенту + **пометка** для администратора (отмена разрешена) |
| 3.2 | Что считается «**заранее**» — 24 часа, 3 часа, 1 час, любое время до порога поздней отмены? | **За 1 час и более** до начала — ранняя отмена |
| 3.3 | Нужен ли **учёт** частых поздних отмен и **неявок**? Есть ли последствия для клиента в первой версии? | **Учёт** ведётся; **пометки у администраторов**; последствий для клиента в MVP **нет** |
| 3.4 | После отмены клиентом **карт сразу** становится доступен другим (и для листа ожидания)? | **Да**, место сразу доступно другим |
| 3.5 | При **поздней отмене** карт простаивает — показывать ли клиенту отдельное **предупреждение** о последствиях? | **Да**, показывать предупреждение |

## 4. Отмена центром и погода

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 4.1 | **Подтверждение (R-008):** при отмене центром бронь получает статус **«Отменён центром»** с **причиной**, клиенту приходит **push**, повторная запись на **тот же слот запрещена**? | **Да** (R-008) |
| 4.2 | **Отмена из-за погоды** (дождь, гололёд) — тот же статус и сценарий, что при отмене центром, или отдельный тип причины («Погода»)? | **Тот же статус** «Отменён центром» (причина может указывать на погоду) |
| 4.3 | Типовые **причины** отмены для клиента: погода, поломка, нехватка маршала — **фиксированный список** или свободный текст из админки? | **Фиксированный список** + возможность **свободного текста** из админки |
| 4.4 | Клиенту предлагать **перезапись на другой заезд** из push / экрана детали записи? | **Из push** (перезапись на другой заезд) |
| 4.5 | Если позже подключим **онлайн-оплату** — что делать с деньгами при отмене центром: возврат, перенос, абонемент? | **Оба варианта:** перенос, если клиент согласен; **возврат**, если не хочет переносить |

## 5. Оценки маршалов

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 5.1 | Когда клиент может поставить оценку — только после **посещённого** заезда? В течение какого **срока**? | **Только после посещённого** заезда; срок — **в течение недели** |
| 5.2 | Какой **формат** оценки: звёзды 1–5, текстовый отзыв, анонимно или с именем? | Только **звёзды** (без текстового отзыва) |
| 5.3 | Видят ли **другие клиенты** рейтинги маршалов при выборе слота, или оценки только для владельца? | **Да**, клиенты **видят** рейтинги при выборе слота |
| 5.4 | Оценка привязана к **конкретной записи** (один заезд — одна оценка) или **один пользователь — одна оценка на маршала** с возможностью изменить? | **Один пользователь — одна оценка на маршала** (с возможностью изменить в период оценивания) |

## Открытые противоречия из брифа — статус

| # | Противоречие / пробел | Решение |
| :- | :-- | :-- |
| О-1 | «За 10 минут до старта отмена — проблема» vs правило в приложении | **Предупреждение**; ранняя отмена — **≥ 1 ч**; поздняя — предупреждение + пометка админу (§3) |
| О-2 | Онлайн-оплата на старте vs «наличка и перевод» | **Не входит** в v1; оплата на месте, цена в UI (§7) |
| О-3 | 14 картов vs лимит 8 на новичковых заездах | Лимит определяется **типом слота** (§2.1) |
| О-4 | Постоянные клиенты — «отметить / что-то приятное» | **Метка** в профиле в MVP; скидки — позже (§7.3) |
| О-5 | Погода — отмена заезда целиком vs отдельный UX | **Тот же статус** «Отменён центром»; push; без отдельного погодного UI (§4.2) |
| О-6 | В брифе — экран владельца; в скоупе (R-028) — только клиент | **Не входит** — существующая админка (§8.3) |
| О-7 | Бриф: «10 мин до старта» vs ответ: «1 час» | Принято **1 час** как порог ранней/поздней отмены (§3.2) |
