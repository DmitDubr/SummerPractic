# Бизнес-требования — гончарная мастерская «Глина»

> Источник: [domain-description.md](../1-elicitation/domain-description.md).
> Приоритет: **Must** — обязательно в MVP; **Should** — важно, но может следовать за MVP; **Could** — желательно позже.

| ID | Формулировка | Приоритет | Источник |
| :- | :-- | :--: | :-- |
| BR-001 | Мастерская должна предоставить клиентам **самообслуживание записи** на мастер-классы через мобильное приложение, чтобы снизить нагрузку на ручную обработку заявок (Instagram-директ, ежедневник). | Must | [domain-description.md §1, §7](../1-elicitation/domain-description.md#7-болевые-точки) |
| BR-002 | Система записи должна **исключить двойное бронирование** одного места (в т.ч. на один гончарный круг), чтобы клиенты не приходили на уже занятый слот. | Must | [domain-description.md §3, §7](../1-elicitation/domain-description.md#3-бизнес-правила) |
| BR-003 | Клиент должен **самостоятельно видеть доступность мест** («есть места / мест нет») и записываться на занятие без участия владельца в каждой операции. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#3-бизнес-правила) |
| BR-004 | Операционное управление расписанием, мастерами и отменами по форс-мажору остаётся в **существующей инфраструктуре**; новое решение дополняет её клиентским каналом, не заменяя админку. | Must | [domain-description.md §1, §6](../1-elicitation/domain-description.md#6-границы-системы) |
| BR-005 | При отмене занятия **мастерской** (форс-мажор) клиент должен **своевременно получать информацию** о статусе брони, причине отмены и возможности перезаписаться на другое занятие. | Must | [domain-description.md §3, §5.5](../1-elicitation/domain-description.md#отмена-мастерской-форс-мажор-r-008) |
| BR-006 | Мастерская должна получать **обратную связь по качеству работы мастеров** от клиентов после занятий для оценки нагрузки и удовлетворённости. | Must | [domain-description.md §3, §5.4](../1-elicitation/domain-description.md#оценки-мастеров) |
| BR-007 | Система должна способствовать **снижению неявок** на записанные занятия за счёт push-напоминаний клиентам. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#уведомления-mvp) |
| BR-008 | Групповые занятия должны соблюдать **лимиты вместимости**: до **10** гончарных кругов; на **новичковые группы** — не более **6** человек; лимит на **занятиях на круге** — по **свободным кругам** (до 10); конкретный лимит настраивается **мастером**, ведущим занятие. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#вместимость-и-расписание) |
| BR-009 | Клиент должен иметь возможность указать **своё или прокатное** снаряжение (инструменты, фартук) при записи, чтобы мастерская планировала выдачу инвентаря. | Must | [domain-description.md §2, §5.2](../1-elicitation/domain-description.md#бронирование-и-экипировка) |
| BR-010 | При **ранней** отмене клиентом (≥ 3 ч до начала) освобождённое место должно **сразу** становиться доступным для других. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#отмена-клиентом) |
| BR-011 | **Постоянные клиенты** должны быть **отмечены** в системе и получать **приоритет записи** для удержания аудитории. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#постоянные-клиенты) |
