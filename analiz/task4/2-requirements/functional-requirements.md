# Функциональные требования — гончарная мастерская «Глина»

> Источник: [domain-description.md](../1-elicitation/domain-description.md).
> ФТ описывают **что** делает система. Нефункциональные аспекты — в [non-functional-requirements.md](non-functional-requirements.md).
> Приоритет: **Must** / **Should** / **Could**.

| ID | Формулировка | Приоритет | Источник |
| :- | :-- | :--: | :-- |
| FR-001 | Клиентское приложение отображает список доступных слотов на **ближайшие 7 дней** от текущего момента (R-027). | Must | [domain-description.md §3, §5.1](../1-elicitation/domain-description.md#51-просмотр-расписания) |
| FR-002 | Клиент может **расширить период** просмотра расписания с помощью фильтра дат. | Must | [domain-description.md §3, §5.1](../1-elicitation/domain-description.md#51-просмотр-расписания) |
| FR-003 | Клиент может **фильтровать** расписание по **времени суток**, **уровню** и **программе** (лепка / круг). | Must | [domain-description.md §3, §5.1](../1-elicitation/domain-description.md#вместимость-и-расписание) |
| FR-004 | Для каждого слота отображаются: **время начала**, **программа (кратко)**, **мастер**, **рейтинг мастера**, **«есть места» / «мест нет»**, **цена**. | Must | [domain-description.md §5.1](../1-elicitation/domain-description.md#51-просмотр-расписания) |
| FR-005 | При отсутствии доступных слотов на ближайшие дни приложение показывает empty state: **«Пока нет доступных занятий»**. | Must | [domain-description.md §5.1, §8](../1-elicitation/domain-description.md#8-глоссарий) |
| FR-006 | Клиент может **записаться на выбранный слот**, указав **имя и телефон** (при первой записи). | Must | [domain-description.md §3, §5.2](../1-elicitation/domain-description.md#бронирование-и-экипировка) |
| FR-007 | При записи клиент указывает **своё снаряжение** или **прокат** (инструменты, фартук). | Must | [domain-description.md §3, §5.2](../1-elicitation/domain-description.md#52-бронирование) |
| FR-008 | При исчерпании прокатного фонда запись на слот **возможна только «со своим»** снаряжением. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#бронирование-и-экипировка) |
| FR-009 | При бронировании приложение отправляет запрос бэкенду и **отображает результат**: подтверждение или отказ при отсутствии мест (R-004). | Must | [domain-description.md §3, §5.2](../1-elicitation/domain-description.md#52-бронирование) |
| FR-010 | Система не допускает **более одной записи в день** и **более одного участника** в одной записи на клиента. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#вместимость-и-расписание) |
| FR-011 | При заполненной группе приложение отображает **«мест нет»** (лист ожидания не реализуется). | Must | [domain-description.md §3, §6](../1-elicitation/domain-description.md#вне-скоупа--backlog) |
| FR-012 | Приложение **отображает цену** занятия (зависит от программы); оплата выполняется **на месте**. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#бронирование-и-экипировка) |
| FR-013 | Клиент может **просматривать свои брони** со статусами: активна, отменена клиентом, отменена мастерской, посещена и др. | Must | [domain-description.md §2](../1-elicitation/domain-description.md#2-ключевые-сущности) |
| FR-014 | Клиент может **отменить свою бронь**; при отмене **≥ 3 ч** до начала место в слоте **сразу** освобождается. | Must | [domain-description.md §3, §5.3](../1-elicitation/domain-description.md#53-отмена-брони) |
| FR-015 | При отмене **менее чем за 3 ч** до начала приложение показывает **предупреждение** о заготовленной глине; отмена **разрешена**. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#отмена-клиентом) |
| FR-016 | При отмене занятия **мастерской** бронь сохраняется со статусом **«Отменено мастерской»** и **причиной** (R-008). | Must | [domain-description.md §3, §5.5](../1-elicitation/domain-description.md#отмена-мастерской-форс-мажор-r-008) |
| FR-017 | Клиенту отправляется **push-уведомление** при отмене занятия мастерской с просьбой **перезаписаться на другое занятие** (R-008). | Must | [domain-description.md §3](../1-elicitation/domain-description.md#отмена-мастерской-форс-мажор-r-008) |
| FR-018 | Клиент может **перезаписаться на другое занятие** из push-уведомления и с **экрана детали** записи. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#отмена-мастерской-форс-мажор-r-008) |
| FR-019 | Повторная запись клиента на **тот же отменённый мастерской слот** запрещена (R-008). | Must | [domain-description.md §3](../1-elicitation/domain-description.md#отмена-мастерской-форс-мажор-r-008) |
| FR-020 | Клиенту отправляется **push-уведомление** при **переносе** занятия (смена времени или мастера). | Must | [domain-description.md §3](../1-elicitation/domain-description.md#отмена-мастерской-форс-мажор-r-008) |
| FR-021 | После **посещённого** занятия клиент может **оценить мастера** звёздами в течение **недели**. | Must | [domain-description.md §3, §5.4](../1-elicitation/domain-description.md#54-оценка-мастера) |
| FR-022 | Клиент может иметь **не более одной оценки на мастера**; оценку можно **изменить** в течение недели или при повторном занятии с тем же мастером. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#оценки-мастеров) |
| FR-023 | **Рейтинги мастеров** отображаются **другим клиентам** при выборе слота. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#оценки-мастеров) |
| FR-024 | Приложение отправляет клиенту **push-уведомления**: напоминание за день, за 2 часа, подтверждение записи, отмена (клиент / мастерская), перенос занятия. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#уведомления-mvp) |
| FR-025 | Для **постоянных клиентов** отображается **метка** в профиле; при записи применяется **приоритет**. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#постоянные-клиенты) |
| FR-026 | Слоты, программы и данные мастеров **поступают из существующего бэкенда** через API; приложение их потребляет, но **не создаёт и не редактирует**. | Must | [domain-description.md §1, §6](../1-elicitation/domain-description.md#6-границы-системы) |
| FR-027 | Клиентское приложение реализует сценарии **только для роли «Клиент»** (R-028). | Must | [domain-description.md §4, §6](../1-elicitation/domain-description.md#4-акторы) |
| FR-028 | Client API обеспечивает операции по контракту: **слоты**, **бронирования**, **профиль**, **мастера/рейтинги**, **прокат** (R-015). | Must | [domain-description.md §6](../1-elicitation/domain-description.md#в-скоупе-mvp) |
