## Обзор

Экран показывает полную информацию о конкретной записи клиента и доступные действия по статусу. Экран открывается как из списка SCR-008, так и через deep link из push-уведомлений (отмена мастерской, перенос, напоминания), включая отображение причины `CANCELLED_BY_WORKSHOP` и данных о переносе.

### User Story

> Как клиент, я хочу открыть детальную карточку записи, чтобы понять актуальный статус, причину изменений и выполнить действие (отмена, оценка мастера, перезапись).

**Не в MVP:** лист ожидания при `NO_SPOTS` (FR-011) — только возврат к расписанию; фильтр по мастеру; iOS; редактирование аллергий (в домене «Глина» отсутствует).

## Используемые запросы

### getBooking

**Метод:** GET  
**Путь:** `/bookings/{bookingId}`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `getBooking`

**Обработка ответа:**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 200 + data | Content: заполнение всех блоков, CTA по статусу |
| 200 + `status=CANCELLED_BY_WORKSHOP` | Показ warning-блока с `cancellationReason` |
| 200 + `rescheduleInfo != null` | Показ info-блока переноса с новыми данными |
| 404 | Экран «Запись не найдена» + возврат на SCR-008 |
| 401 | Завершение сессии и выход в поток повторной идентификации |
| 5xx / timeout | Error + «Повторить» |
| offline + кэш | Offline state с read-only деталями |

## Макет экрана

> Блоки **warning «Отменено мастерской»** и **info «Перенос»** — **взаимоисключающие** варианты (или перенос показывается для `ACTIVE` при `rescheduleInfo != null`). Ниже — два примера состояний, не один экран.

**Активная запись с переносом:**

```
┌─────────────────────────────────────┐
│ ← Деталь записи                     │
├─────────────────────────────────────┤
│ [ЗАПИСАН]                           │
│ Ср, 9 июл · 14:00–16:30             │
│ Программа: Лепка для начинающих     │
│ Мастер: Анна К. ★ 4.8               │
│ ℹ Занятие перенесено на 12 июл 15:00│
│   Мастер: Мария П.                  │
│ [ Отменить запись ]                 │
└─────────────────────────────────────┘
```

**Отменена мастерской:**

```
┌─────────────────────────────────────┐
│ ← Деталь записи                     │
├─────────────────────────────────────┤
│ [ОТМЕНЕНО МАСТЕРСКОЙ]              │
│ Ср, 9 июл · 14:00–16:30             │
│ ⚠ Причина: Поломка печи             │
│ [Перезаписаться на другое занятие]  │
└─────────────────────────────────────┘
```

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Бейдж статуса | Крупный индикатор состояния записи | `status` | Карта значений как на SCR-008 |
| Блок занятия | Программа, мастер с рейтингом, адрес, время | `slot.*`, `workshop.*` | Читаемое представление для открытия из push |
| Блок цены | Стоимость и подпись «оплата на месте» | `totalPrice` | Не пересчитывать на клиенте |
| Блок снаряжения | Своё / прокат | `equipmentChoice` | Только отображение; редактирование не предусмотрено |
| Warning отмены мастерской | Причина отмены | `cancellationReason` | Показывать только для `CANCELLED_BY_WORKSHOP` |
| Info переноса | Детали изменения слота | `rescheduleInfo` | Показывать при наличии данных о переносе (новое время и/или мастер) |
| CTA «Отменить запись» | Переход в SCR-010 | `canCancel` | Только для активной, не начавшейся записи |
| CTA «Оценить мастера» | Переход в SCR-011 | `status`, `masterRating`, дата окончания | Только для `ATTENDED` в окне 7 дней |
| CTA «Перезаписаться на другое занятие» | Переход в SCR-001 | `status` | Только для `CANCELLED_BY_WORKSHOP` (FR-018) |

**Терминология:** **мастер**, **программа занятия** (лепка / работа на круге), **мастерская «Глина»**; прокат (инструменты, фартук) **не влияет на цену** (FR-012).

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** вход на экран по `bookingId`, **Когда** `getBooking=200`, **Тогда** отображаются дата/время, программа, мастер, цена, снаряжение, статус и данные записи. |
| AC-002 | **Дано** запись со статусом `CANCELLED_BY_WORKSHOP`, **Когда** открыт SCR-009, **Тогда** показывается warning-блок с текстом `cancellationReason` и CTA «Перезаписаться на другое занятие» → SCR-001. |
| AC-003 | **Дано** запись содержит `rescheduleInfo`, **Когда** экран загружен, **Тогда** отображается отдельный инфо-блок переноса с изменившимися полями (время и/или мастер). |
| AC-004 | **Дано** запись `ACTIVE` и `canCancel=true`, **Когда** пользователь нажимает «Отменить запись», **Тогда** выполняется переход на SCR-010 с параметрами отмены. |
| AC-005 | **Дано** запись `ATTENDED`, `masterRating=null` и с момента окончания прошло не более 7 дней, **Когда** открыт экран, **Тогда** доступен CTA «Оценить мастера». |
| AC-006 | **Дано** push с deep link `app://bookings/{bookingId}`, **Когда** пользователь тапает по уведомлению, **Тогда** открывается SCR-009 с деталями соответствующей записи (NFR-010). |
| AC-007 | **Дано** устройство offline и есть кэш детали, **Когда** открыт SCR-009, **Тогда** показываются кэшированные данные, а действия отмены/оценки отключены. |
