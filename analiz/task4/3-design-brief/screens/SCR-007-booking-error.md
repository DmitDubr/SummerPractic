## Обзор

Dialog/modal информирует клиента об отказе бронирования после submit на SCR-005. Покрывает гонку за последнее место, лимит одной записи в день, отмену слота, исчерпание проката и запрет повторной записи на отменённый мастерской слот.

### User Story

> Как клиент, я хочу понять, почему запись не удалась, и быстро выбрать следующий шаг — исправить форму, перейти к расписанию или открыть «Мои записи».

**Не в MVP:** лист ожидания при `NO_SPOTS` (FR-011) — только возврат к расписанию; фильтр по мастеру; iOS.

## Навигация

### Входящая

| Источник | Триггер | Условие | Параметры |
| :-- | :-- | :-- | :-- |
| SCR-005 | Ошибка `createBooking` | HTTP 409, 403, 422 или доменный код | `errorCode`, `slotId` |
| SCR-005 | Сетевая ошибка после retry | 5xx, timeout | `errorCode = NETWORK_ERROR`, `slotId` |

### Исходящая

| Назначение | Триггер | Параметры |
| :-- | :-- | :-- |
| SCR-001 | CTA «К расписанию» | Закрыть modal + обновить список слотов |
| SCR-008 | CTA «Мои записи» | При `ALREADY_BOOKED_TODAY` |
| SCR-005 | CTA «Понятно» / закрытие (✕) / tap outside | Форма с сохранёнными данными |
| SCR-005 | CTA «Повторить» | При `NETWORK_ERROR` — повторный `POST /bookings` |

> **Нижняя навигация:** 2 вкладки — «Расписание» (SCR-001) | «Мои записи» (SCR-008). Отдельной вкладки «Профиль» нет.

## Используемые запросы

### createBooking (контекст ошибки, выполнен на SCR-005)

**Метод:** POST  
**Путь:** `/bookings`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `createBooking`

**Обработка ответа (переход на SCR-007):**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 403 `SLOT_REBOOK_FORBIDDEN` | SCR-007 modal |
| 409 `NO_SPOTS` | SCR-007 modal, **без waitlist CTA** |
| 409 `ALREADY_BOOKED_TODAY` | SCR-007 modal + CTA «Мои записи» |
| 409 `SLOT_CANCELLED` | SCR-007 modal |
| 409 `RENTAL_UNAVAILABLE` | SCR-007 modal + рекомендация «со своим» |
| 5xx / timeout | SCR-007 modal с `NETWORK_ERROR` или Generic |
| 422 / прочие 4xx | SCR-007 modal Generic |

**Доменные коды createBooking:** `NO_SPOTS`, `ALREADY_BOOKED_TODAY`, `SLOT_CANCELLED`, `RENTAL_UNAVAILABLE`, `SLOT_REBOOK_FORBIDDEN`.

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Кнопка закрытия (✕) | Закрывает modal | Локально | Форма SCR-005 сохранена; эквивалент «Понятно» |
| Иконка | Предупреждение (amber), не panic-red | Локально | Ситуация штатная, не системный сбой |
| Заголовок | Зависит от `errorCode` | `errorCode` | Коды ошибок не показывать пользователю |
| Текст пояснения | 1-2 предложения, конкретная причина | `errorCode` | Дружелюбный тон |
| CTA «К расписанию» | Primary для большинства сценариев | Навигация | После перехода обновить слоты |
| CTA «Мои записи» | Secondary при `ALREADY_BOOKED_TODAY` | Навигация | Переход в SCR-008 |
| CTA «Понятно» | Закрыть modal, остаться на форме | Локально | Доступен где указано в таблице состояний |
| CTA «Повторить» | При `NETWORK_ERROR` | SCR-005 | Повторный `createBooking` без потери полей |
| Затемнение фона | Overlay 40-60% | Локально | Tap outside = «Понятно» (где применимо) |

**Терминология:** **мастер**, **занятие**, **программа**; прокат (инструменты, фартук) **не влияет на цену** (FR-012).

## Состояния экрана

| `errorCode` (API) | Заголовок | Текст | CTA |
| :-- | :-- | :-- | :-- |
| `NO_SPOTS` | Мест больше нет | Пока вы оформляли запись, последнее место заняли. Выберите другое занятие в расписании. | «К расписанию» |
| `ALREADY_BOOKED_TODAY` | Уже есть запись на этот день | Можно записаться не более одного раза в день. Отмените текущую запись или выберите другой день. | «К расписанию», «Мои записи» |
| `SLOT_CANCELLED` | Занятие отменено | Это занятие больше недоступно. Выберите другой слот в расписании. | «К расписанию» |
| `RENTAL_UNAVAILABLE` | Прокат закончился | Инструменты или фартук на это время уже разобрали. Запишитесь **со своим** снаряжением или выберите другое занятие. | «Понятно», «К расписанию» |
| `SLOT_REBOOK_FORBIDDEN` | Запись на это занятие недоступна | Вы уже были записаны на это занятие, которое отменила мастерская. Выберите другой слот. | «К расписанию» |
| `NETWORK_ERROR` | Не удалось записаться | Проверьте подключение к интернету и попробуйте снова. | «Повторить», «К расписанию» |
| Generic | Не удалось записаться | Попробуйте ещё раз или выберите другое занятие. | «Понятно», «К расписанию» |

> SCR-007 — dialog overlay поверх SCR-005. Фоновая форма остаётся в Content или переходит в Submitting при «Повторить».

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** `createBooking` вернул 409 `NO_SPOTS`, **Когда** открывается SCR-007, **Тогда** показаны заголовок «Мест больше нет» и CTA «К расписанию»; CTA «В лист ожидания» отсутствует. |
| AC-002 | **Дано** `createBooking` вернул 409 `ALREADY_BOOKED_TODAY`, **Когда** modal открыт, **Тогда** доступны CTA «К расписанию» и «Мои записи». |
| AC-003 | **Дано** `createBooking` вернул 409 `RENTAL_UNAVAILABLE`, **Когда** пользователь нажал «Понятно», **Тогда** modal закрывается, SCR-005 сохраняет данные формы и позволяет выбрать «Со своим». |
| AC-004 | **Дано** `createBooking` вернул 403 `SLOT_REBOOK_FORBIDDEN`, **Когда** modal открыт, **Тогда** отображается текст о запрете повторной записи и CTA «К расписанию». |
| AC-005 | **Дано** сетевая ошибка (5xx / timeout), **Когда** SCR-007 с `NETWORK_ERROR`, **Тогда** показаны CTA «Повторить» и «К расписанию», при этом «Повторить» инициирует повторный submit на SCR-005. |
| AC-006 | **Дано** modal открыт поверх SCR-005, **Когда** пользователь нажал ✕ или «Понятно», **Тогда** форма SCR-005 отображается с сохранёнными полями. |
| AC-007 | **Дано** пользователь выбрал «К расписанию», **Когда** выполнен переход в SCR-001, **Тогда** список слотов обновляется для актуальной доступности «есть места / мест нет». |
| AC-008 | **Дано** любой `errorCode`, **Когда** modal отображён, **Тогда** технические коды (`NO_SPOTS`, `ALREADY_BOOKED_TODAY` и т.д.) не видны пользователю; показываются только русские заголовки и пояснения. |
