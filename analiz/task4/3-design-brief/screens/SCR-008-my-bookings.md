## Обзор

Экран показывает список записей текущего клиента и разделяет его на сегменты «Предстоящие» и «Прошедшие». Данные загружаются через `listBookings` под `ClientSession`, а при отсутствии сети используется локальный кэш последнего успешного ответа (NFR-009).

### User Story

> Как клиент, я хочу видеть свои предстоящие и прошедшие записи, чтобы контролировать посещения мастер-классов и быстро открывать детали нужной брони.

**Не в MVP:** лист ожидания при `NO_SPOTS` (FR-011) — только возврат к расписанию; фильтр по мастеру; iOS.

## Используемые запросы

### listBookings

**Метод:** GET  
**Путь:** `/bookings`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `listBookings`

**Обработка ответа:**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 200 + `items.length > 0` | Content, сохранение в офлайн-кэш |
| 200 + `items.length = 0` | Empty (текст зависит от активного сегмента) |
| 401 | Завершение сессии, переход в поток первичной идентификации |
| 5xx / timeout | Error + «Повторить» |
| offline + есть кэш | Offline state + баннер «Показаны сохранённые данные» |

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Сегмент «Предстоящие/Прошедшие» | Локальное переключение двух списков | UI state + `listBookings.items` | Не вызывает повторный API-запрос |
| Карточка записи | Дата, время, программа, мастер, цена, статус | `slot.startsAt`, `slot.program.name`, `slot.master.fullName`, `totalPrice`, `status` | Вся карточка кликабельна, переход на SCR-009 |
| Бейдж статуса | Текстовое состояние записи | `status` | `ACTIVE`→«Записан», `CANCELLED_BY_CLIENT`→«Отменена вами», `CANCELLED_BY_WORKSHOP`→«Отменено мастерской», `ATTENDED`→«Посещена» |
| Офлайн-баннер | Сообщение о кэше | Статус сети + timestamp кэша | Показывается только в offline режиме |
| Empty state | Сообщение об отсутствии записей | Пустой список в активном сегменте | Для «Предстоящие» CTA на SCR-001 |
| Pull-to-refresh | Обновление данных | Жест пользователя | Активен только online |

**Терминология:** **мастер**, **программа занятия** (лепка / работа на круге), **мастерская «Глина»**; прокат (инструменты, фартук) **не влияет на цену** (FR-012).

## Состояния экрана

| Состояние | Условие | Отображение |
| :-- | :-- | :-- |
| Loading | Первый запрос в процессе | Skeleton карточек |
| Content | 200 + записи есть в выбранном сегменте | Список карточек |
| Empty | 200 + выбранный сегмент пуст | Текст + CTA (для «Предстоящие») |
| Error | 4xx / 5xx на первой загрузке | Ошибка + «Повторить» |
| Offline | Нет сети, кэш есть | Кэшированный список + офлайн-баннер |
| Refreshing | Pull-to-refresh online | Индикатор обновления поверх списка |

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** открыта вкладка «Мои записи» online, **Когда** `listBookings` возвращает список, **Тогда** показывается сегмент «Предстоящие» по умолчанию и карточки ближайших записей. |
| AC-002 | **Дано** в активном сегменте нет записей, **Когда** экран перешёл в Empty, **Тогда** показывается релевантный текст и CTA «Посмотреть расписание» для «Предстоящие». |
| AC-003 | **Дано** пользователь переключил сегмент, **Когда** меняется `selectedSegment`, **Тогда** список фильтруется локально без повторного вызова `listBookings`. |
| AC-004 | **Дано** устройство offline и есть кэш списка, **Когда** экран открыт, **Тогда** показываются сохранённые данные и баннер «Показаны сохранённые данные». |
| AC-005 | **Дано** выполнен tap по карточке записи, **Когда** передан `bookingId`, **Тогда** открывается SCR-009 с деталями выбранной записи. |
| AC-006 | **Дано** пользователь вернулся из SCR-010 после успешной отмены, **Когда** экран получает `refresh=true`, **Тогда** карточка отменённой записи показывает статус «Отменена вами». |
| AC-007 | **Дано** на карточке отображается цена и выбран прокат, **Когда** строится UI, **Тогда** сумма берётся из API и не меняется из-за проката. |
