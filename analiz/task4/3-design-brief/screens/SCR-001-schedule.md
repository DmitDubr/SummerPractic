## Обзор

Главный экран приложения гончарной мастерской «Глина». Показывает список доступных занятий за период, по умолчанию — ближайшие 7 дней (R-027), позволяет открыть фильтр дат и фильтр занятий, а также перейти в детали выбранного мастер-класса.

### User Story

> Как клиент, я хочу видеть ближайшие мастер-классы и быстро отфильтровать расписание, чтобы выбрать подходящее занятие и перейти к записи.

**Не в MVP:** лист ожидания, фильтр по мастеру, номер гончарного круга, iOS.

## Используемые запросы

### listSlots

**Метод:** GET  
**Путь:** `/slots`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `listSlots`

**Query-параметры экрана:**

| Параметр | Обязательность | Источник |
| :-- | :-- | :-- |
| `dateFrom` | Нет | SCR-002 |
| `dateTo` | Нет | SCR-002 |
| `timeOfDay` | Нет | SCR-003 |
| `level` | Нет | SCR-003 |
| `programIds` | Нет | SCR-003 |
| `limit` | Нет | Клиент (`50`) |
| `offset` | Нет | Клиент (`0`) |

**Обработка ответа:**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 200 + `items.length > 0` | Content: карточки, отсортированные по `startsAt` и сгруппированные по дням |
| 200 + `items.length = 0` + дефолтный период/фильтры | Empty: «Пока нет доступных занятий» |
| 200 + `items.length = 0` + активные фильтры | Empty: «Ничего не найдено», CTA «Сбросить фильтры» |
| 400 | Error: «Проверьте параметры фильтра» + «Повторить» |
| 500 / timeout | Error: «Не удалось загрузить расписание» + «Повторить» |

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Заголовок «Расписание» | Название вкладки | Константа UI | Непрокручиваемый header |
| Чип «Период» | Текущее окно дат | Локальное состояние + SCR-002 | По умолчанию: «7 дней»; при кастомном диапазоне: `D–D MMM` |
| Чип «Фильтры» | Открытие фильтров занятий | Локальное состояние + SCR-003 | Badge = число активных категорий (0..3) |
| Заголовок группы дня | День и дата | `items[].startsAt` | Формат: «Сегодня/Завтра/...», sticky |
| Время начала | Время конкретного занятия | `items[].startsAt` | Формат `HH:mm`, локаль ru-RU |
| Название программы | Краткое имя занятия | `items[].program.name` | Максимум 2 строки, затем ellipsis |
| Имя мастера | Ведущий занятие | `items[].master.fullName` | Обязательно для отображения карточки |
| Рейтинг мастера | Публичный рейтинг | `items[].master.avgRating`, `items[].master.ratingCount` | При `ratingCount = 0` — «Пока нет оценок»; при `avgRating` — `★ X.X` |
| Доступность мест | Наличие мест | `items[].freeSpots` | При `freeSpots > 0` — «Есть места»; при `freeSpots = 0` — «Мест нет». Счётчик X/Y и номер круга **не показываются** |
| Цена | Цена занятия | `items[].price` | Формат: `N ₽` |
| Карточка занятия | CTA переход в детали | `items[].id` | Тап по всей карточке ведёт в SCR-004 с `slotId` |

**Терминология:** использовать «мастер», «мастер-класс», «занятие», «программа», «мастерская». Термины «шеф», «класс кулинарии», «тип кухни» не использовать.

## Состояния экрана

| Состояние | Условие | Отображение |
| :-- | :-- | :-- |
| Loading | Первый запрос `listSlots` в процессе | Skeleton списка под группами дней |
| Content | 200 + `items.length > 0` | Сгруппированный список карточек |
| Empty | 200 + `items.length = 0` | «Пока нет доступных занятий» / «Ничего не найдено» с CTA |
| Error | 4xx/5xx/timeout | Сообщение об ошибке + «Повторить» |
| Offline | Нет сети на SCR-001 | Показывается как Error (кэш не используется в MVP) |
| Submitting | Применение фильтров/refresh в процессе | Локальный loader в контролах, контент не блокируется полностью |

> Для pull-to-refresh используется под-состояние Refreshing поверх Content согласно LOGIC-008.

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** пользователь открыл SCR-001 без параметров, **Когда** выполняется `listSlots`, **Тогда** отправляется запрос без `dateFrom/dateTo` и показываются ближайшие 7 дней. |
| AC-002 | **Дано** `listSlots` вернул непустой список, **Когда** экран отрисован, **Тогда** карточки отсортированы по `startsAt` и сгруппированы по дням. |
| AC-003 | **Дано** `listSlots` вернул пустой список при дефолтных фильтрах, **Когда** загрузка завершилась, **Тогда** отображается текст «Пока нет доступных занятий». |
| AC-004 | **Дано** активны фильтры и `listSlots` вернул пусто, **Когда** экран показывает Empty, **Тогда** отображается «Ничего не найдено» и CTA «Сбросить фильтры». |
| AC-005 | **Дано** в слоте `freeSpots = 0`, **Когда** карточка отображается, **Тогда** пользователь видит «Мест нет» (без счётчика X/Y) и может открыть детали занятия (без листа ожидания). |
| AC-006 | **Дано** в слоте `freeSpots > 0`, **Когда** карточка отображается, **Тогда** пользователь видит «Есть места» (без счётчика X/Y и номера круга). |
| AC-007 | **Дано** `master.ratingCount > 0`, **Когда** карточка занятия отображается, **Тогда** рейтинг показан в формате `★ X.X`; при `ratingCount = 0` — текст «Пока нет оценок». |
| AC-008 | **Дано** пользователь потянул список вниз, **Когда** запрос refresh завершился ошибкой, **Тогда** ранее загруженный список остаётся на экране и показывается сообщение об ошибке обновления. |
| AC-009 | **Дано** пользователь нажал «Фильтры», выбрал 2 категории и применил, **Когда** возврат на SCR-001 завершён, **Тогда** chip «Фильтры» показывает badge `2` и список перезагружен с новыми параметрами. |
