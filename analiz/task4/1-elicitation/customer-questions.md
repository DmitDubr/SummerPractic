## 1. Запись и расписание

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 1.1 | Кто может записаться на мастер-класс? Нужна ли регистрация/авторизация, или достаточно **имени и телефона** при первой записи? | Достаточно **имени и телефона** (первая итерация) |
| 1.2 | Может ли один человек записать **нескольких участников** (пара, ребёнок с родителем, друзья)? | **Нет** — в одной записи один человек |
| 1.3 | Сколько **активных записей** может иметь один клиент одновременно? Можно ли записаться на **два занятия в один день**? | **Запись на 2 занятия в 1 день невозможна** (не более 1 записи в день) |
| 1.4 | Если группа заполнена — показываем только «мест нет» или нужен **лист ожидания** при освобождении места? | Только **«мест нет»** (лист ожидания не нужен) |
| 1.5 | **Подтверждение (R-027):** по умолчанию показываем слоты на **7 дней**, более длинный период — через фильтр дат? | **Да**, по R-027 |
| 1.6 | Какие **фильтры** нужны клиенту в первой версии: мастер, время суток, уровень (новичок / опытный), **программа** (лепка / круг)? | **Время суток**, **уровень**, **программа** (лепка / круг) — мастер — не в первой версии |
| 1.7 | Длительность занятия **2–2,5 часа** — всегда диапазон или фиксированная длительность по программе? Нужно ли показывать **время окончания** в приложении? | Пока все занятия **2–2,5 часа**; время окончания — по усмотрению UI (не уточнялось) |
| 1.8 | Нужно ли в карточке занятия показывать **описание программы** — полностью, кратко или только название? | **Кратко** |

## 2. Программы, круги и вместимость

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 2.1 | Как система определяет лимит **6 человек на новичковую группу** — отдельный **тип программы**, правило «если программа X — лимит 6» или настройка **на уровне мастера**? | Настраивается **в зависимости от мастера**, который ведёт занятие |
| 2.2 | На занятиях **на гончарном круге** — лимит привязан к числу **свободных кругов** (до 10) или есть отдельный потолок группы? | Лимит привязан к **свободным кругам** — **до 10** |
| 2.3 | Нужна ли **проверка уровня** клиента? Может ли новичок записаться на программу «**работа на круге**» без предварительного опыта? | **Нет** — проверка не нужна, запись на усмотрение клиента |
| 2.4 | Что именно выбирает клиент при записи: **свои инструменты и фартук** или **прокат** (инструменты, фартук, оба)? Влияет ли выбор на **цену**? | Выбор своё / прокат (инструменты, фартук); на **цену не влияет** |
| 2.5 | Ограничен ли **прокатный фонд** (инструменты / фартуки)? Что показывать, если прокат на слот закончился — слот **недоступен** или запись **только «со своим»**? | Запись **возможна**, но **только со своим** снаряжением |
| 2.6 | **Сезонные помощники** — для клиента это просто другой мастер в карточке или нужна отдельная пометка («помощник» / «стажёр»)? | Просто **другой мастер** в карточке (без отдельной пометки) |
| 2.7 | Клиенту важно видеть **номер круга** или достаточно «есть места / мест нет» (назначение круга — на месте)? | Достаточно **«есть места» / «мест нет»** (номер круга в UI не показываем) |

## 3. Отмена клиентом

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 3.1 | Правило «**за 10 минут до начала**» — жёсткий **запрет** отмены, **штраф**, **предупреждение** или только **пометка** для администратора? | **Предупреждение** (отмена разрешена; штрафов в MVP нет) |
| 3.2 | Что считается «**заранее**» — 24 часа, 3 часа, 1 час, любое время до порога поздней отмены? | **За 3 часа и более** до начала — ранняя отмена |
| 3.3 | Нужен ли **учёт** частых поздних отмен и **неявок**? Есть ли последствия для клиента в первой версии? | **Учёт нужен**; **последствий для клиента пока нет** |
| 3.4 | После отмены клиентом место **сразу** становится доступным другим (и для листа ожидания)? | **Да**, место сразу доступно другим |
| 3.5 | При **поздней отмене** заготовленная **глина простаивает** — нужно ли показывать клиенту отдельное **предупреждение** о последствиях? | **Да**, показывать предупреждение |

## 4. Отмена занятия мастерской (форс-мажор)

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 4.1 | **Подтверждение (R-008):** при отмене мастерской бронь получает статус «**Отменено мастерской**» с **причиной**, клиенту приходит **push**, повторная запись на **тот же слот запрещена**? | **Да** (R-008): статус + причина + запрет повторной записи на слот; **push** с просьбой **перезаписаться на другое занятие** |
| 4.2 | Типовые **причины** отмены для клиента: поломка печи, отключение электричества, болезнь мастера — **фиксированный список** или **свободный текст** из админки? | **Фиксированный список** + возможность **свободного текста** из админки (непредвиденные обстоятельства) |
| 4.3 | Клиенту предлагать **перезапись на другое занятие** из push / экрана детали записи? | **Да** — из **push** и с **экрана детали** записи |
| 4.4 | Если позже подключим **онлайн-оплату** — что делать с деньгами при отмене мастерской: возврат, перенос, абонемент? | **Перенос**, если клиент согласен; **возврат**, если не хочет переносить |
| 4.5 | Нужно ли уведомление о **переносе** занятия (смена времени / мастера **без** полной отмены)? | **Да**, нужно |

## 5. Оценки мастеров

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 5.1 | Когда клиент может поставить оценку — только после **посещённого** занятия? В течение какого **срока**? | **Только после посещённого** занятия; срок — **в течение недели** |
| 5.2 | Какой **формат** оценки: звёзды 1–5, текстовый отзыв, анонимно или с именем? | Только **звёзды** (без текстового отзыва) |
| 5.3 | Видят ли **другие клиенты** рейтинги мастеров при выборе слота, или оценки только для владельца? | **Да**, клиенты **видят** рейтинги при выборе слота |
| 5.4 | Оценка привязана к **конкретной записи** (одно занятие — одна оценка) или можно оценить мастера **повторно** при следующем визите? | **Один пользователь — одна оценка на мастера**; можно **изменить** в течение периода оценивания (неделя) или при **повторном занятии** с тем же мастером |

## 7. Оплата и лояльность

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 7.1 | В MVP **без онлайн-оплаты** клиент записывается бесплатно и платит на месте (наличные / перевод)? Нужно ли **показывать цену** в приложении? | **Запись бесплатно**, оплата **на месте**; **цену в приложении показываем** |
| 7.2 | Цена зависит только от **программы** или ещё от **проката** инструментов / фартука? | Цена зависит от **программы**; **прокат не влияет** на цену *(исключение: повреждение / поломка прокатного оборудования — отдельный случай)* |
| 7.3 | Что нужно для **постоянных клиентов** в первой версии — метка в профиле, скидка, приоритет записи, или откладываем? | **Метка** в профиле + **приоритет записи** (без скидок на старте) |
| 7.4 | **Подтверждение:** онлайн-оплата точно **не входит** в первую версию? | **Точно не входит** в первую версию |

## Открытые противоречия из брифа — статус

| # | Противоречие / пробел | Решение |
| :- | :-- | :-- |
| О-1 | «За 10 минут до начала отмена — проблема» vs правило в приложении | **Предупреждение**; ранняя отмена — **≥ 3 ч**; поздняя — предупреждение о заготовленной глине (§3) |
| О-2 | Онлайн-оплата на старте | **Не входит** в первую версию; оплата на месте, цена в UI (§7) |
| О-3 | 10 кругов vs лимит 6 на новичковые группы | Лимит настраивается **в зависимости от мастера**, ведущего занятие (§2.1) |
| О-4 | Постоянные клиенты | **Метка** + **приоритет записи** в MVP; скидки — позже (§7.3) |
| О-5 | Марина хочет экран расписания для себя | **Не входит** в текущую поставку (R-028, §8.2) |
| О-6 | Двойная запись на один круг | Гарантия на стороне **бэкенда** (R-004); клиент обрабатывает отказ API |
