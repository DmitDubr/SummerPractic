# Бизнес-требования — кулинарная студия «Шеф-стол»

> Источник: [domain-description.md](../1-elicitation/domain-description.md).
> Приоритет: **Must** — обязательно в MVP; **Should** — важно, но может следовать за MVP; **Could** — желательно позже.

| ID | Формулировка | Приоритет | Источник |
| :- | :-- | :--: | :-- |
| BR-001 | Студия должна предоставить клиентам **самообслуживание записи** на кулинарные классы через мобильное приложение, чтобы снизить нагрузку на ручную обработку заявок (WhatsApp, Google-таблица). | Must | [domain-description.md §1, §7](../1-elicitation/domain-description.md#7-болевые-точки) |
| BR-002 | Система записи должна **исключить двойное бронирование** одного рабочего места, чтобы клиенты не приходили на уже занятый класс. | Must | [domain-description.md §3, §7](../1-elicitation/domain-description.md#3-бизнес-правила) |
| BR-003 | Клиент должен **самостоятельно видеть свободные места** и записываться на класс без участия владельца в каждой операции. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#3-бизнес-правила) |
| BR-004 | Операционное управление расписанием, шефами и отменами по форс-мажору остаётся в **существующей инфраструктуре**; новое решение дополняет её клиентским каналом, не заменяя админку. | Must | [domain-description.md §1, §6](../1-elicitation/domain-description.md#6-границы-системы) |
| BR-005 | При отмене класса **студией** (форс-мажор) клиент должен **своевременно получать информацию** о статусе брони, причине отмены и возможности перезаписаться на другой класс. | Must | [domain-description.md §3, §5.6](../1-elicitation/domain-description.md#отмена-студией-форс-мажор-r-008) |
| BR-006 | Студия должна получать **обратную связь по качеству работы шефов** от клиентов после классов для оценки нагрузки и удовлетворённости. | Must | [domain-description.md §3, §5.5](../1-elicitation/domain-description.md#оценки-шефов) |
| BR-007 | Система должна способствовать **снижению неявок** на записанные классы за счёт push-напоминаний клиентам. | Must | [domain-description.md §3, §5.4](../1-elicitation/domain-description.md#уведомления-mvp) |
| BR-008 | Групповые классы должны соблюдать **лимиты вместимости**: до 12 рабочих мест; лимит **8 или 12** на конкретный класс определяется **шефом**, ведущим занятие. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#вместимость-и-расписание) |
| BR-009 | Клиент должен иметь возможность указать **своё или прокатное** снаряжение (фартук, ножи) при записи, чтобы студия планировала выдачу инвентаря. | Must | [domain-description.md §2, §5.2](../1-elicitation/domain-description.md#бронирование-и-экипировка) |
| BR-010 | При **ранней** отмене клиентом (≥ 3 ч до начала) освобождённое место должно **сразу** становиться доступным для других. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#отмена-клиентом) |
| BR-011 | Студия должна **заранее получать информацию об аллергиях** клиента, чтобы снизить риски на кухне. | Must | [domain-description.md §3, §5.2, §5.4](../1-elicitation/domain-description.md#аллергии) |
| BR-012 | **Постоянные клиенты** должны быть **отмечены** в системе и получать **приоритет записи** для удержания аудитории. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#постоянные-клиенты) |
