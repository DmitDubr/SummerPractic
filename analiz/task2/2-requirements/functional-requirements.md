# Функциональные требования — кулинарная студия «Шеф-стол»

> Источник: [domain-description.md](../1-elicitation/domain-description.md).
> ФТ описывают **что** делает система. Нефункциональные аспекты — в [non-functional-requirements.md](non-functional-requirements.md).
> Приоритет: **Must** / **Should** / **Could**.

| ID | Формулировка | Приоритет | Источник |
| :- | :-- | :--: | :-- |
| FR-001 | Клиентское приложение отображает список доступных слотов на **ближайшие 7 дней** от текущего момента (R-027). | Must | [domain-description.md §3, §5.1](../1-elicitation/domain-description.md#51-просмотр-расписания) |
| FR-002 | Клиент может **расширить период** просмотра расписания с помощью фильтра дат. | Must | [domain-description.md §3, §5.1](../1-elicitation/domain-description.md#51-просмотр-расписания) |
| FR-003 | Клиент может **фильтровать** расписание по **времени суток**, **уровню** и **типу кухни**. | Must | [domain-description.md §3, §5.1](../1-elicitation/domain-description.md#вместимость-и-расписание) |
| FR-004 | Для каждого слота отображаются: **время начала**, **программа (кратко)**, **шеф**, **рейтинг шефа**, **количество свободных мест**, **цена**. | Must | [domain-description.md §5.1](../1-elicitation/domain-description.md#51-просмотр-расписания) |
| FR-005 | При отсутствии доступных слотов на ближайшие дни приложение показывает empty state: **«Пока нет доступных классов»**. | Must | [domain-description.md §5.1, §8](../1-elicitation/domain-description.md#8-глоссарий) |
| FR-006 | Клиент может **записаться на выбранный слот**, указав **имя и телефон** (при первой записи). | Must | [domain-description.md §3](../1-elicitation/domain-description.md#бронирование-и-экипировка) |
| FR-007 | При записи клиент указывает **своё снаряжение** или **прокат** (фартук, набор ножей). | Must | [domain-description.md §3, §5.2](../1-elicitation/domain-description.md#52-бронирование) |
| FR-008 | При исчерпании прокатного фонда запись на слот **возможна только «со своим»** снаряжением. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#бронирование-и-экипировка) |
| FR-009 | При бронировании приложение отправляет запрос бэкенду и **отображает результат**: подтверждение или отказ при отсутствии мест (R-004). | Must | [domain-description.md §3, §5.2](../1-elicitation/domain-description.md#52-бронирование) |
| FR-010 | Система не допускает **более одной записи в день** и **более одного участника** в одной записи на клиента. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#вместимость-и-расписание) |
| FR-011 | При заполненной группе приложение отображает **«мест нет»** (лист ожидания не реализуется). | Must | [domain-description.md §3, §6](../1-elicitation/domain-description.md#вне-скоупа--backlog) |
| FR-012 | При записи клиент проходит **обязательный шаг** указания **аллергий** (свободное текстовое поле); при первой записи — обязательно, далее — сохранённое значение с возможностью изменения. | Must | [domain-description.md §3, §5.2](../1-elicitation/domain-description.md#аллергии) |
| FR-013 | Клиент может **изменить аллергии** в существующей записи. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#аллергии) |
| FR-014 | При несовместимости аллергий с меню класса приложение показывает **предупреждение** (запись не блокируется). | Must | [domain-description.md §3](../1-elicitation/domain-description.md#аллергии) |
| FR-015 | Приложение **отображает цену** класса (зависит от программы); оплата выполняется **на месте**. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#бронирование-и-экипировка) |
| FR-016 | Клиент может **просматривать свои брони** со статусами: активна, отменена клиентом, отменена студией, посещена и др. | Must | [domain-description.md §2](../1-elicitation/domain-description.md#2-ключевые-сущности) |
| FR-017 | Клиент может **отменить свою бронь**; при отмене **≥ 3 ч** до начала место в слоте **сразу** освобождается. | Must | [domain-description.md §3, §5.3](../1-elicitation/domain-description.md#53-отмена-брони) |
| FR-018 | При отмене **менее чем за 3 ч** до начала приложение показывает **предупреждение** о закупленных продуктах; отмена **разрешена**. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#отмена-клиентом) |
| FR-019 | При отмене класса **студией** бронь сохраняется со статусом **«Отменён студией»** и **причиной** (R-008). | Must | [domain-description.md §3, §5.6](../1-elicitation/domain-description.md#отмена-студией-форс-мажор-r-008) |
| FR-020 | Клиенту отправляется **push-уведомление** при отмене класса студией с просьбой **перезаписаться на другой класс** (R-008). | Must | [domain-description.md §3](../1-elicitation/domain-description.md#отмена-студией-форс-мажор-r-008) |
| FR-021 | Клиент может **перезаписаться на другой класс** из push-уведомления и с **экрана детали** записи. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#отмена-студией-форс-мажор-r-008) |
| FR-022 | Повторная запись клиента на **тот же отменённый студией слот** запрещена (R-008). | Must | [domain-description.md §3](../1-elicitation/domain-description.md#отмена-студией-форс-мажор-r-008) |
| FR-023 | Клиенту отправляется **push-уведомление** при **переносе** класса (смена времени или шефа). | Must | [domain-description.md §3](../1-elicitation/domain-description.md#отмена-студией-форс-мажор-r-008) |
| FR-024 | После **посещённого** класса клиент может **оценить шефа** звёздами в течение **недели**. | Must | [domain-description.md §3, §5.5](../1-elicitation/domain-description.md#55-оценка-шефа) |
| FR-025 | Клиент может иметь **не более одной оценки на шефа**; оценку можно **изменить** в течение недели или при повторном классе с тем же шефом. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#оценки-шефов) |
| FR-026 | **Рейтинги шефов** отображаются **другим клиентам** при выборе слота. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#оценки-шефов) |
| FR-027 | Приложение отправляет клиенту **push-уведомления**: напоминание за день, за 2 часа, подтверждение записи, отмена (клиент / студия), запрос аллергий. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#уведомления-mvp) |
| FR-028 | Для **постоянных клиентов** отображается **метка** в профиле; при записи применяется **приоритет**. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#постоянные-клиенты) |
| FR-029 | Слоты, программы и данные шефов **поступают из существующего бэкенда** через API; приложение их потребляет, но **не создаёт и не редактирует**. | Must | [domain-description.md §1, §6](../1-elicitation/domain-description.md#6-границы-системы) |
| FR-030 | Клиентское приложение реализует сценарии **только для роли «Клиент»** (R-028). | Must | [domain-description.md §4, §6](../1-elicitation/domain-description.md#4-акторы) |
| FR-031 | Client API обеспечивает операции по контракту: **слоты**, **бронирования**, **профиль**, **шефы/рейтинги**, **аллергии**, **прокат** (R-015). | Must | [domain-description.md §6](../1-elicitation/domain-description.md#в-скоупе-mvp) |
