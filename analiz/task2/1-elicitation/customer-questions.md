## 1. Запись и расписание

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 1.1 | Кто может записаться на класс? Нужна ли регистрация/авторизация, или достаточно **имени и телефона** при первой записи? | Достаточно **имени и телефона** (первая итерация) |
| 1.2 | Может ли один человек записать **нескольких участников** (пара, ребёнок с родителем, друзья)? | **Нет** — в одной записи один человек |
| 1.3 | Сколько **активных записей** может иметь один клиент одновременно? Можно ли записаться на **два класса в один день**? | **Запись на 2 класса в 1 день невозможна** (не более 1 записи в день) |
| 1.4 | Если группа заполнена (8/8 или 12/12) — показываем только «мест нет» или нужен **лист ожидания** при освобождении места? | Только **«мест нет»** (лист ожидания не нужен) |
| 1.5 | **Подтверждение (R-027):** по умолчанию показываем слоты на **7 дней**, более длинный период — через фильтр дат? | **Да**, по R-027 |
| 1.6 | Какие **фильтры** нужны клиенту в первой версии: шеф, время суток, уровень (новичок / опытный), тип кухни / программа? | **Время суток**, **уровень**, **тип кухни** (шеф — не в первой версии) |
| 1.7 | Длительность класса **всегда ~3 часа** или может отличаться? Нужно ли показывать время окончания в приложении? | Пока все классы **~3 часа**; время окончания — по усмотрению UI (не уточнялось) |

## 2. Программы классов и вместимость

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 2.1 | Как система определяет лимит **8 vs 12 мест** — отдельный тип программы в расписании или правило «если программа X (духовка / сложная техника) — лимит 8»? | Настраивается **в зависимости от шефа**, который ведёт занятие |
| 2.2 | Нужна ли **проверка уровня** клиента? Может ли новичок записаться на «продвинутую» программу? | **Нет** — проверка не нужна, запись на усмотрение клиента |
| 2.3 | Что именно выбирает клиент при записи: **свой фартук и ножи** или **прокат** (фартук, набор ножей, оба)? Влияет ли выбор на **цену**? | Выбор своё / прокат (фартук, ножи); на **цену не влияет** |
| 2.4 | Ограничен ли **прокатный фонд** (фартуки / ножи)? Что показывать, если прокат на слот закончился — слот недоступен или запись только «со своим»? | Запись **возможна**, но **только со своим** снаряжением |
| 2.5 | Нужно ли в карточке класса показывать **меню / программу** (что готовим) — полностью, кратко или только название программы? | **Кратко** |
| 2.6 | **Приглашённые шефы** в сезон — для клиента это просто другой шеф в карточке или нужна отдельная пометка («гостевой шеф»)? | Просто **другой шеф** в карточке (без отдельной пометки) |

## 3. Аллергии и ограничения питания

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 3.1 | Сбор **аллергий** — обязателен при каждой записи, только при первой, или опционально? | **Обязательно при первой** записи; далее используется сохранённое значение и/или изменяется |
| 3.2 | Какой **формат**: свободное текстовое поле, чекбоксы (глютен, лактоза, орехи…), оба варианта? | **Свободное текстовое поле** |
| 3.3 | Можно ли **изменить** аллергии в уже созданной записи, или только при новой? | **Да**, можно изменить |
| 3.4 | Если аллергия **несовместима** с меню класса — блокируем запись, показываем предупреждение или только информируем? | **Предупреждение** (запись не блокируется) |
| 3.5 | **Push «заранее спросить про аллергии»** из брифа — это напоминание до класса или обязательный шаг в приложении? | **Обязательный шаг** в приложении |

## 4. Отмена клиентом

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 4.1 | Правило «**за 10 минут до начала**» — жёсткий запрет отмены, штраф, предупреждение или только пометка для администратора? | **Предупреждение** (отмена разрешена; штрафов в MVP нет) |
| 4.2 | Что считается «**заранее**» — 24 часа, 3 часа, 1 час, любое время до порога поздней отмены? | **За 3 часа и более** до начала — ранняя отмена |
| 4.3 | Нужен ли **учёт** частых поздних отмен и неявок? Есть ли последствия для клиента в первой версии? | **Учёт нужен**; **последствий для клиента пока нет** |
| 4.4 | После отмены клиентом место **сразу** становится доступным другим (и для листа ожидания)? | **Да**, место сразу доступно другим |
| 4.5 | При **поздней отмене** продукты уже закуплены — нужно ли показывать клиенту отдельное предупреждение о последствиях? | **Да**, показывать предупреждение |

## 5. Отмена класса студией (форс-мажор)

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 5.1 | **Подтверждение (R-008):** при отмене студией бронь получает статус «**Отменён студией**» с **причиной**, клиенту приходит **push**, повторная запись на **тот же слот запрещена**? | **Да** (R-008): статус + причина + запрет повторной записи на слот; **push** с просьбой **перезаписаться на другой класс** |
| 5.2 | Типовые **причины** отмены для клиента: срыв поставки, поломка оборудования, болезнь шефа — фиксированный список или свободный текст от админки? | **Фиксированный список** + возможность **свободного текста** из админки (непредвиденные обстоятельства) |
| 5.3 | Клиенту предлагать **перезапись на другой класс** из push / экрана детали записи? | **Да** — из **push** и с **экрана детали** записи |
| 5.4 | Если позже подключим **онлайн-оплату** — что делать с деньгами при отмене студией: возврат, перенос, абонемент? | **Перенос**, если клиент согласен; **возврат**, если не хочет переносить |

## 6. Оценки шефов

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 6.1 | Когда клиент может поставить оценку — только после **посещённого** класса? В течение какого **срока**? | **Только после посещённого** класса; срок — **в течение недели** |
| 6.2 | Какой **формат** оценки: звёзды 1–5, текстовый отзыв, анонимно или с именем? | Только **звёзды** (без текстового отзыва) |
| 6.3 | Видят ли **другие клиенты** рейтинги шефов при выборе слота, или оценки только для владельца? | **Да**, клиенты **видят** рейтинги при выборе слота |
| 6.4 | Оценка привязана к **конкретной записи** (один класс — одна оценка) или можно оценить шефа повторно? | **Один пользователь — одна оценка на шефа**; можно **изменить** в течение периода оценивания (неделя) или при **повторном классе** с тем же шефом |

## 8. Оплата и лояльность

| # | Вопрос | Ответ |
| :- | :-- | :-- |
| 8.1 | В MVP **без онлайн-оплаты** клиент записывается бесплатно и платит на месте (наличные / перевод)? Нужно ли **показывать цену** в приложении? | **Запись бесплатно**, оплата **на месте**; **цену в приложении показываем** |

| 8.4 | **Подтверждение:** онлайн-оплата точно **не входит** в первую версию? | **Точно не входит** в первую версию |
