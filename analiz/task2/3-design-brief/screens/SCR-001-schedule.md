## Обзор

Главный экран приложения «Шеф-стол». Показывает список доступных классов за период, по умолчанию — ближайшие 7 дней (R-027), позволяет открыть фильтр дат и фильтр классов, а также перейти в детали выбранного класса.

### User Story

> Как клиент, я хочу видеть ближайшие кулинарные классы и быстро отфильтровать расписание, чтобы выбрать подходящий класс и перейти к записи.

**Не в MVP:** лист ожидания, фильтр по шефу, iOS.

## Используемые запросы

### listSlots

**Метод:** GET  
**Путь:** `/slots`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `listSlots`

**Query-параметры экрана:**

| Параметр | Обязательность | Источник |
| :-- | :-- | :-- |
| `dateFrom` | Нет | SCR-002 |
| `dateTo` | Нет | SCR-002 |
| `timeOfDay` | Нет | SCR-003 |
| `level` | Нет | SCR-003 |
| `cuisineTypeIds` | Нет | SCR-003 |
| `limit` | Нет | Клиент (`50`) |
| `offset` | Нет | Клиент (`0`) |

**Обработка ответа:**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 200 + `items.length > 0` | Content: карточки, отсортированные по `startsAt` и сгруппированные по дням |
| 200 + `items.length = 0` + дефолтный период/фильтры | Empty: «Пока нет доступных классов» |
| 200 + `items.length = 0` + активные фильтры | Empty: «Ничего не найдено», CTA «Сбросить фильтры» |
| 400 | Error: «Проверьте параметры фильтра» + «Повторить» |
| 500 / timeout | Error: «Не удалось загрузить расписание» + «Повторить» |

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Заголовок «Расписание» | Название вкладки | Константа UI | Непрокручиваемый header |
| Чип «Период» | Текущее окно дат | Локальное состояние + SCR-002 | По умолчанию: «7 дней»; при кастомном диапазоне: `D–D MMM` |
| Чип «Фильтры» | Открытие фильтров классов | Локальное состояние + SCR-003 | Badge = число активных категорий (0..3) |
| Заголовок группы дня | День и дата | `items[].startsAt` | Формат: «Сегодня/Завтра/...», sticky |
| Время начала | Время конкретного класса | `items[].startsAt` | Формат `HH:mm`, локаль ru-RU |
| Название программы | Краткое имя класса | `items[].program.name` | Максимум 2 строки, затем ellipsis |
| Имя шефа | Ведущий класса | `items[].chef.fullName` | Обязательно для отображения карточки |
| Рейтинг шефа | Публичный рейтинг | `items[].chef.avgRating`, `items[].chef.ratingCount` | При `avgRating = null` блок скрывается |
| Свободные места | Наличие мест | `items[].freeSpots`, `items[].capacity` | При `freeSpots = 0` текст «Мест нет», лист ожидания не показывается |
| Цена | Цена класса | `items[].price` | Формат: `N ₽` |
| Карточка класса | CTA переход в детали | `items[].id` | Тап по всей карточке ведёт в SCR-004 с `slotId` |

**Терминология:** использовать «шеф», «класс», «тип кухни». Термин «тренировка/инструктор» не использовать.

## Состояния экрана

| Состояние | Условие | Отображение |
| :-- | :-- | :-- |
| Loading | Первый запрос `listSlots` в процессе | Skeleton списка под группами дней |
| Content | 200 + `items.length > 0` | Сгруппированный список карточек |
| Empty | 200 + `items.length = 0` | «Пока нет доступных классов» / «Ничего не найдено» с CTA |
| Error | 4xx/5xx/timeout | Сообщение об ошибке + «Повторить» |
| Offline | Нет сети на SCR-001 | Показывается как Error (кэш не используется в MVP) |
| Submitting | Применение фильтров/refresh в процессе | Локальный loader в контролах, контент не блокируется полностью |

> Для pull-to-refresh используется под-состояние Refreshing поверх Content согласно LOGIC-008.

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** пользователь открыл SCR-001 без параметров, **Когда** выполняется `listSlots`, **Тогда** отправляется запрос без `dateFrom/dateTo` и показываются ближайшие 7 дней. |
| AC-002 | **Дано** `listSlots` вернул непустой список, **Когда** экран отрисован, **Тогда** карточки отсортированы по `startsAt` и сгруппированы по дням. |
| AC-003 | **Дано** `listSlots` вернул пустой список при дефолтных фильтрах, **Когда** загрузка завершилась, **Тогда** отображается текст «Пока нет доступных классов». |
| AC-004 | **Дано** активны фильтры и `listSlots` вернул пусто, **Когда** экран показывает Empty, **Тогда** отображается «Ничего не найдено» и CTA «Сбросить фильтры». |
| AC-005 | **Дано** в слоте `freeSpots = 0`, **Когда** карточка отображается, **Тогда** пользователь видит «Мест нет» и может открыть детали класса (без листа ожидания). |
| AC-006 | **Дано** в ответе есть `chef.avgRating`, **Когда** карточка класса отображается, **Тогда** рейтинг шефа показан в формате `★ X.X`. |
| AC-007 | **Дано** пользователь потянул список вниз, **Когда** запрос refresh завершился ошибкой, **Тогда** ранее загруженный список остаётся на экране и показывается сообщение об ошибке обновления. |
| AC-008 | **Дано** пользователь нажал «Фильтры», выбрал 2 категории и применил, **Когда** возврат на SCR-001 завершён, **Тогда** chip «Фильтры» показывает badge `2` и список перезагружен с новыми параметрами. |
