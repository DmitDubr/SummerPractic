## Обзор

Экран показывает полную информацию о конкретной записи клиента и доступные действия по статусу. Экран открывается как из списка SCR-008, так и через deep link из push-уведомлений (отмена студией, перенос, напоминания), включая отображение причины `CANCELLED_BY_STUDIO` и данных о переносе.

### User Story

> Как клиент, я хочу открыть детальную карточку записи, чтобы понять актуальный статус, причину изменений и выполнить действие (отмена, оценка шефа, изменение аллергий).

**Не в MVP:** лист ожидания при `NO_SPOTS` (FR-011) — только возврат к расписанию; фильтр по шефу; iOS.

## Используемые запросы

### getBooking

**Метод:** GET  
**Путь:** `/bookings/{bookingId}`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `getBooking`

**Обработка ответа:**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 200 + data | Content: заполнение всех блоков, CTA по статусу |
| 200 + `status=CANCELLED_BY_STUDIO` | Показ warning-блока с `cancellationReason` |
| 200 + `rescheduleInfo != null` | Показ info-блока переноса с новыми данными |
| 404 | Экран «Запись не найдена» + возврат на SCR-008 |
| 401 | Завершение сессии и выход в поток повторной идентификации |
| 5xx / timeout | Error + «Повторить» |
| offline + кэш | Offline state с read-only деталями |

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Бейдж статуса | Крупный индикатор состояния записи | `status` | Карта значений как на SCR-008 |
| Блок класса | Программа, шеф с рейтингом, адрес, время | `slot.*`, `studio.*` | Читаемое представление для открытия из push |
| Блок цены | Стоимость и подпись «оплата на месте» | `totalPrice` | Не пересчитывать на клиенте |
| Блок аллергий | Текущие аллергии клиента в записи | `allergies.allergiesText` | CTA «Изменить» только при `status=ACTIVE` |
| Warning отмены студией | Причина отмены | `cancellationReason` | Показывать только для `CANCELLED_BY_STUDIO` |
| Info переноса | Детали изменения слота | `rescheduleInfo` | Показывать при наличии данных о переносе |
| CTA «Отменить запись» | Переход в SCR-010 | `canCancel` | Только для активной, не начавшейся записи |
| CTA «Оценить шефа» | Переход в SCR-011 | `status`, `chefRating`, дата окончания | Только для `ATTENDED` в окне 7 дней |
| CTA «Записаться на другой класс» | Возврат в расписание | `status` | Только для `CANCELLED_BY_STUDIO` |

**Терминология:** **шеф**, **программа класса**, **тип кухни**; прокат (фартук, ножи) **не влияет на цену** (FR-015).

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** вход на экран по `bookingId`, **Когда** `getBooking=200`, **Тогда** отображаются дата/время, программа, шеф, цена, статус и персональные данные записи. |
| AC-002 | **Дано** запись со статусом `CANCELLED_BY_STUDIO`, **Когда** открыт SCR-009, **Тогда** показывается warning-блок с текстом `cancellationReason` и CTA «Записаться на другой класс». |
| AC-003 | **Дано** запись содержит `rescheduleInfo`, **Когда** экран загружен, **Тогда** отображается отдельный инфо-блок переноса с изменившимися полями. |
| AC-004 | **Дано** запись `ACTIVE` и `canCancel=true`, **Когда** пользователь нажимает «Отменить запись», **Тогда** выполняется переход на SCR-010 с параметрами отмены. |
| AC-005 | **Дано** запись `ATTENDED`, `chefRating=null` и с момента окончания прошло не более 7 дней, **Когда** открыт экран, **Тогда** доступен CTA «Оценить шефа». |
| AC-006 | **Дано** запись `ACTIVE`, **Когда** пользователь нажимает «Изменить аллергии», **Тогда** открывается SCR-012 и после сохранения на SCR-009 виден обновленный текст аллергий. |
| AC-007 | **Дано** устройство offline и есть кэш детали, **Когда** открыт SCR-009, **Тогда** показываются кэшированные данные, а действия отмены/оценки/редактирования отключены. |
