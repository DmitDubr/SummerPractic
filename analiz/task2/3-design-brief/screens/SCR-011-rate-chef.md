## Обзор

Bottom sheet для оценки шефа после посещённого класса. Экран поддерживает только выбор звёзд 1-5 и отправку `createOrUpdateChefRating`; текстовый отзыв, теги и другие поля в MVP не используются.

Оценка доступна только при `Booking.status = ATTENDED` и только в пределах 7 дней после окончания класса. Модель оценки — upsert по паре клиент-шеф: повторная отправка обновляет существующую оценку.

### User Story

> Как клиент, я хочу быстро поставить оценку шефу после класса, чтобы повлиять на его публичный рейтинг без длинного отзыва.

**Не в MVP:** текстовые отзывы; отдельный полноэкранный экран оценки; оценка до посещения; оценка после 7 дней.

## Используемые запросы

### createOrUpdateChefRating

**Метод:** POST  
**Путь:** `/ratings`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `createOrUpdateChefRating`

**Обработка ответа:**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 201 + data | Новая оценка создана, sheet закрывается, toast «Спасибо за оценку!» |
| 200 + data | Оценка обновлена (upsert), sheet закрывается, на SCR-009 показывается обновлённое значение |
| 400 `VALIDATION_ERROR` | Inline ошибка, кнопка снова активна |
| 403 `BOOKING_NOT_ATTENDED` | Sheet закрывается, на SCR-009 сообщение «Оценить можно после посещения» |
| 403 `RATING_PERIOD_EXPIRED` | Sheet закрывается, на SCR-009 сообщение «Срок оценки истёк» |
| 401 / 5xx / timeout | Error состояние с retry, выбранные звёзды сохраняются локально |

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Drag handle | Индикатор bottom sheet | UI | Закрытие свайпом вниз |
| Аватар шефа | Круглое фото/placeholder | `chefAvatarUrl` | Placeholder при `null` |
| Имя шефа | Заголовок карточки | `chefName` | Обязательное поле, без сокращений «инструктор» |
| Блок звёзд | 5 tappable иконок | Локальный state | Допустимы только целые 1-5 |
| CTA «Отправить» | Основное действие | Локальный state | Disabled при `stars=null`; в Submitting блокирует повторный тап |
| Inline сообщение | Текст ошибки/успеха | API response | Успех: краткое «Спасибо за оценку!» |

**Терминология:** **шеф**, **программа класса**, **тип кухни**; прокат (фартук, ножи) **не влияет на цену** (FR-015).

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** бронь со статусом `ATTENDED` и с момента класса прошло не более 7 дней, **Когда** клиент открывает SCR-011, **Тогда** экран доступен и позволяет выбрать 1-5 звёзд. |
| AC-002 | **Дано** клиент не выбрал ни одной звезды, **Когда** открыт SCR-011, **Тогда** кнопка «Отправить» неактивна. |
| AC-003 | **Дано** клиент отправляет оценку с валидным `bookingId`/`chefId`, **Когда** API возвращает 201, **Тогда** sheet закрывается и на SCR-009 отображается сохранённая оценка. |
| AC-004 | **Дано** ранее уже была оценка этого шефа текущим клиентом, **Когда** выполняется повторная отправка в окне 7 дней, **Тогда** вызывается `createOrUpdateChefRating` и при 200 обновляется существующая оценка (upsert). |
| AC-005 | **Дано** прошло более 7 дней после класса, **Когда** клиент пытается открыть SCR-011, **Тогда** оценка недоступна и показывается сообщение «Оценить можно в течение недели после класса». |
| AC-006 | **Дано** статус брони не `ATTENDED`, **Когда** клиент открывает экран оценки по CTA или deep link, **Тогда** отправка блокируется и показывается сообщение «Оценить можно после посещения класса». |
| AC-007 | **Дано** пользовательский интерфейс SCR-011, **Когда** пользователь взаимодействует с формой, **Тогда** доступны только звёзды 1-5 без поля текстового отзыва, тегов и иных дополнительных полей. |
| AC-008 | **Дано** экран в состоянии Content, **Когда** пользователь тапает по звёздам, **Тогда** каждая звезда имеет tappable-зону не менее 44dp и визуально различаются выбранное/невыбранное состояние. |
