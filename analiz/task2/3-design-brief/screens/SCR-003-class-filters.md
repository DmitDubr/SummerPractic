## Обзор

Bottom sheet фильтрации расписания на SCR-001 по трём критериям MVP: **тип кухни**, **время суток**, **уровень**. Фильтр **по шефу не реализуется**. При «Применить» sheet закрывается и передаёт параметры в SCR-001 для `listSlots`.

### User Story

> Как клиент, я хочу отфильтровать классы по типу кухни, времени и уровню, чтобы быстрее найти подходящее занятие.

**Не в MVP:** фильтр по шефу, iOS.

## Используемые запросы

### listCuisineTypes

**Метод:** GET  
**Путь:** `/cuisine-types`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `listCuisineTypes`

**Обработка ответа:**

| HTTP | UI-реакция |
| :-- | :-- |
| 200 + items | Чекбоксы/чипы типов кухни (`isActive = true`) |
| 500 / timeout | Inline «Не удалось загрузить типы кухни» + «Повторить» в sheet |
| 200 + empty | Секция типов кухни скрыта или текст «Справочник недоступен» |

### listSlots (косвенно через SCR-001)

**Query после «Применить»:**

| Параметр | Источник |
| :-- | :-- |
| `cuisineTypeIds` | Выбранные типы (OR внутри группы) |
| `timeOfDay` | Сегмент времени суток |
| `level` | Выбранный уровень |

Период (`dateFrom`/`dateTo`) **не сбрасывается** при смене фильтров классов (LOGIC-005).

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Drag handle | Свайп закрытия | UI | Без применения |
| Секция «Тип кухни» | Multi-select чипы/чекбоксы | `listCuisineTypes.items[]` | OR внутри группы |
| Секция «Время суток» | Single-select | Локально | `morning` 06–11:59, `afternoon` 12–16:59, `evening` 17–23:59 |
| Секция «Уровень» | Single-select | Локально | `BEGINNER` / `INTERMEDIATE` / `ADVANCED` |
| «Сбросить» | Сброс всех фильтров sheet | Локально | Не сбрасывает период SCR-002 |
| «Применить» | Подтверждение | Локально | Закрывает sheet → reload SCR-001 |

**Badge на SCR-001:** число заполненных категорий (0–3), не число выбранных типов кухни.

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** открыт SCR-003, **Когда** sheet открывается, **Тогда** вызывается `listCuisineTypes` и отображаются активные типы кухни. |
| AC-002 | **Дано** выбраны 2 типа кухни и «Вечер», **Когда** «Применить», **Тогда** SCR-001 перезагружает `listSlots` с `cuisineTypeIds` и `timeOfDay=evening`, badge = 2. |
| AC-003 | **Дано** выбран только уровень «Начинающий», **Когда** «Применить», **Тогда** query содержит `level=BEGINNER`, badge = 1. |
| AC-004 | **Дано** активны фильтры, **Когда** «Сбросить», **Тогда** query без `cuisineTypeIds`/`timeOfDay`/`level`, badge = 0. |
| AC-005 | **Дано** `listSlots` вернул пусто при фильтрах, **Когда** SCR-001, **Тогда** «Ничего не найдено» + «Сбросить фильтры». |
| AC-006 | **Дано** изменён период на SCR-002, **Когда** применены фильтры SCR-003, **Тогда** оба параметра переданы в один `listSlots`. |
| AC-007 | **Дано** ошибка загрузки типов кухни, **Когда** retry в sheet, **Тогда** повторный `listCuisineTypes` без закрытия sheet. |
