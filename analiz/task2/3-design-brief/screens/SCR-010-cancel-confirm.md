## Обзор

Bottom sheet запрашивает явное подтверждение отмены активной брони на кулинарный класс. При **поздней отмене** (менее чем за **3 часа** до начала) показывается жёлтое предупреждение о закупленных продуктах **без блокировки** действия в MVP. После успешной отмены — toast «Запись отменена» и возврат к списку записей с обновлённым статусом.

### User Story

> Как клиент, я хочу подтвердить отмену записи с понятным предупреждением о последствиях, чтобы осознанно отменить бронь и освободить место.

**Не в MVP:** лист ожидания при `NO_SPOTS` (FR-011) — только возврат к расписанию; фильтр по шефу; iOS; штрафы и блокировка поздней отмены.

## Применяемые логики

| Логика | Элемент / триггер | Описание |
| :-- | :-- | :-- |
| [LOGIC-004_Отмена-правило-3-часов](../../5-mobile-app-spec/09_Логики/LOGIC-004_Отмена-правило-3-часов.md) | Блок предупреждения; видимость кнопки на SCR-009 | Ранняя отмена ≥ 3 ч — без warning; поздняя < 3 ч — warning, отмена разрешена; штрафов нет |

**Формула для разработки и QA:**

```
minutesUntilStart = (slot.startsAt - now) в минутах
canCancel = status == ACTIVE AND minutesUntilStart > 0
isEarlyCancel = canCancel AND minutesUntilStart >= 180
isLateCancel = canCancel AND minutesUntilStart < 180
showWarning = isLateCancel
```

Если класс уже начался — кнопка «Отменить» на SCR-009 не показывается (вне скоупа SCR-010).

## Используемые запросы

### cancelBooking

**Метод:** POST  
**Путь:** `/bookings/{bookingId}/cancel`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `cancelBooking`

**Обработка ответа:**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 200 + `status: CANCELLED_BY_CLIENT` | Закрыть sheet → toast «Запись отменена» → SCR-008; место освобождается на бэкенде |
| 200 + `isLateCancellation: true` | То же; поздняя отмена учитывается для аналитики (FR-018) |
| 401 | «Сессия истекла» → SCR-001 или повторный вход через запись |
| 409 `BOOKING_ALREADY_CANCELLED` | «Запись уже отменена» → закрыть sheet → обновить SCR-009/008 |
| 409 / 403 прочие | «Не удалось отменить запись» + «Повторить» / «Закрыть» |
| 5xx / timeout | «Не удалось отменить запись» + «Повторить»; sheet остаётся открытым |
| Offline | Запрос не отправляется; на SCR-009 кнопка «Отменить» disabled |

**Сценарии и переходы:**

1. **Ранняя отмена (UC-004, ≥ 3 ч):** SCR-009 → «Отменить запись» → SCR-010 (без предупреждения) → `cancelBooking` → место **сразу** освобождается (FR-017) → toast → SCR-008.
2. **Поздняя отмена (< 3 ч, MVP):** SCR-010 с жёлтым предупреждением → клиент подтверждает (штрафов нет, FR-018) → `cancelBooking` → SCR-008.
3. **Отказ от отмены:** «Оставить запись» или свайп / overlay → sheet закрыт, SCR-009 без изменений.
4. **Двойной тап:** debounce на confirm; повторный запрос не отправлять.

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Заголовок | «Отменить запись?» | — | Всегда виден |
| Краткое напоминание | Дата, время, программа класса | `slot.startsAt`, `slot.program` | Формат «Ср, 9 июл · 11:00» |
| Блок предупреждения | Жёлтый фон, иконка ⚠ | LOGIC-004 `showWarning` | **Только** при `isLateCancel`; не error, без упоминания штрафов |
| Текст предупреждения | «До начала класса осталось меньше 3 часов. Продукты для занятия уже закуплены.» | LOGIC-004 | Нейтрально-информативный; отмена **не блокируется** |
| Кнопка «Оставить запись» | Primary / filled — безопасное действие по умолчанию | — | Закрывает sheet без API |
| Кнопка «Отменить запись» | Destructive (красный текст или outline) | — | Вызывает `cancelBooking`; loader во время запроса |
| Индикатор загрузки | На кнопке «Отменить запись» | Состояние Submitting | Обе кнопки disabled; sheet **не закрывается** до ответа |
| Toast (после успеха) | «Запись отменена» | — | 2–3 сек, на SCR-008, не блокирует список |

**Критерии дизайна:**

- **Безопасное действие по умолчанию** — «Оставить запись» (primary, верхняя или левая кнопка).
- Жёлтый блок — не error; читаемый контраст; зарезервировать место под блок, чтобы layout не прыгал (или плавная анимация).
- **Не пугать** формулировками про штрафы — в MVP их нет.
- Согласовать стиль sheet с SCR-007 (ошибка записи).
- Модал доступен **только онлайн**; в офлайне кнопка «Отменить» на SCR-009 disabled.

**Терминология:** **шеф**, **программа класса**, **тип кухни**; прокат (фартук, ножи) **не влияет на цену** (FR-015).

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** до начала класса ≥ 3 ч и статус `ACTIVE`, **Когда** открыт SCR-010, **Тогда** жёлтый блок предупреждения **не** показывается. |
| AC-002 | **Дано** до начала класса < 3 ч и > 0, **Когда** открыт SCR-010, **Тогда** показывается warning-блок о закупленных продуктах и кнопка «Отменить запись» **активна** (FR-018). |
| AC-003 | **Дано** пользователь на SCR-010 с ранней отменой, **Когда** нажимает «Отменить запись» и `cancelBooking` вернул 200, **Тогда** sheet закрывается, показывается toast «Запись отменена», выполняется переход на SCR-008. |
| AC-004 | **Дано** пользователь на SCR-010, **Когда** нажимает «Оставить запись» или закрывает sheet свайпом, **Тогда** sheet закрывается без API-запроса и SCR-009 остаётся без изменений. |
| AC-005 | **Дано** `cancelBooking` в процессе, **Когда** ожидается ответ, **Тогда** loader на confirm-кнопке, обе кнопки disabled, sheet не закрывается. |
| AC-006 | **Дано** `cancelBooking` вернул 409 «уже отменена», **Когда** отображается ошибка, **Тогда** sheet закрывается и список/деталь записи обновляются. |
| AC-007 | **Дано** нет сети, **Когда** пользователь на SCR-009, **Тогда** кнопка «Отменить запись» disabled и SCR-010 недоступен. |
| AC-008 | **Дано** класс уже начался (`slot.startsAt <= now`), **Когда** открыт SCR-009, **Тогда** кнопка «Отменить запись» **скрыта** (LOGIC-004). |
