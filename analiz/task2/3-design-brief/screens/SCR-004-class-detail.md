## Обзор

Экран показывает полную информацию о выбранном кулинарном классе: время, длительность, программу, шефа, рейтинг, свободные места и цену. Это точка входа в запись из расписания.

### User Story

> Как клиент, я хочу увидеть деталь класса, чтобы понять подходит ли мне занятие и перейти к записи.

**Не в MVP:** лист ожидания при `NO_SPOTS` (FR-011) — только возврат к расписанию; фильтр по шефу; iOS.

## Используемые запросы

### getSlot

**Метод:** GET  
**Путь:** `/slots/{slotId}`  
**Спецификация:** [../../api/openapi.yaml](../../api/openapi.yaml) → `getSlot`

**Обработка ответа:**

| HTTP / код | UI-реакция |
| :-- | :-- |
| 200 + data (`freeSpots > 0`) | Content + активный CTA «Записаться» |
| 200 + data (`freeSpots = 0`) | Content + бейдж «Мест нет», CTA скрыт |
| 200 + data (`rentalAvailability.rentalFullyExhausted = true`) | Инфо «Прокат закончился, можно записаться со своим» |
| 404 | Error state + «К расписанию» |
| 5xx / timeout | Error state + «Повторить» |

## Элементы экрана

| Элемент | Описание | Источник данных | Валидация / поведение |
| :-- | :-- | :-- | :-- |
| Кнопка «Назад» | Возврат к расписанию | Навигация | Возврат без потери контекста списка |
| Дата и время | Старт класса | `slot.startsAt` | Формат «Сб, 5 июля · 11:00» |
| Длительность | Продолжительность класса | `slot.durationMinutes` | Отображать как `~3 ч` |
| Бейдж типа кухни | Тип кухни/программа | `slot.program.cuisineTypeName` | Всегда виден при 200 |
| Счётчик мест | Свободно из вместимости | `slot.freeSpots`, `slot.capacity` | Формат `X / Y` |
| Карточка шефа | Имя и рейтинг | `slot.chef.*` | Если `ratingCount = 0` → «Пока нет оценок» |
| Описание программы | Краткое меню | `slot.programInfo.menuSummary` | 1-3 строки, остальное обрезать |
| Блок цены | Стоимость класса | `slot.programInfo.price` / `slot.price` | Цена зависит только от программы |
| Инфо по прокату | Доступность проката | `slot.rentalAvailability` | При полном исчерпании показать инфо про «со своим» |
| CTA «Записаться» | Переход к форме | Локально от `slot` | Доступен только если `freeSpots > 0` и `status = OPEN` |
| Бейдж «Мест нет» | Слот заполнен | Локально от `slot` | Показывается при `freeSpots = 0`; waitlist CTA отсутствует |

**Терминология:** **шеф**, **программа класса**, **тип кухни**; прокат (фартук, ножи) **не влияет на цену** (FR-015).

## Состояния экрана

| Состояние | Условие | Отображение |
| :-- | :-- | :-- |
| Loading | `getSlot` в процессе | Skeleton блоков + placeholder CTA |
| Content | 200 + data | Полный контент экрана |
| Empty | Не используется | Для `SCR-004` отсутствует отдельный Empty |
| Error | 404 / 5xx / timeout | Баннер + кнопки «Повторить» и «К расписанию» |
| Offline | Нет сети | Сообщение «Нет подключения к интернету» + retry |
| Full | `freeSpots = 0` | Бейдж «Мест нет», CTA записи скрыт |
| Rental exhausted | `rentalFullyExhausted = true` | Инфо «Прокат закончился, можно записаться со своим» |

## Критерии приёмки

| ID | Критерий |
| :-- | :-- |
| AC-001 | **Дано** пользователь открыл класс из SCR-001, **Когда** `getSlot` вернул 200, **Тогда** показываются дата, программа, шеф, места и цена. |
| AC-002 | **Дано** `freeSpots > 0` и `status = OPEN`, **Когда** экран в Content, **Тогда** CTA «Записаться» активен и ведёт в SCR-005 с `slotId`. |
| AC-003 | **Дано** `freeSpots = 0`, **Когда** экран загружен, **Тогда** показывается «Мест нет», CTA записи и CTA waitlist отсутствуют. |
| AC-004 | **Дано** `rentalAvailability.rentalFullyExhausted = true`, **Когда** экран загружен, **Тогда** показан информационный текст про запись «со своим». |
| AC-005 | **Дано** `getSlot` вернул 404, **Когда** открыт экран, **Тогда** отображается Error с возможностью перейти к расписанию. |
| AC-006 | **Дано** нет сети, **Когда** открывается экран, **Тогда** показывается офлайн-ошибка и кнопка «Повторить». |
| AC-007 | **Дано** у шефа `ratingCount = 0`, **Когда** отображается карточка шефа, **Тогда** показывается текст «Пока нет оценок». |
