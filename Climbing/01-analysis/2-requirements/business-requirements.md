# Бизнес-требования — скалодром «Вертикаль»

> Источник: [domain-description.md](../1-elicitation/domain-description.md).
> Приоритет: **Must** — обязательно в MVP; **Should** — важно, но может следовать за MVP; **Could** — желательно позже.

| ID | Формулировка | Приоритет | Источник |
| :- | :-- | :--: | :-- |
| BR-001 | Скалодром должен предоставить клиентам **самообслуживание записи** на групповые тренировки через мобильное приложение, чтобы снизить нагрузку на ручную обработку заявок (Telegram, тетрадь). | Must | [domain-description.md §1, §7](../1-elicitation/domain-description.md#1-предметная-область) |
| BR-002 | Система записи должна **исключить двойное бронирование** одного места в группе, чтобы клиенты не приходили на уже занятую тренировку. | Must | [domain-description.md §3, §7](../1-elicitation/domain-description.md#3-бизнес-правила) |
| BR-003 | Клиент должен **самостоятельно видеть свободные места** и записываться на тренировку без участия владельца в каждой операции. | Must | [domain-description.md §3](../1-elicitation/domain-description.md#3-бизнес-правила) |
| BR-004 | Операционное управление расписанием, инструкторами и профилактикой зон остаётся в **существующей инфраструктуре**; новое решение дополняет её клиентским каналом, не заменяя админку. | Must | [domain-description.md §1, §6](../1-elicitation/domain-description.md#6-границы-системы) |
| BR-005 | При отмене тренировки скалодромом (в т. ч. из-за профилактики) клиент должен **своевременно получать информацию** о статусе брони и причине отмены. | Must | [domain-description.md §3, §5.6](../1-elicitation/domain-description.md#36-профилактика-зоны) |
| BR-006 | Скалодром должен получать **обратную связь по качеству работы инструкторов** от клиентов после тренировок для оценки нагрузки и удовлетворённости. | Should | [domain-description.md §3, §5.5](../1-elicitation/domain-description.md#55-оценка-инструктора) |
| BR-007 | Система должна способствовать **снижению неявок** на записанные тренировки за счёт напоминаний клиентам. | Should | [domain-description.md §5.4](../1-elicitation/domain-description.md#54-напоминания) |
| BR-008 | Групповые тренировки должны соблюдать **лимиты вместимости**: до 16 человек в группе; на новичковых тренировках — не более 8 (требования безопасности). | Must | [domain-description.md §3](../1-elicitation/domain-description.md#3-бизнес-правила) |
| BR-009 | Клиент должен иметь возможность выбрать **своё снаряжение или прокат** при записи, чтобы скалодром заранее планировал выдачу инвентаря. | Should | [domain-description.md §2, §5.2](../1-elicitation/domain-description.md#52-бронирование) |
| BR-010 | При отмене брони клиентом **заблаговременно** освобождённое место должно становиться доступным для других клиентов. | Must | [domain-description.md §3, §5.3](../1-elicitation/domain-description.md#53-отмена-брони) |
