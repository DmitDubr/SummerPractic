# SCR-001 — Расписание

| Поле | Значение |
| :-- | :-- |
| ID | SCR-001 |
| Тип | Экран (вкладка нижней навигации) |
| Приоритет | Must |
| Связь | UC-001; FR-001, FR-002, FR-003, FR-004; US-001–US-004; Q 1.5, 1.6, 5.3, 7.1; R-027 |

## Назначение

Главный экран приложения «Вертикаль»: клиент просматривает доступные групповые тренировки, фильтрует их и выбирает слот для записи. По умолчанию показываются слоты на **ближайшие 7 дней** (R-027). Экран — основная точка входа в сценарий записи (UC-001 → SCR-004).

## Точки входа

- Запуск приложения (вкладка «Расписание» — первая/дефолтная).
- Нижняя навигация: вкладка **«Расписание»**.
- Deep link из push-уведомления (отмена скалодромом, освободилось место) → SCR-001 с актуальным списком.
- Возврат из SCR-004, SCR-002, SCR-003 (без сброса позиции скролла, если возможно).
- SCR-009: действие «Перезаписаться на другой слот» после отмены скалодромом.

## Точки выхода

| Действие | Переход |
| :-- | :-- |
| Тап по карточке слота | SCR-004 (деталь слота) |
| Тап по чипу «Период» / «Даты» | SCR-002 (bottom sheet) |
| Тап по чипу «Фильтры» | SCR-003 (bottom sheet) |
| Нижняя навигация «Мои записи» | SCR-008 |

## Структура экрана

```
┌─────────────────────────────────────┐
│  Расписание                    [?]  │  ← заголовок (без поиска в MVP)
├─────────────────────────────────────┤
│  [Период ▾]  [Фильтры •2]           │  ← горизонтальная полоса чипов-фильтров
│  [12–18 июл]  (активный период)     │  ← чип периода, если ≠ дефолт 7 дней
├─────────────────────────────────────┤
│  ▼ Сегодня, 3 июля                  │  ← заголовок группы дня (sticky)
│  ┌─────────────────────────────┐    │
│  │ 18:00 · Болдеринг для новичков│  │  ← карточка слота
│  │ Анна К.  ★ 4.8              │    │
│  │ 3 места · 1 200 ₽           │    │
│  └─────────────────────────────┘    │
│  ┌─────────────────────────────┐    │
│  │ ...                         │    │
│  └─────────────────────────────┘    │
│  ▼ Завтра, 4 июля                   │
│  ...                                │
├─────────────────────────────────────┤
│  [Расписание]    [Мои записи]       │  ← нижняя навигация
└─────────────────────────────────────┘
```

Вертикальный скролл: список слотов, сгруппированный по календарным дням. Заголовки дней — **sticky** при прокрутке внутри группы.

## Элементы UI

### Шапка

- Заголовок экрана: **«Расписание»**.
- Логотип/название «Вертикаль» — опционально в шапке или только на splash; на экране достаточно заголовка.

### Полоса фильтров (chips)

- **Чип «Период»** — открывает SCR-002. В дефолтном состоянии (7 дней) подпись: «7 дней» или «Ближайшие 7 дней». При выбранном кастомном диапазоне — компактный чип с датами, напр. «12–18 июл» (см. SCR-002).
- **Чип «Фильтры»** — открывает SCR-003. При активных фильтрах — **badge** с числом активных критериев (1–3). Иконка воронки/фильтра.
- Чипы расположены в одной горизонтальной строке с горизонтальным скроллом при нехватке места.

### Заголовок группы дня

- Формат: **«Сегодня, 3 июля»**, **«Завтра, 4 июля»**, далее **«Пятница, 5 июля»** (день недели + дата).
- Если в дне нет слотов после фильтрации — группа **не отображается** (не показывать пустые дни).

### Карточка слота

Обязательные поля (FR-003):

| Элемент | Описание |
| :-- | :-- |
| Время начала | Крупно, напр. **18:00** (локальное время скалодрома) |
| Зона/формат | Вторичная строка, напр. «Болдеринг для новичков», «Трассы с верёвкой» |
| Инструктор | Имя (и при необходимости фото-аватар, круглый, 32–40 dp) |
| Рейтинг инструктора | Звёзды или число + звезда, напр. **★ 4.8** (публичный рейтинг, Q 5.3) |
| Свободные места | «**N мест**» или «**Мест нет**» — визуально отличается (при 0 мест карточка остаётся тапабельной → SCR-004 / лист ожидания) |

Дополнительно:

| Элемент | Описание |
| :-- | :-- |
| Цена | Показывать, **если API вернул** `price` (Q 7.1). Формат: «1 200 ₽». Если цены нет — строку не рендерить, без заглушки «—» |
| Состояние заполненности | При 0 свободных мест — приглушённый стиль карточки + бейдж «Мест нет»; тап ведёт на SCR-004 (лист ожидания) |

Вся карточка — единая тап-зона (min height ~72 dp, удобно для большого пальца).

### Pull-to-refresh

- Жест «потянуть вниз» в верхней части списка → повторный запрос слотов.
- Индикатор обновления в системном стиле платформы.

## Состояния (loading / empty / error / offline)

### Loading (первичная загрузка)

- **Skeleton**: 3–5 placeholder-карточек под заголовком «Сегодня» + 2–3 под «Завтра». Пульсирующие прямоугольники повторяют структуру карточки (время, 2 строки текста, бейдж мест).
- Шапка и чипы фильтров видны сразу; skeleton только в области списка.
- Длительность: до ответа API; при >3 с — допустим полноэкранный индикатор поверх skeleton (на усмотрение, но skeleton — обязателен).

### Empty

- Условие: API вернул пустой список слотов на выбранный период (с учётом фильтров).
- Текст **дословно**: «**Пока нет доступных тренировок**» (FR-004).
- Подзаголовок (рекомендация): «Попробуйте изменить период или сбросить фильтры».
- CTA: кнопка **«Сбросить фильтры»** — если активны фильтры или нестандартный период; иначе скрыть или заменить на «Обновить».
- Иллюстрация: нейтральная (скалодром/календарь), не обязательна в wireframe.

### Error

- Условие: сетевой сбой или ошибка сервера при загрузке.
- Иконка + текст: «Не удалось загрузить расписание».
- Кнопка **«Повторить»** → повторный запрос.
- Если есть **кэшированные** данные предыдущей успешной загрузки — показать их с баннером «Данные могут быть устаревшими» + «Обновить». (Для SCR-001 кэш расписания не обязателен в MVP; приоритет — retry.)

### Offline

- Нет сети при открытии экрана: состояние как **Error**, текст: «Нет подключения к интернету. Проверьте сеть и попробуйте снова».
- Офлайн-кэш расписания в MVP **не требуется** (офлайн — для «Мои записи», Q 9.2). Не показывать пустой список без пояснения.

### Обновление (pull-to-refresh)

- Во время refresh список остаётся видимым; индикатор в шапке списка. При ошибке refresh — toast «Не удалось обновить» без очистки списка.

## Сценарии и переходы

| # | Сценарий | Поведение |
| :-- | :-- | :-- |
| 1 | Первое открытие | Запрос `GET /slots?from=today&to=today+7d` → skeleton → список по дням |
| 2 | Скролл по дням | Sticky-заголовки дней; плавный вертикальный скролл |
| 3 | Тап на слот | Переход на SCR-004 с `slotId` |
| 4 | Тап «Период» | Открытие SCR-002; после «Применить» — перезагрузка списка, чип периода обновляется |
| 5 | Тап «Фильтры» | Открытие SCR-003; после «Применить» — перезагрузка/локальная фильтрация, badge на чипе |
| 6 | Pull-to-refresh | Повтор запроса с текущими параметрами периода и фильтров |
| 7 | Нет слотов (дефолт 7 дней) | Empty state FR-004 |
| 8 | Слот без свободных мест | Карточка с «Мест нет»; тап → SCR-004 (запись / лист ожидания) |
| 9 | Возврат с SCR-004 после записи | Опционально: обновить счётчик мест на карточке без полной перезагрузки |

## Данные с API

### Запрос списка слотов

```
GET /api/v1/slots
  ?dateFrom=2026-07-03      // ISO date, включительно
  &dateTo=2026-07-10        // ISO date, включительно
  &instructorIds=uuid,...   // опционально, из SCR-003
  &timeOfDay=morning|afternoon|evening  // опционально
  &level=beginner|intermediate|advanced // опционально
```

Дефолт без SCR-002: `dateFrom` = сегодня, `dateTo` = сегодня + 6 дней (7 календарных дней включая сегодня).

### Ответ (элемент массива `slots[]`)

| Поле | Тип | Отображение на SCR-001 |
| :-- | :-- | :-- |
| `id` | string | — (для навигации) |
| `startAt` | ISO 8601 | Время начала, группировка по дню |
| `format` / `zone` | string | Зона/формат на карточке |
| `instructor.id` | string | — |
| `instructor.name` | string | Имя инструктора |
| `instructor.avatarUrl` | string? | Аватар |
| `instructor.rating` | number (1–5) | ★ 4.8 (округление до 1 знака; при отсутствии — не показывать блок рейтинга) |
| `freeSpots` | number | «N мест» / «Мест нет» при 0 |
| `capacity` | number | Не на карточке MVP (только на SCR-004 при необходимости) |
| `price` | number? | «N ₽» если присутствует |
| `level` | enum | Не на карточке MVP (используется для фильтра) |
| `isBookable` | boolean | Если `false` (прокат исчерпан и т.п.) — бейдж «Недоступно», тап → SCR-004 с пояснением |

Клиентская сортировка внутри дня: по `startAt` по возрастанию.

## Правила и ограничения

- Только **русский** язык (Q 9.3).
- **Mobile-first**: целевая ширина 360–414 dp; использование одной рукой в зале скалодрома.
- Фильтр по формату болдеринг/трассы **не входит** в MVP (Q 1.6) — только инструктор, время суток, уровень (SCR-003).
- Дефолтный горизонт — **7 дней** (R-027, FR-001); расширение — только через SCR-002 (FR-002).
- Рейтинг инструктора на карточке — **публичный** агрегат (Q 5.3), не личная оценка пользователя.
- Цена информативная; оплата на месте (Q 7.1) — без акцента на оплату в приложении.
- Прошедшие слоты текущего дня: не показывать или показывать в конце дня с пометкой «Завершено» — **рекомендация**: скрывать слоты, у которых `startAt` + длительность < now (уточнить с бэкендом; по умолчанию бэкенд не отдаёт прошедшие).
- Доступность: контраст текста WCAG AA; размер тап-зон ≥ 44×44 pt.

## Заметки для дизайнера

- Экран — **самый частый** в приложении; визуальная иерархия: время → формат → инструктор + рейтинг → места/цена.
- Рейтинг помогает выбрать «звёздного» инструктора (бриф заказчика); не перегружать карточку — одна строка с ★ и числом достаточна.
- Карточки слотов визуально единообразны; различие форматов — только текстом, без цветовой кодировки зон в MVP.
- Чип активного периода на SCR-001 должен **отличаться** от дефолта (заливка/обводка), чтобы пользователь видел нестандартный диапазон.
- Badge на «Фильтрах» — число активных **категорий** (инструктор считается как 1, даже если выбрано несколько имён), не количество выбранных инструкторов.
- Skeleton и empty state прорисовать в UI-kit как переиспользуемые паттерны.
- Учесть safe area (вырез, home indicator) для нижней навигации и pull-to-refresh.
- Для wireframe достаточно 2–3 карточек в день + все 4 состояния (loading, content, empty, error).
