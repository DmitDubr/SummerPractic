# SCR-003 — Фильтры слотов

| Поле | Значение |
| :-- | :-- |
| ID | SCR-003 |
| Тип | Bottom sheet (модальный оверлей) |
| Приоритет | Must |
| Связь | UC-001; Q 1.6, 5.3; FR-003 (отображение отфильтрованных слотов) |

## Назначение

Уточнение списка тренировок на SCR-001 по трём критериям MVP: **инструктор**, **время суток**, **уровень**. Фильтр по формату (болдеринг / трассы) **намеренно исключён** из первой версии (Q 1.6). После применения обновляет список и показывает **badge с числом активных фильтров** на чипе «Фильтры» в SCR-001.

## Точки входа

- SCR-001: тап по чипу **«Фильтры»** (с badge или без).

## Точки выхода

| Действие | Результат |
| :-- | :-- |
| **Применить** | Закрытие sheet → SCR-001 с отфильтрованным списком, обновление badge |
| **Сбросить** | Очистка всех критериев → применение → SCR-001 без фильтров, badge скрыт |
| Свайп вниз / тап по backdrop | Закрытие без применения (сохраняются последние применённые значения) |
| Системная «Назад» | То же, что свайп вниз |

## Структура экрана

```
┌─────────────────────────────────────┐
│  ░░░░░░░ SCR-001 (затемнён) ░░░░░░░ │
├─────────────────────────────────────┤
│  ───  (drag handle)                 │
│  Фильтры                            │
│                                     │
│  Инструктор                         │
│  [ Все ] [Анна К.] [Игорь П.] ...   │  ← chips / multi-select list
│  [ Показать всех ▾ ]                │  ← если инструкторов >5
│                                     │
│  Время суток                        │
│  ( ) Утро    ( ) День    ( ) Вечер  │  ← single select, radio/chips
│                                     │
│  Уровень                            │
│  ( ) Начинающий  ( ) Средний        │

│                                     │

```

Контент sheet — вертикальный скролл, если не помещается на экране. Футер с кнопками — sticky.

## Элементы UI

### Заголовок

- **«Фильтры»** — semibold, 18–20 sp.

### Секция «Инструктор»

| Элемент | Описание |
| :-- | :-- |
| Заголовок секции | «Инструктор» |
| Режим выбора | **Множественный** (multi-select): можно выбрать одного или нескольких инструкторов |
| Список | Чипы с именем + мини-аватар (опционально) + **★ рейтинг** (Q 5.3), напр. «Анна К. ★ 4.8» |
| «Все» / снятие выбора | Чип «Все инструкторы» или снятие всех чипов = без фильтра по инструктору |
| Раскрытие | При >5 инструкторах — первые 4 + ссылка «Ещё N» → раскрытие полного списка или отдельный compact picker |

Данные инструкторов: `GET /api/v1/instructors` (кэш при открытии sheet).

### Секция «Время суток»

| Значение | API | Подпись в UI | Интервал (рекомендация) |
| :-- | :-- | :-- | :-- |
| `morning` | morning | **Утро** | 06:00–11:59 |
| `afternoon` | afternoon | **День** | 12:00–16:59 |
| `evening` | evening | **Вечер** | 17:00–23:59 |

- **Single select** (radio group или mutually exclusive chips).
- Состояние «не выбрано» = без фильтра по времени суток.
- Повторный тап на активный chip снимает выбор.

### Секция «Уровень»

| Значение | API | Подпись в UI |
| :-- | :-- | :-- |
| `beginner` | beginner | **Начинающий** |
| `intermediate` | intermediate | **Средний** |
| `advanced` | advanced | **Продвинутый** |

- **Single select**; без принудительной проверки уровня клиента при записи (Q 2.2).
- Состояние «не выбрано» = все уровни.

### Кнопки действий

| Кнопка | Поведение |
| :-- | :-- |
| **Сбросить** | Очистить инструкторов, время суток, уровень; применить и закрыть |
| **Применить** | Сохранить выбор, закрыть, обновить SCR-001 |

«Применить» активна всегда (в т. ч. при пустом выборе = показать все слоты периода).

### Badge на SCR-001 (связанный элемент)

- На чипе «Фильтры» в SCR-001: круглый badge с числом **активных категорий** (0–3):
  - +1 если выбран ≥1 инструктор
  - +1 если выбрано время суток
  - +1 если выбран уровень
- Пример: выбраны 2 инструктора + «Вечер» → badge **«2»**, не «3».

## Состояния (loading / empty / error / offline)

### Loading

- При открытии sheet: если список инструкторов не в кэше — skeleton чипов в секции «Инструктор» (3–5 placeholder).
- Секции «Время суток» и «Уровень» — статичны, видны сразу.

### Empty

- Список инструкторов пуст (крайний случай): текст «Инструкторы пока не добавлены»; секция disabled.
- Результат «0 слотов» после применения — empty state на SCR-001, не в sheet.

### Error

- Ошибка загрузки инструкторов: «Не удалось загрузить список инструкторов» + кнопка **«Повторить»** в секции.
- Фильтры по времени и уровню доступны даже при ошибке инструкторов.

### Offline

- Нет сети, нет кэша инструкторов: секция инструкторов в error; время/уровень работают.
- Есть кэш: показать кэшированный список с пометкой «Обновлено …» (опционально).

## Сценарии и переходы

| # | Сценарий | Поведение |
| :-- | :-- | :-- |
| 1 | Первое открытие | Все секции без выбора (или последние применённые значения) |
| 2 | Выбор 2 инструкторов + Вечер | Preview в sheet; по «Применить» — запрос с query params |
| 3 | Только уровень «Начинающий» | Badge «1» на SCR-001 |
| 4 | Сбросить | Все критерии сняты, badge скрыт, полный список периода |
| 5 | Закрыть без применения | SCR-001 без изменений |
| 6 | Фильтр даёт 0 слотов | SCR-001 empty + подсказка «Сбросить фильтры» |
| 7 | Комбинация с периодом SCR-002 | Фильтры сохраняются при смене периода и наоборот |

## Данные с API

### Список инструкторов (для секции выбора)

```
GET /api/v1/instructors
```

| Поле | Использование |
| :-- | :-- |
| `id` | Параметр `instructorIds` |
| `name` | Подпись чипа |
| `avatarUrl` | Аватар (опционально) |
| `rating` | ★ на чипе (Q 5.3) |

### Фильтрация слотов (совместно с SCR-001)

```
GET /api/v1/slots
  ?dateFrom=...
  &dateTo=...
  &instructorIds=uuid1,uuid2    // если выбраны инструкторы
  &timeOfDay=evening            // morning | afternoon | evening
  &level=beginner               // beginner | intermediate | advanced
```

Фильтрация на стороне **бэкенда** (предпочтительно). Клиентская фильтрация допустима только если API не поддерживает params в MVP — зафиксировать в реализации.

### Маппинг времени суток

Границы интервалов задаёт бэкенд; клиент передаёт только enum. При расхождении с локальным временем — источник истины бэкенд.

## Правила и ограничения

- **Не включать** фильтр «Формат: болдеринг / трассы» в MVP (Q 1.6).
- Инструктор — **multi-select**; время суток и уровень — **single-select** с возможностью снять выбор.
- Фильтры **не сбрасываются** при смене периода (SCR-002) и при pull-to-refresh.
- Проверка соответствия уровня клиента слоту **не выполняется** (Q 2.2) — фильтр только для удобства поиска.
- Рейтинг на чипе инструктора — публичный агрегат, тот же что на карточке слота.
- Только русский язык в подписях.
- Логика badge: количество **заполненных категорий**, не количество выбранных инструкторов.

## Заметки для дизайнера

- Визуально отделить три секции (отступы, dividers); порядок фиксирован: Инструктор → Время → Уровень.
- Для инструкторов предпочтительны **chips** с возможностью multi-select (заливка при выборе); для времени и уровня — **segmented control** или radio chips (один активный).
- Показ рейтинга на чипе инструктора помогает выбору без перехода на карточку слота (согласовано с Q 5.3).
- Не дублировать фильтр формата даже как disabled — чтобы не создавать ожидание в MVP.
- Прорисовать: sheet без выбора, с частичным выбором (badge «2» на SCR-001), состояние загрузки инструкторов.
- Кнопка «Сбросить» должна быть заметна, но вторична относительно «Применить».
- Учесть длинные имена инструкторов — обрезка с ellipsis, полное имя в accessibility label.
- Sticky footer не перекрывает последнюю секцию при скролле (padding-bottom).
