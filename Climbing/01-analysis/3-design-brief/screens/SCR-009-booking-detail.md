# SCR-009 — Деталь записи

| Поле | Значение |
| :-- | :-- |
| **ID** | SCR-009 |
| **Название** | Деталь записи |
| **Тип** | Экран (стек поверх SCR-008 или deep link) |
| **Приоритет** | Must |
| **Use Case / FR** | UC-003, UC-004, UC-005; FR-007–011; Q 4.1 |
| **Связанные экраны** | SCR-008 (назад), SCR-010 (отмена), SCR-011 (оценка), SCR-001 (перезапись) |

---

## Назначение

Показать полную информацию по одной брони и контекстные действия в зависимости от статуса. Экран — **целевая страница deep link** из push-уведомлений (отмена скалодромом, напоминания). Поддерживает офлайн-просмотр закэшированной детали (Q 9.2).

---

## Точки входа

| Источник | Действие |
| :-- | :-- |
| SCR-008 | Тап по карточке брони |
| Push (отмена скалодромом, FR-010) | Deep link `app://bookings/{id}` → SCR-009 |
| Push (напоминание о тренировке) | Deep link на SCR-009 активной брони |
| SCR-006 | Опционально «Открыть запись» сразу после бронирования |

---

## Точки выхода

| Действие | Куда |
| :-- | :-- |
| Назад / свайп | SCR-008 |
| «Отменить запись» (статус active) | SCR-010 |
| «Выбрать другую тренировку» (cancelled_by_gym) | SCR-001 |
| «Оценить инструктора» (attended, не оценено) | SCR-011 |
| «Покинуть очередь» (waitlist) | Диалог подтверждения → SCR-008 с toast |

## Элементы UI

### Общие (все статусы)

| Элемент | Описание |
| :-- | :-- |
| Кнопка «Назад» | Возврат на SCR-008 |
| Бейдж статуса | Тот же набор, что на SCR-008 |
| Дата и время слота | День недели, дата, интервал ~1,5 ч |
| Блок «Информация о тренировке» | Формат/зона, инструктор (с рейтингом, Q 5.3), адрес скалодрома, цена |
| Блок «Ваша запись» | Своё снаряжение / прокат (что выбрано при бронировании) |
| Офлайн-баннер | Как на SCR-008 |

### Блоки по статусу

| Статус | Дополнительный UI | Основной CTA | Вторичный CTA |
| :-- | :-- | :-- | :-- |
| **active** | Инфо: «Место забронировано» | **Отменить запись** (destructive/outline) | — |
| **cancelled_by_client** | Текст: «Вы отменили эту запись»; дата отмены | — | — |
| **cancelled_by_gym** | Alert-блок: причина отмены (`cancellation_reason`); иконка предупреждения | **Выбрать другую тренировку** → SCR-001 | — |
| **attended** | Текст: «Тренировка завершена» | **Оценить инструктора** → SCR-011 (если `rating == null`) | — |
| **attended** (оценено) | «Вы оценили инструктора: ★★★★☆» | — | — |
| **waitlist** | Блок очереди: «Вы в листе ожидания», **место в очереди: N** | **Покинуть очередь** (destructive) | — |

**Причина отмены скалодромом** — в выделенном блоке (фон warning/light), текст от бэкенда, например: «Профилактика зоны: перекрутка трасс».

## Состояния

| Состояние | Поведение |
| :-- | :-- |
| **Загрузка** | Скелетон заголовка и полей |
| **Данные (онлайн)** | Полная деталь; действия активны |
| **Офлайн с кэшем** | Баннер; деталь из кэша; кнопки действий disabled или с пояснением «Требуется интернет» |
| **Офлайн без кэша** | «Запись недоступна офлайн» + «Назад» |
| **Ошибка загрузки** | Сообщение + «Повторить» |
| **Не найдено (404)** | «Запись не найдена» + возврат на SCR-008 |

---

## Сценарии и переходы

1. **Просмотр активной брони:** клиент видит детали → тап «Отменить запись» → SCR-010.
2. **Отмена скалодромом (UC-005):** push → deep link на SCR-009 → бейдж «Отменена скалодромом» + причина → CTA «Выбрать другую тренировку» → SCR-001 (тот же слот недоступен для записи, FR-011).
3. **Посещённая, не оценена:** CTA «Оценить инструктора» → SCR-011 → после оценки возврат, CTA скрыт, показаны звёзды.
4. **Лист ожидания:** показ позиции → «Покинуть очередь» → confirm dialog → `DELETE /bookings/{id}/waitlist` → toast «Вы вышли из очереди» → SCR-008.
5. **Освободилось место (push):** deep link на SCR-009; статус сменился на `active` — обновить UI и бейдж.
6. **Возврат с SCR-010 после отмены:** статус `cancelled_by_client`; кнопка отмены скрыта.

## Данные с API

**Запрос:** `GET /bookings/{id}`

| Поле | Использование |
| :-- | :-- |
| `id` | Deep link, навигация |
| `status` | Бейдж и условные блоки |
| `slot.starts_at`, `slot.ends_at` | Дата/время |
| `slot.format`, `slot.zone` | Формат тренировки |
| `slot.instructor.id`, `.name`, `.rating` | Инструктор |
| `slot.price` | Цена (оплата на месте) |
| `gym.address` | Адрес |
| `equipment` | Своё / прокат (детализация) |
| `cancellation_reason` | Только для `cancelled_by_gym` |
| `cancelled_at` | Для отменённых клиентом / скалодромом |
| `waitlist_position` | Для `waitlist` |
| `instructor_rating` | null = можно оценить; объект/число = уже оценено |
| `slot.id` | Для проверки запрета повторной записи (FR-011 — на бэкенде) |

**Действия:**

| Действие | API |
| :-- | :-- |
| Покинуть очередь | `DELETE /bookings/{id}` или `POST /bookings/{id}/leave-waitlist` (по контракту) |
| Отмена | Через SCR-010 → `POST /bookings/{id}/cancel` |

## Правила и ограничения

- Deep link из push при отмене скалодромом (FR-010, Q 4.1) **обязателен** в MVP.
- CTA «Выбрать другую тренировку» ведёт на SCR-001 **без** предзаполнения отменённого слота; повторная запись на тот же слот запрещена (FR-011, R-008) — UI на SCR-004 покажет слот недоступным.
- Оценка доступна **только** для `attended` и если `instructor_rating == null` (FR-012, Q 5.1).
- Отмена активной брони — только онлайн.
- Поздняя отмена (< 1 ч) инициируется здесь, предупреждение показывается на SCR-010 (Q 3.1, 3.2).
- Заблаговременная отмена (≥ 1 ч): место освобождается сразу (FR-008, Q 3.4).

---

## Заметки для дизайнера

- Экран должен хорошо читаться при открытии из push: статус и причина отмены скалодромом — **выше сгиба**, без прокрутки до CTA.
- Блок причины отмены скалодромом — визуально отличим (warning), не путать с ошибкой сети.
- Кнопка «Отменить запись» — destructive, но не primary; основной акцент на информации о тренировке.
- Для `waitlist` позиция в очереди — крупно и понятно («Вы 3-й в очереди»).
- Рейтинг инструктора в блоке информации — мелкие звёзды + число (как на SCR-004).
- После оценки не показывать CTA повторно; можно дать ссылку «Изменить оценку» — **не в MVP** (Q 5.1: без ограничений по сроку, но одна оценка).
- Согласовать визуал бейджей с SCR-008.
- При disabled-действиях в офлайне — tooltip или подпись под кнопкой, а не молчаливый gray-out.
